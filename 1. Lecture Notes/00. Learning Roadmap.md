# Learning Roadmap

This guide helps you choose which lecture notes to study based on your goal. You don't need to read everything — pick the path that fits where you want to end up.

---

## The Core Foundation (Everyone)

Regardless of which path you take, start here. These notes cover tools and concepts every developer needs.

| # | Topic | Why it matters |
| - | ----- | -------------- |
| 01 | VS Code Basics | Your editor — learn it well |
| 02 | CLI and Terminal Basics | You'll live in the terminal |
| 03 | Git and GitHub | Version control is non-negotiable |
| 04 | HTML Foundations | The structure of every web page |
| 05 | CSS Foundations | How pages look |

After these five, the paths diverge.

## Path 1 — Frontend Developer

You build what users see and interact with. You'll work primarily in the browser using JavaScript and React.

**Stack**: HTML → CSS → JavaScript → TypeScript → React → Next.js

```
Core Foundation (01–05)

        ▼

07. Advanced HTML and CSS
08. SEO and Accessibility
        │
        ▼
09. JavaScript Foundations
10. JavaScript Intermediate
11. JavaScript Advanced
        │
        ▼
12. TypeScript Foundations      ← learn this before React
13. TypeScript Intermediate
        │
        ▼
15. APIs                        ← how to talk to backends
16. React Foundations
17. React Intermediate
18. React Advanced
        │
        ▼
19. Next.js Foundations         ← the production-grade React framework

```

**Optional extras for frontend:**
- `14. TypeScript Advanced` — if you want deep TS mastery
- `26. Web Security` — understanding XSS, CSRF from the client side

**What you'll be able to build**: responsive websites, single-page applications, interactive UIs, dashboards, and full frontend applications that consume APIs built by others.

## Path 2 — Backend Developer

You build the server, the database, and the API. The two main backend paths use different languages — pick one to start with.

### Path 2a — Node.js Backend (JavaScript)

Stays in JavaScript, so it's a natural extension of frontend knowledge.

```
Core Foundation (01–05)
        │
        ▼
09. JavaScript Foundations
10. JavaScript Intermediate
11. JavaScript Advanced
12. TypeScript Foundations
        │
        ▼
15. APIs                        ← how REST APIs work
21. Node.js Foundations
22. Node.js Advanced
        │
        ▼
23. SQL Foundations (Databases)
26. Web Security
```

**Database choice — add one:**
- `28. MongoDB` — if your data is flexible/document-shaped, or you're building fast
- `23. SQL Foundations` — if your data is relational (users, orders, products, finance)

### Path 2b — Python Backend

Python is cleaner for data-heavy work and has an excellent ecosystem for APIs and automation.

```
Core Foundation (01–05)
        │
        ▼
15. APIs
24. Python Foundations
        │
        ▼
23. SQL Foundations (Databases)
        │
        ├── 25. Python Full Stack (FastAPI)   ← lean, fast, modern
        │
        └── 27. Django                        ← full-featured, built-in admin
```

**Which Python framework?**

| Choose FastAPI (25) if... | Choose Django (27) if... |
| ------------------------- | ------------------------ |
| You're building a pure JSON API | You need an admin panel |
| You want async-first performance | You want auth, ORM, and routing built in |
| Your frontend is separate (React) | You're building a traditional web app |
| You prefer assembling your own stack | You want "batteries included" |

**Optional extras for Python backend:**
- `28. MongoDB` — if you want a NoSQL database instead of PostgreSQL
- `29. Pandas and NumPy` — if your backend does data processing or analytics

## Path 3 — Full Stack Developer (JavaScript)

You build everything — frontend and backend — using JavaScript/TypeScript throughout. This is the most common path for solo developers and startups.

**Stack**: HTML → CSS → JavaScript → TypeScript → React → Next.js → Node.js → Database

```
Core Foundation (01–05)
        │
        ▼
06. Intermediate HTML and CSS
07. Advanced HTML and CSS
08. SEO and Accessibility
        │
        ▼
09. JavaScript Foundations
10. JavaScript Intermediate
11. JavaScript Advanced
12. TypeScript Foundations
13. TypeScript Intermediate
        │
        ▼
15. APIs
16. React Foundations
17. React Intermediate
18. React Advanced
        │
        ▼
19. Next.js Foundations         ← Next.js can handle both frontend AND backend (API routes)
20. Next.js Advanced
        │
        ▼
21. Node.js Foundations
22. Node.js Advanced
        │
        ▼
23. SQL Foundations (Databases)
26. Web Security
```

**Database choice — add one or both:**

| `28. MongoDB` | `23. SQL Foundations` |
| ------------- | --------------------- |
| Great with Next.js and Node.js — all JavaScript | Better for relational data and complex queries |
| Schema-flexible — fast to iterate | Strict schema — better data integrity |
| Common choice for startups and MVPs | Common choice for finance, e-commerce, SaaS |

**What you'll be able to build**: complete web applications — frontend, backend API, and database — fully independently.

You know JavaScript for the frontend and Python for the backend. Common in companies that use Python for data work or machine learning alongside a React frontend.

## Path 5 — Data Science / Data Engineering

You work with data — cleaning it, analysing it, and extracting meaning from it. Web development is secondary; Python and data libraries are primary.

```
Core Foundation (01–05)
        │
        ▼
24. Python Foundations
        │
        ▼
23. SQL Foundations (Databases)   ← SQL is essential for data work
        │
        ▼
29. Pandas and NumPy
        │
        ▼
25. Python Full Stack (FastAPI)   ← optional: expose your analysis as an API
27. Django                        ← optional: build a data dashboard
30. DevOps and AWS                ← optional: deploy models and pipelines
```
