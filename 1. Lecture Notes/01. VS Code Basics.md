# VS Code Basics

---

## What Is VS Code?

Visual Studio Code (VS Code) is a free, open-source code editor made by Microsoft. It's the most popular editor in the world for web development — fast, extensible, and works on Mac, Windows, and Linux. VS Code is not a full IDE like IntelliJ or Xcode — it's a lightweight editor that becomes powerful through extensions.

---

## Installation

Download from [code.visualstudio.com](https://code.visualstudio.com). The installer handles everything.

### Opening a project

```bash
# From the terminal: open the current folder in VS Code
code .

# Open a specific folder
code ~/Desktop/my-project
```

Or: File → Open Folder → navigate to your project.

## The Interface

```
┌─────────────────────────────────────────────────────────────────┐
│  Activity Bar  │          Editor Tabs                           │
│  (left strip)  ├─────────────────────────────────────────────── │
│                │                                                │
│  📁 Explorer   │   // index.js                                 │
│  🔍 Search     │   const app = express();                      │
│  🌿 Source     │   app.listen(3000);                           │
│     Control    │                                                │
│  🔧 Extensions │                                                │
│  🐛 Debug      ├─────────────────────────────────────────────── │
│                │   TERMINAL                                     │
│  Status Bar    │   $ npm run dev                               │
└─────────────────────────────────────────────────────────────────┘
```

| Area | Purpose |
| ---- | ------- |
| **Activity Bar** | Left strip of icons — switch between Explorer, Search, Git, etc. |
| **Primary Sidebar** | Shows whatever the active Activity Bar icon manages (files, search results, etc.) |
| **Editor** | Where you write code. Open multiple files as tabs. |
| **Terminal** | Integrated terminal — `` Ctrl+` `` to open/close |
| **Status Bar** | Bottom bar — shows branch, errors/warnings, language mode, encoding |

## The Command Palette

`Cmd+Shift+P` (Mac) or `Ctrl+Shift+P` (Win) opens the Command Palette — you can type any command here. Some useful ones:

- `> Format Document` — format current file
- `> Sort Lines Ascending` — sort selected lines alphabetically
- `> Toggle Word Wrap` — wrap long lines
- `> Change Language Mode` — switch syntax highlighting
- `> Open User Settings (JSON)` — edit settings directly
- `> Preferences: Open Keyboard Shortcuts` — browse all shortcuts
- `> Git: Commit` — commit staged changes

## Settings

Open settings with `Cmd+,` (Mac) or `Ctrl+,` (Win). You can also edit the raw JSON directly.

### Recommended settings (settings.json)

```json
{
  // Format on save — always keep code formatted
  "editor.formatOnSave": true,
  "editor.defaultFormatter": "esbenp.prettier-vscode",

  // Use Prettier for these file types
  "[javascript]": { "editor.defaultFormatter": "esbenp.prettier-vscode" },
  "[typescript]": { "editor.defaultFormatter": "esbenp.prettier-vscode" },
  "[json]": { "editor.defaultFormatter": "esbenp.prettier-vscode" },

  // Editor appearance
  "editor.fontSize": 14,
  "editor.fontFamily": "JetBrains Mono, Fira Code, monospace",
  "editor.fontLigatures": true,    // enables ligatures like → ≠ <=
  "editor.lineHeight": 1.6,
  "editor.wordWrap": "on",
  "editor.minimap.enabled": false,  // the tiny code preview on the right

  // Tab settings
  "editor.tabSize": 2,
  "editor.detectIndentation": true,

  // Files
  "files.autoSave": "onFocusChange",  // save when you click away
  "files.trimTrailingWhitespace": true,
  "files.insertFinalNewline": true,
  "editor.bracketPairColorization.enabled": true,
  "editor.guides.bracketPairs": true,  // coloured bracket guides

  // Terminal
  "terminal.integrated.fontSize": 13,

  // Explorer
  "explorer.confirmDelete": false,    // no confirmation when deleting files
  "explorer.confirmDragAndDrop": false
}
```

## Integrated Git

VS Code has excellent built-in Git support. The Source Control panel (`Ctrl+Shift+G`) shows changed files. Click any file to see a diff. You can stage, unstage, commit, push, and pull without leaving VS Code.

GitLens (extension) enhances this significantly — hover over any line of code to see who last changed it and when, view full file history, and explore branch history visually.

## Debugging

VS Code has a built-in debugger that works for JavaScript, TypeScript, Python, and more — no print statements needed.

```json
// .vscode/launch.json — configure the debugger
{
  "version": "0.2.0",
  "configurations": [
    {
      "name": "Debug Node.js",
      "type": "node",
      "request": "launch",
      "program": "${workspaceFolder}/src/index.js"
    },
    {
      "name": "Debug Python",
      "type": "python",
      "request": "launch",
      "program": "${workspaceFolder}/main.py"
    }
  ]
}
```

1. Click in the gutter (left of line numbers) to set a **breakpoint** (red dot)
2. Press `F5` to start debugging
3. Code will pause at the breakpoint
4. Use the debug toolbar: Step Over (`F10`), Step Into (`F11`), Continue (`F5`)
5. Hover over variables to see their current values

## Workspaces and Multi-root

If you have a project with separate frontend and backend folders, you can open both as a **workspace**:

File → Add Folder to Workspace → select the second folder.

VS Code will show both folder trees in the Explorer, and you can have separate settings and extensions per workspace.

## The `.vscode` Folder

A `.vscode` folder in your project can contain project-specific settings that are shared with your team:

```
.vscode/
├── settings.json     — project-specific editor settings
├── launch.json       — debugger configurations
├── extensions.json   — recommended extensions for this project
└── tasks.json        — custom build/test tasks
```

```json
// .vscode/extensions.json — prompt teammates to install these
{
  "recommendations": [
    "esbenp.prettier-vscode",
    "dbaeumer.vscode-eslint",
    "eamodio.gitlens"
  ]
}
```
