# CLI and Terminal Basics

---

## What Is the Terminal?

The **terminal** (also called the command line, shell, or console) is a text-based interface to your computer. Instead of clicking icons, you type commands. This might feel slower at first — but the terminal is dramatically more powerful than any GUI for programming tasks.

Every professional developer uses the terminal daily. It's where you run your code, install packages, manage files, use git, deploy applications, and interact with servers.

### Terminal vs Shell

- **Terminal** — the window/application you type in (e.g. Terminal.app on Mac, Windows Terminal, VS Code's integrated terminal)
- **Shell** — the program that interprets your commands (e.g. `bash`, `zsh`, `fish` on Mac/Linux; `PowerShell`, `cmd` on Windows)

Most developers on Mac/Linux use **zsh** (default on Mac since Catalina) or **bash**. On Windows, most developers use **Git Bash**, **WSL** (Windows Subsystem for Linux), or **PowerShell**. The commands in this guide work in bash/zsh.

## Navigating the Filesystem

Your terminal always has a **current working directory** — the folder you're "inside". All relative paths are relative to this.

```bash
# Print Working Directory — show where you are
pwd
# /Users/alice/Desktop/my-project

# List files and folders in the current directory
ls
ls -l    # detailed view (permissions, size, date)
ls -la   # include hidden files (names starting with .)
ls -lh   # human-readable file sizes

# Change Directory
cd Documents          # go into Documents folder
cd ..                 # go up one level (parent folder)
cd ../..              # go up two levels
cd ~                  # go to your home directory (/Users/you)
cd -                  # go back to the previous directory
cd "My Projects"      # quote paths with spaces

# Shortcuts
~   # home directory (/Users/you)
.   # current directory
..  # parent directory
```

## Working with Files and Folders

```bash
# Create a directory
mkdir my-folder
mkdir -p path/to/nested/folder   # create all intermediate folders

# Create a file
touch index.html     # creates empty file (or updates timestamp if exists)
touch file1.txt file2.txt file3.txt  # multiple at once

# Copy files
cp source.txt destination.txt
cp -r source-folder/ destination-folder/   # -r for directories (recursive)

# Move and rename
mv old-name.txt new-name.txt     # rename a file
mv file.txt Documents/           # move to a folder
mv file.txt Documents/file2.txt  # move AND rename

# Delete (be careful — no recycle bin in the terminal!)
rm file.txt                      # delete a file
rm -r folder/                    # delete a folder and everything inside
rm -rf folder/                   # force delete (no prompts) — dangerous

# Print file contents to the terminal
cat file.txt                     # entire file
head -20 file.txt                # first 20 lines
tail -20 file.txt                # last 20 lines
tail -f logs/app.log             # follow a file as it grows (useful for logs)

# Search inside files
grep "error" app.log             # find lines containing 'error'
grep -r "TODO" src/              # search recursively in a folder
grep -n "TODO" src/index.js      # show line numbers
```

## Environment Variables

Environment variables are key-value pairs accessible to any running process. They're used to pass configuration (API keys, database URLs) to your app without hardcoding them.

```bash
# Print all environment variables
env
printenv

# Print a specific one
echo $HOME
echo $PATH    # directories where the shell looks for commands
echo $USER    # your username

# Set one temporarily (only for the current shell session)
export MY_VAR="hello"
echo $MY_VAR  # hello

# Set one for a single command
PORT=4000 node server.js

# Permanently add to shell (add to ~/.zshrc or ~/.bashrc)
export MY_API_KEY="abc123"
```

In projects, you store environment variables in a `.env` file and load them with a library (`dotenv` in Node.js, `python-dotenv` in Python). Never commit `.env` to git.

## Useful Commands

```bash
# Clear the terminal screen
clear    # or Ctrl+L

# Search command history
history           # see recent commands
Ctrl+R            # search backwards through history — type to filter

# Stop a running process
Ctrl+C            # interrupt (stop) the running command

# Run two commands in sequence
npm install && npm run dev   # run second only if first succeeds
npm install ; npm run dev    # run second regardless

# Run in background
npm run dev &     # runs in background, terminal stays usable

# Pipe — send output of one command as input to another
ls -la | grep ".json"     # list files, then filter for .json files
cat app.log | tail -50    # show file, then take last 50 lines

# Redirect output to a file
npm run build > build.log 2>&1   # save stdout and stderr to file

# Find files
find . -name "*.js"              # find all .js files from current directory
find . -name "*.js" -not -path "*/node_modules/*"  # exclude node_modules

# Get help
man ls        # manual page for 'ls' (press q to quit)
ls --help     # many commands support --help
```

## Shell Configuration

Your shell reads a configuration file when it starts. You can add aliases, functions, and environment variables here.

```bash
# For zsh (Mac default) — edit ~/.zshrc
# For bash — edit ~/.bashrc
nano ~/.zshrc    # open in nano editor
code ~/.zshrc    # open in VS Code

# After editing, apply changes to current session:
source ~/.zshrc
```

### Useful aliases to add

```bash
# In ~/.zshrc or ~/.bashrc:
alias ll="ls -la"                    # detailed list with hidden files
alias ..="cd .."                     # go up one level
alias ...="cd ../.."                 # go up two levels
alias gs="git status"
alias gc="git commit"
alias gp="git push"
alias dev="npm run dev"

# Make a directory and immediately go into it
mkcd() { mkdir -p "$1" && cd "$1"; }
```

## Process Management

```bash
# See running processes
ps aux            # all processes
ps aux | grep node  # filter for node processes

# Kill a process by PID (process ID)
kill 1234          # send SIGTERM (graceful shutdown request)
kill -9 1234       # send SIGKILL (force kill — instant)

# Find what's using a port
lsof -i :3000      # show process using port 3000
# If port 3000 is busy and you need it:
lsof -i :3000 | grep LISTEN | awk '{print $2}' | xargs kill
```

## SSH — Remote Servers

SSH (Secure Shell) lets you connect to a remote computer and run commands on it as if you were sitting in front of it.

```bash
# Connect to a server
ssh username@server-ip
ssh alice@192.168.1.100
ssh alice@myserver.com

# Connect using an SSH key file (more secure than password)
ssh -i ~/.ssh/my-key.pem alice@myserver.com

# Copy files to/from server
scp local-file.txt alice@server:/home/alice/
scp alice@server:/home/alice/remote-file.txt ./

# Copy a whole folder
scp -r ./build alice@server:/var/www/app/

# More efficient file sync (only copies changed files)
rsync -avz ./build alice@server:/var/www/app/
```

## Terminal Shortcuts

| Shortcut | What it does |
| -------- | ------------ |
| `Ctrl+C` | Stop the current command |
| `Ctrl+D` | Exit the shell (logout) |
| `Ctrl+L` | Clear the screen |
| `Ctrl+R` | Search command history |
| `Ctrl+A` | Move cursor to start of line |
| `Ctrl+E` | Move cursor to end of line |
| `Ctrl+U` | Delete from cursor to start of line |
| `Ctrl+K` | Delete from cursor to end of line |
| `Tab` | Autocomplete file/folder/command names |
| `Tab Tab` | Show all possible completions |
| `↑` / `↓` | Navigate command history |
| `!!` | Repeat the last command |
| `!$` | Last argument of the previous command |
