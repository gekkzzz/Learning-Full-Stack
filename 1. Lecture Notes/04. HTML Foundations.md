# HTML Foundations

---

## Introduction to HTML

HTML stands for **HyperText Markup Language**. It is the language used to build the structure and content of every webpage on the internet. Think of it as the skeleton of a webpage — it defines what is on the page (headings, paragraphs, images, links, buttons) but not how it looks. That job belongs to CSS.

Every webpage you've ever visited is built from HTML. When you view the source of any page in your browser, you'll see HTML tags wrapping all the content.

### The three languages of the web

| Language | What it does | Real-world analogy |
| -------- | ------------ | ------------------ |
| **HTML** | Defines the structure and content | The walls and rooms of a house |
| **CSS** | Controls the appearance and layout | The paint, furniture, and decoration |
| **JavaScript** | Adds interactivity and logic | The electrical wiring and plumbing |

---

## HTML Boilerplate

Every HTML file starts with the same basic structure. This is called the **boilerplate** — a template you copy at the start of every project.

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My Website</title>
    <link rel="stylesheet" href="style.css">
  </head>

  <body>
    <header>
      <h1>Welcome to My Website</h1>
    </header>

    <main>
      <p>This is where your content goes.</p>
    </main>

    <footer>
      <p>&copy; 2024 Your Name</p>
    </footer>
  </body>
</html>
```

Here is what each part does:

- **`<!DOCTYPE html>`** — Tells the browser this is an HTML5 document. Always put this on the very first line. Without it, browsers may render the page incorrectly.
- **`<html lang="en">`** — The root element that wraps everything. The `lang="en"` tells search engines and screen readers the page is in English.
- **`<head>`** — An invisible section that contains information *about* the page. Nothing inside `<head>` appears visually on the page.
- **`<meta charset="UTF-8">`** — Tells the browser to use the UTF-8 character encoding, which supports virtually every character and symbol in the world (including emoji).
- **`<meta name="viewport" ...>`** — Makes the page scale correctly on mobile devices. Without this, mobile browsers zoom out and show a tiny version of the desktop layout.
- **`<title>`** — Sets the text shown in the browser tab and in search engine results.
- **`<link rel="stylesheet" href="style.css">`** — Loads an external CSS file to style the page.
- **`<body>`** — Everything the user actually *sees* on the page goes inside here.

## What is an HTML Tag?

HTML is made up of **tags**. A tag is a keyword wrapped in angle brackets. Most tags come in pairs — an **opening tag** and a **closing tag** — with content in between.

```html
<p>This is a paragraph of text.</p>
```

- `<p>` is the opening tag
- `</p>` is the closing tag (note the forward slash)
- The text between them is the **content**

Together, the opening tag + content + closing tag is called an **element**.

Some tags are **self-closing** — they have no content and no separate closing tag:

```html
<img src="photo.jpg" alt="A photo">
<br>
<hr>
```

Tags can also have **attributes** — extra information placed inside the opening tag. For example:

```html
<a href="https://google.com" target="_blank">Visit Google</a>
```

Here, `href` and `target` are attributes. `href` sets the link destination, and `target="_blank"` tells the browser to open the link in a new tab.

## 1. Document Structure & Metadata

These tags form the essential backbone of any HTML document. They set up the page for the browser but are not visible to the user.

| Tag | Name | Description |
| --- | ---- | ----------- |
| `<!DOCTYPE html>` | Document Type | Tells the browser this is an HTML5 document. Must be on line 1. |
| `<html>` | HTML Root | Wraps everything on the page. Always includes `lang` attribute. |
| `<head>` | Document Head | Contains invisible metadata: title, character set, stylesheet links. |
| `<meta>` | Metadata | Provides extra info to the browser — character encoding, mobile scaling, SEO description. |
| `<title>` | Page Title | Sets the text shown in the browser tab and in search results. |
| `<link>` | External Resource | Links to external files, most commonly a CSS stylesheet. |
| `<body>` | Document Body | Contains all visible content — text, images, links, everything the user sees. |

## 2. Structural & Semantic Layout Elements

Semantic HTML tags give **meaning** to your layout. Instead of using a generic `<div>` for everything, you use tags whose names describe the purpose of the content inside them.

This matters for two reasons:

1. **Accessibility** — screen readers use semantic tags to help visually impaired users navigate a page.
2. **SEO** — search engines like Google use semantic structure to understand your content and rank it correctly.

```html
<body>
  <header>
    <h1>My Blog</h1>
    <nav>
      <a href="/">Home</a>
      <a href="/about">About</a>
      <a href="/contact">Contact</a>
    </nav>
  </header>

  <main>
    <article>
      <h2>My First Post</h2>
      <p>This is the article content...</p>
    </article>

    <aside>
      <h3>About the Author</h3>
      <p>A short bio goes here.</p>
    </aside>
  </main>

  <footer>
    <p>&copy; 2024 My Blog</p>
  </footer>
</body>
```

Here is what each semantic tag means:

- **`<header>`** — The introductory area at the top of a page or section. Usually contains a logo, site title, or navigation bar.
- **`<nav>`** — A block of navigation links. This tells browsers and screen readers "these links are for navigating the site".
- **`<main>`** — The primary, unique content of the page. There should only be **one** `<main>` per page. Sidebar and navigation do not belong here.
- **`<section>`** — A thematic group of content with its own heading. Use it when content forms a distinct chapter or topic area.
- **`<article>`** — A self-contained piece of content that makes sense on its own, even if you copy it elsewhere — like a blog post, news story, or product card.
- **`<aside>`** — Content that is related but not essential to the main content, like a sidebar, a tip box, or a related links panel.
- **`<footer>`** — The closing area of a page or section. Usually contains copyright info, contact links, or legal text.
- **`<div>`** — A generic block container with no semantic meaning. Use it only when none of the above tags fit.

> **Rule of thumb:** Always reach for a semantic tag first. Only use `<div>` if no semantic tag describes the content.

## 3. Text Formatting & Typography

These tags control how written content is structured and emphasised on the page.

### Headings & Paragraphs

Headings create a hierarchy of importance on the page — like chapters and sub-chapters in a book. There are six levels, `<h1>` through `<h6>`.

```html
<h1>Main Page Title</h1>
<h2>Section Heading</h2>
<h3>Sub-section Heading</h3>
<h4>Smaller heading</h4>

<p>This is a paragraph. It wraps in a block and has space above and below it automatically.</p>

<p>This is a second paragraph. Notice it starts below the first one.</p>
```

**Important rules for headings:**

- Only use **one `<h1>`** per page — it represents the main topic.
- Do not skip levels (don't go from `<h2>` straight to `<h5>`).
- Use headings for structure, not just to make text big. Use CSS for sizing.

| Tag | Name | Description |
| --- | ---- | ----------- |
| `<h1>` to `<h6>` | Headings | Section headings. `<h1>` is the most important. |
| `<p>` | Paragraph | A block of body text. Adds space above and below automatically. |
| `<br>` | Line Break | Forces a new line without starting a new paragraph. Self-closing. |
| `<hr>` | Horizontal Rule | Draws a horizontal dividing line across the page. Self-closing. |

---

### Emphasis & Importance

These inline tags change the meaning or appearance of words within a sentence.

```html
<p>
  This is <strong>very important</strong> information.
  This is <em>slightly stressed</em> information.
  This is <mark>highlighted</mark> text.
  The price was <s>£99.99</s> and is now £49.99.
</p>
```

| Tag | Renders as | Has semantic meaning? | Use for |
| --- | ---------- | --------------------- | ------- |
| `<strong>` | **Bold** | Yes — important | Content that is genuinely important |
| `<b>` | **Bold** | No | Visually bold text with no extra meaning (e.g., a keyword) |
| `<em>` | *Italic* | Yes — stressed | A word you'd stress if reading aloud |
| `<i>` | *Italic* | No | Technical terms, foreign words, or thoughts |
| `<mark>` | Highlighted | No | Search match results, relevant passages |
| `<s>` | ~~Strikethrough~~ | No | Old or crossed-out information (e.g., old prices) |
| `<small>` | Small text | No | Fine print, copyright notices, disclaimers |
| `<del>` | ~~Strikethrough~~ | Yes — deleted | Content that was removed from the document |
| `<ins>` | Underlined | Yes — inserted | Content that was added to the document |

> **`<strong>` vs `<b>` and `<em>` vs `<i>`:** Use `<strong>` and `<em>` when the emphasis has meaning (they affect how screen readers read the content aloud). Use `<b>` and `<i>` for purely visual styling.

### Superscript, Subscript & Abbreviations

```html
<!-- Superscript — exponents and footnotes -->
<p>The area is 25m<sup>2</sup>.</p>
<p>See footnote<sup>1</sup>.</p>

<!-- Subscript — chemical formulas -->
<p>Water is H<sub>2</sub>O.</p>

<!-- Abbreviation — hover over it to see the full text -->
<p>I'm learning <abbr title="HyperText Markup Language">HTML</abbr>.</p>
```

| Tag | Name | Use for |
| --- | ---- | ------- |
| `<sup>` | Superscript | Exponents (x²), footnote markers |
| `<sub>` | Subscript | Chemical formulas (H₂O), mathematical notation |
| `<abbr>` | Abbreviation | Acronyms and abbreviations — the `title` attribute shows the full form on hover |

---

### Quotations & Citations

```html
<!-- Block quote — for longer quotations from an external source -->
<blockquote cite="https://example.com/source">
  <p>The only way to do great work is to love what you do.</p>
  <footer>— <cite>Steve Jobs</cite></footer>
</blockquote>

<!-- Inline quote — for short quotations within a sentence -->
<p>Einstein once said, <q>Imagination is more important than knowledge.</q></p>
```

| Tag | Name | Use for |
| --- | ---- | ------- |
| `<blockquote>` | Block Quote | A long quotation displayed as its own block, usually indented |
| `<q>` | Inline Quote | A short quotation within a line of text — browsers add quotation marks automatically |
| `<cite>` | Citation | The title of a creative work (book, film, article) |

---

### Code & Preformatted Text

```html
<!-- Inline code — for a short snippet in a sentence -->
<p>Use the <code>console.log()</code> function to print to the console.</p>

<!-- Preformatted code block — preserves spaces and line breaks -->
<pre><code>
function greet(name) {
  return "Hello, " + name;
}
</code></pre>

<!-- Keyboard input -->
<p>Press <kbd>Ctrl</kbd> + <kbd>S</kbd> to save.</p>
```

| Tag | Name | Use for |
| --- | ---- | ------- |
| `<code>` | Code | Short code snippets inline within text |
| `<pre>` | Preformatted | Blocks where whitespace and line breaks must be preserved (wraps `<code>`) |
| `<kbd>` | Keyboard Input | Keys the user should press |
| `<samp>` | Sample Output | Output from a program or command |

---

### Generic Containers

`<div>` and `<span>` are containers with **no semantic meaning**. They exist purely to let you group elements so you can target them with CSS or JavaScript.

```html
<!-- div groups a block-level "card" -->
<div class="card">
  <h2>Card Title</h2>
  <p>Some description text with a <span class="highlight">key phrase</span> highlighted.</p>
</div>
```

| Tag | Default display | Use for |
| --- | --------------- | ------- |
| `<div>` | Block | Grouping multiple block elements for layout purposes |
| `<span>` | Inline | Targeting a word or phrase inside a line of text |

## 4. Hyperlinks & Multimedia

### `<a>` — Hyperlinks

The `<a>` (anchor) tag creates a clickable link to another page, a file, or a location on the same page.

```html
<!-- Link to another website -->
<a href="https://google.com">Go to Google</a>

<!-- Link to another page in the same project -->
<a href="/about.html">About Us</a>

<!-- Open in a new browser tab -->
<a href="https://google.com" target="_blank">Open Google in a new tab</a>

<!-- Link to a section on the same page (using an ID) -->
<a href="#contact">Jump to Contact section</a>

<!-- The section being linked to -->
<section id="contact">
  <h2>Contact</h2>
</section>

<!-- Email link — opens the user's mail app -->
<a href="mailto:hello@example.com">Send me an email</a>
```

Key attributes:

- `href` — The destination. Required. Can be a URL, a relative path, an `#id`, or `mailto:`.
- `target="_blank"` — Opens the link in a new tab instead of the current one.

---

### `<img>` — Images

The `<img>` tag embeds an image. It is **self-closing** — it has no closing tag.

```html
<!-- Basic image -->
<img src="cat.jpg" alt="A fluffy orange cat sitting on a chair">

<!-- Image from a URL -->
<img src="https://example.com/photo.jpg" alt="A mountain at sunset">

<!-- Image with explicit dimensions -->
<img src="logo.png" alt="Company logo" width="200" height="80">
```

Key attributes:

- `src` — The file path or URL of the image. Required.
- `alt` — A text description of the image. **Always include this.** It is read aloud by screen readers and displayed if the image fails to load. It also helps search engines understand the image.

> **Never leave `alt` empty unless the image is purely decorative** (background graphics, spacers). For decorative images, use `alt=""` — this signals to screen readers to skip it.

---

### `<video>` — Video

```html
<video src="clip.mp4" controls width="640">
  Your browser does not support the video element.
</video>

<!-- Multiple formats for browser compatibility -->
<video controls width="640">
  <source src="clip.mp4" type="video/mp4">
  <source src="clip.webm" type="video/webm">
  Your browser does not support the video element.
</video>
```

Key attributes:

- `controls` — Displays play/pause/volume controls. Without this, the video has no interface.
- `autoplay` — Starts playing automatically (most browsers mute it automatically if this is set).
- `loop` — Replays the video when it ends.
- `muted` — Starts the video muted.
- `width` / `height` — Sets the display size.

---

### `<audio>` — Audio

```html
<audio src="song.mp3" controls>
  Your browser does not support the audio element.
</audio>
```

Key attributes:

- `controls` — Shows play/pause/volume controls.
- `autoplay` — Starts playing automatically.
- `loop` — Replays when it ends.

## 5. Lists

HTML has three types of list. All of them use `<li>` (list item) for each individual item.

### Unordered List — Bulleted

Use when the **order doesn't matter** (a shopping list, a set of features).

```html
<ul>
  <li>Apples</li>
  <li>Bananas</li>
  <li>Oranges</li>
</ul>
```

Renders as:

- Apples
- Bananas
- Oranges

---

### Ordered List — Numbered

Use when the **order matters** (steps in a recipe, a ranked list).

```html
<ol>
  <li>Boil the water</li>
  <li>Add the pasta</li>
  <li>Cook for 10 minutes</li>
  <li>Drain and serve</li>
</ol>
```

Renders as:

1. Boil the water
2. Add the pasta
3. Cook for 10 minutes
4. Drain and serve

---

### Description List — Term / Definition

Use for glossaries, FAQs, or any key-value pair content.

```html
<dl>
  <dt>HTML</dt>
  <dd>HyperText Markup Language — the language used to structure web content.</dd>

  <dt>CSS</dt>
  <dd>Cascading Style Sheets — the language used to style web content.</dd>
</dl>
```

- `<dl>` — The description list wrapper
- `<dt>` — The term being defined
- `<dd>` — The definition or description

---

### Nesting Lists

Lists can be placed inside other lists to create sub-levels:

```html
<ul>
  <li>Frontend
    <ul>
      <li>HTML</li>
      <li>CSS</li>
      <li>JavaScript</li>
    </ul>
  </li>
  <li>Backend
    <ul>
      <li>Node.js</li>
      <li>Python</li>
    </ul>
  </li>
</ul>
```

## 6. Tables

Tables are for displaying structured **data** in rows and columns — like a spreadsheet. Do not use tables for page layout (use CSS Flexbox or Grid for that).

```html
<table>
  <thead>
    <tr>
      <th>Name</th>
      <th>Age</th>
      <th>City</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Alice</td>
      <td>28</td>
      <td>London</td>
    </tr>
    <tr>
      <td>Bob</td>
      <td>34</td>
      <td>Manchester</td>
    </tr>
  </tbody>
</table>
```

This renders as:

| Name | Age | City |
| ---- | --- | ---- |
| Alice | 28 | London |
| Bob | 34 | Manchester |

| Tag | Name | Description |
| --- | ---- | ----------- |
| `<table>` | Table Wrapper | The outer container for the entire table. |
| `<thead>` | Table Head | Groups the header row(s) at the top. |
| `<tbody>` | Table Body | Groups the data rows. |
| `<tr>` | Table Row | One horizontal row of cells. |
| `<th>` | Table Header Cell | A header cell — bold and centred by default. Goes inside `<thead>`. |
| `<td>` | Table Data Cell | A regular data cell. Goes inside `<tbody>`. |

## 7. Forms & Input Elements

Forms are how users send data to a website — logging in, signing up, submitting a search query, filling in a checkout form. The `<form>` element is the container, and various input elements go inside it.

```html
<form action="/submit" method="POST">

  <!-- Text input -->
  <label for="username">Username:</label>
  <input type="text" id="username" name="username" placeholder="Enter your username">

  <!-- Password input -->
  <label for="password">Password:</label>
  <input type="password" id="password" name="password">

  <!-- Email input -->
  <label for="email">Email:</label>
  <input type="email" id="email" name="email">

  <!-- Multi-line text -->
  <label for="message">Message:</label>
  <textarea id="message" name="message" rows="4"></textarea>

  <!-- Checkboxes -->
  <input type="checkbox" id="terms" name="terms">
  <label for="terms">I agree to the terms and conditions</label>

  <!-- Radio buttons — only one in the group can be selected -->
  <input type="radio" id="small" name="size" value="small">
  <label for="small">Small</label>

  <input type="radio" id="large" name="size" value="large">
  <label for="large">Large</label>

  <!-- Drop-down menu -->
  <label for="country">Country:</label>
  <select id="country" name="country">
    <option value="uk">United Kingdom</option>
    <option value="us">United States</option>
    <option value="au">Australia</option>
  </select>

  <!-- Submit button -->
  <button type="submit">Send</button>

</form>
```

Key elements:

| Element | What it does |
| ------- | ------------ |
| `<form>` | The wrapper. `action` sets where the data is sent; `method` sets how (`GET` or `POST`). |
| `<label>` | Provides a text label for an input. The `for` attribute must match the input's `id`. Clicking the label focuses the input. |
| `<input>` | The main input element. Its behaviour changes completely based on the `type` attribute. |
| `<textarea>` | A multi-line text box. Use `rows` and `cols` to set the size. |
| `<select>` | A drop-down menu. Contains `<option>` elements for each choice. |
| `<button>` | A clickable button. `type="submit"` submits the form; `type="button"` does nothing by default (use with JavaScript). |

**Why `<label>` matters:** Always link a `<label>` to its input using matching `for` and `id` values. This makes the input clickable via its label text, which is essential for accessibility — screen readers announce the label when the input is focused.

## 8. HTML Indentation & Readability

Good indentation makes HTML much easier to read, debug, and maintain. The rule is simple: **whenever a tag contains other tags, indent the children by one level (2 spaces).**

### Good vs. Bad

```html
<!-- Hard to read — no indentation -->
<body><main><section><h2>About</h2><p>Some text.</p></section></main></body>

<!-- Easy to read — properly indented -->
<body>
  <main>
    <section>
      <h2>About</h2>
      <p>Some text.</p>
    </section>
  </main>
</body>
```

### When to indent

Indent whenever a tag **contains** other tags:

```html
<ul>
  <li>Item one</li>
  <li>Item two</li>
  <li>Item three</li>
</ul>

<table>
  <thead>
    <tr>
      <th>Name</th>
      <th>Age</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Alice</td>
      <td>30</td>
    </tr>
  </tbody>
</table>
```

### When not to indent

Short inline content that fits on one line can stay on the same line as its tag:

```html
<!-- Fine on one line -->
<h1>Welcome to My Site</h1>
<p>A short paragraph.</p>
<li>A single item</li>

<!-- Break it out when content is long or contains nested tags -->
<p>
  This paragraph is long, or it contains
  <strong>nested elements</strong> that make it harder to scan inline.
</p>
```

### Quick rules

- Use **2 spaces** per indent level.
- Each closing tag lines up with its opening tag.
- Add a **blank line** between major sections (`<header>`, `<main>`, `<footer>`) to visually separate them.
- Self-closing tags (`<img>`, `<br>`, `<hr>`, `<meta>`) never have children — never indent inside them.

## Quick Concept Reminder: Block vs. Inline

**Block elements** (`<div>`, `<p>`, `<h1>`, `<section>`) always start on a new line and stretch to fill the full width of their parent container.

**Inline elements** (`<span>`, `<a>`, `<strong>`, `<img>`) do not start on a new line. They only take up as much width as their content and sit alongside other inline elements, like words in a sentence.

```html
<!-- These stack vertically (block) -->
<p>First paragraph</p>
<p>Second paragraph</p>

<!-- These sit side by side (inline) -->
<p>I am <strong>learning</strong> <em>HTML</em> right now.</p>
```
