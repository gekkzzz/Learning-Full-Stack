## Forms and User Input

Forms are how users send data to a server вҖ” login pages, search bars, checkout flows, and contact forms are all built with `<form>` elements.

```html
<form action="/submit" method="POST">
  <label for="name">Full Name</label>
  <input type="text" id="name" name="name" placeholder="Jane Smith" required>

  <label for="email">Email</label>
  <input type="email" id="email" name="email" required>

  <label for="role">Role</label>
  <select id="role" name="role">
    <option value="developer">Developer</option>
    <option value="designer">Designer</option>
  </select>

  <label for="message">Message</label>
  <textarea id="message" name="message" rows="4"></textarea>

  <button type="submit">Send</button>
</form>
```

### Key input types

| Type | Use case |
| ---- | -------- |
| `text` | General short text |
| `email` | Email address (validates format) |
| `password` | Hides characters as user types |
| `number` | Numeric input with optional min/max |
| `checkbox` | Boolean yes/no toggle |
| `radio` | Pick one from a group |
| `file` | Upload a file |
| `date` | Date picker |
| `range` | Slider |

Always pair every `<input>` with a `<label>` using matching `for` and `id` attributes. This improves accessibility and makes the label clickable to focus the input.

### Form validation

HTML5 provides built-in validation without any JavaScript:

```html
<input type="email" required>
<input type="text" minlength="3" maxlength="50">
<input type="number" min="1" max="100">
<input type="text" pattern="[A-Za-z]{3,}" title="Letters only, at least 3 characters">
```

The browser will block submission and show an error if the field fails validation.

### `box-sizing: border-box`

By default, `width` sets only the content area. Padding and border are *added on top*, making elements wider than expected. `border-box` fixes this by including padding and border inside the declared width.

```css
*, *::before, *::after {
  box-sizing: border-box; /* always include this at the top of your CSS */
}
```

## Flexbox

Flexbox is a layout model that arranges items in a single row or column with powerful alignment controls. Think of it as a smart shelf вҖ” the shelf is the **flex container**, and everything on it is a **flex item**.

```css
.container {
  display: flex;
  flex-direction: row;       /* row | column | row-reverse | column-reverse */
  justify-content: center;   /* alignment along the main axis */
  align-items: center;       /* alignment along the cross axis */
  gap: 16px;                 /* space between items */
  flex-wrap: wrap;           /* allow items to wrap to next line */
}
```

### Justify-content values

| Value | Effect |
| ----- | ------ |
| `flex-start` | Pack items to the start |
| `flex-end` | Pack items to the end |
| `center` | Centre items |
| `space-between` | First and last items at edges, equal gaps between |
| `space-around` | Equal space around every item |
| `space-evenly` | Truly equal space between all items and edges |

### Flex item properties

```css
.item {
  flex-grow: 1;    /* how much this item grows to fill space */
  flex-shrink: 1;  /* how much this item shrinks when space is tight */
  flex-basis: 200px; /* starting size before growing/shrinking */

  /* shorthand: grow shrink basis */
  flex: 1 1 200px;

  /* common shorthand: "take up equal share of remaining space" */
  flex: 1;
}
```

## CSS Grid

Grid is a two-dimensional layout system вҖ” it handles both rows *and* columns at the same time. Flexbox is great for one-dimensional strips; Grid is great for full page layouts.

```css
.grid {
  display: grid;
  grid-template-columns: 1fr 2fr 1fr; /* three columns: narrow, wide, narrow */
  grid-template-rows: auto 1fr auto;  /* header, main, footer */
  gap: 24px;
}
```

`fr` is a fractional unit вҖ” `1fr` means "one share of the remaining space."

### Placing items on the grid

```css
.header {
  grid-column: 1 / -1; /* span all columns (1 to last line) */
}

.sidebar {
  grid-column: 1 / 2;
  grid-row: 2 / 3;
}

.main {
  grid-column: 2 / 4;
  grid-row: 2 / 3;
}
```

### Grid template areas

A more readable way to define layouts:

```css
.grid {
  display: grid;
  grid-template-areas:
    "header header header"
    "sidebar main main"
    "footer footer footer";
  grid-template-columns: 200px 1fr 1fr;
}

.header  { grid-area: header; }
.sidebar { grid-area: sidebar; }
.main    { grid-area: main; }
.footer  { grid-area: footer; }
```

## Responsive Design

Responsive design means the layout adapts to the user's screen size вҖ” the same HTML looks right on a phone, tablet, and desktop.

### Media queries

```css
/* Default: mobile styles */
.container {
  flex-direction: column;
  padding: 16px;
}

/* Tablet and up */
@media (min-width: 768px) {
  .container {
    flex-direction: row;
    padding: 32px;
  }
}

/* Desktop and up */
@media (min-width: 1200px) {
  .container {
    max-width: 1100px;
    margin: 0 auto;
  }
}
```

**Mobile-first** is the recommended approach: write base styles for small screens, then use `min-width` media queries to progressively enhance for larger screens.

### Common breakpoints

| Name | Min width | Target |
| ---- | --------- | ------ |
| Mobile | вҖ” | Phones (default) |
| Tablet | 768px | iPads, small laptops |
| Desktop | 1024px | Laptops |
| Wide | 1280px+ | Large monitors |

### Relative units

| Unit | Relative to | Use case |
| ---- | ----------- | -------- |
| `%` | Parent element | Fluid widths |
| `em` | Current font size | Padding, margins that scale with text |
| `rem` | Root font size (usually 16px) | Font sizes, spacing (most common) |
| `vw` | Viewport width | Full-width sections |
| `vh` | Viewport height | Full-height hero sections |
| `clamp(min, ideal, max)` | Viewport | Fluid typography |

```css
h1 {
  font-size: clamp(1.5rem, 5vw, 3rem); /* never smaller than 1.5rem, never larger than 3rem */
}
```

## CSS Custom Properties (Variables)

CSS variables let you store values in one place and reuse them everywhere. When you change the variable, every use of it updates automatically.

```css
:root {
  --color-primary: #3b82f6;
  --color-background: #f8fafc;
  --color-text: #1e293b;
  --spacing-sm: 8px;
  --spacing-md: 16px;
  --spacing-lg: 32px;
  --radius: 8px;
}

.button {
  background-color: var(--color-primary);
  padding: var(--spacing-sm) var(--spacing-md);
  border-radius: var(--radius);
}
```

`:root` is the topmost element вҖ” variables declared here are available everywhere on the page.

## CSS Specificity

When multiple rules target the same element, the browser uses **specificity** to decide which one wins. Higher specificity beats lower specificity, regardless of order.

| Selector | Specificity score |
| -------- | ----------------- |
| Inline style (`style=""`) | 1000 |
| ID (`#id`) | 100 |
| Class, attribute, pseudo-class (`.class`, `[type]`, `:hover`) | 10 |
| Element, pseudo-element (`div`, `::before`) | 1 |

```css
p { color: black; }           /* specificity: 1 */
.text { color: blue; }        /* specificity: 10 вҖ” wins */
#intro { color: red; }        /* specificity: 100 вҖ” wins */
```

`!important` overrides everything, but using it is a sign your specificity architecture needs rethinking.

## Positioning

| Value | Behaviour |
| ----- | --------- |
| `static` | Default. Follows normal document flow. `top/left/right/bottom` have no effect. |
| `relative` | Stays in normal flow but can be nudged with `top/left/right/bottom`. Also creates a positioning context for children. |
| `absolute` | Removed from normal flow. Positioned relative to the nearest `relative`/`absolute`/`fixed` ancestor. |
| `fixed` | Removed from flow. Stays in the same spot on the screen as you scroll. |
| `sticky` | Stays in flow until a scroll threshold, then sticks like `fixed`. |

```css
.card {
  position: relative; /* establishes a context */
}

.badge {
  position: absolute;
  top: -8px;
  right: -8px; /* sits in the top-right corner of .card */
}

.navbar {
  position: sticky;
  top: 0; /* sticks to the top of the viewport once scrolled to */
}
```

## Pseudo-classes and Pseudo-elements

**Pseudo-classes** target an element in a particular *state*:

```css
a:hover   { color: blue; }       /* mouse is over the link */
a:visited { color: purple; }     /* link was already clicked */
input:focus { outline: 2px solid blue; } /* input is focused */
li:first-child { font-weight: bold; }
li:nth-child(even) { background: #f0f0f0; }
button:disabled { opacity: 0.5; }
```

**Pseudo-elements** create virtual elements you can style:

```css
p::first-line { font-weight: bold; }
p::first-letter { font-size: 2em; float: left; }

.button::before {
  content: "вҶ’ ";
}

.button::after {
  content: "";
  display: block;
  width: 100%;
  height: 2px;
  background: currentColor;
}
```

## Transitions and Animations

### Transitions

Transitions smoothly interpolate a property change triggered by a state change (like `:hover`):

```css
.button {
  background-color: #3b82f6;
  transform: scale(1);
  transition: background-color 0.2s ease, transform 0.15s ease;
}

.button:hover {
  background-color: #1d4ed8;
  transform: scale(1.05);
}
```

### Keyframe animations

For more complex animations that run automatically:

```css
@keyframes fadeIn {
  from {
    opacity: 0;
    transform: translateY(20px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

.card {
  animation: fadeIn 0.4s ease forwards;
}
```

| Property | What it controls |
| -------- | ---------------- |
| `animation-name` | Which `@keyframes` to use |
| `animation-duration` | How long one cycle takes |
| `animation-timing-function` | Easing (`ease`, `linear`, `ease-in-out`) |
| `animation-delay` | Wait before starting |
| `animation-iteration-count` | How many times to run (`infinite` for loops) |
| `animation-fill-mode` | State before/after animation (`forwards` keeps the end state) |
