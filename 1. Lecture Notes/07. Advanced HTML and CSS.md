## Semantic HTML (Expert Level)

Good HTML communicates meaning to browsers, search engines, and screen readers — not just visual appearance.

### ARIA (Accessible Rich Internet Applications)

ARIA attributes fill in gaps where HTML semantics alone aren't enough — primarily for dynamic widgets built without native HTML equivalents.

```html
<!-- Role: tells screen readers what kind of widget this is -->
<div role="dialog" aria-modal="true" aria-labelledby="dialog-title">
  <h2 id="dialog-title">Confirm Deletion</h2>
  <button aria-describedby="dialog-desc">Delete</button>
  <p id="dialog-desc">This action cannot be undone.</p>
</div>

<!-- Live regions: announces dynamic content changes -->
<div aria-live="polite" aria-atomic="true">
  Items in cart: <span id="count">3</span>
</div>

<!-- States -->
<button aria-expanded="false" aria-controls="menu">Menu</button>
<nav id="menu" aria-hidden="true">...</nav>

<!-- Labels for icon-only buttons -->
<button aria-label="Close dialog">✕</button>
```

**Rule of thumb**: use native HTML first. `<button>` is already accessible — a `<div role="button">` is not unless you also add `tabindex`, `keydown` handlers, and ARIA. Native elements are always better.

### Landmark roles (implicit in semantic HTML)

| Element | Implicit role |
| ------- | ------------- |
| `<header>` | `banner` |
| `<nav>` | `navigation` |
| `<main>` | `main` |
| `<aside>` | `complementary` |
| `<footer>` | `contentinfo` |
| `<section>` | `region` (only with `aria-label`) |
| `<form>` | `form` (only with `aria-label`) |

### Meta tags for SEO and sharing

```html
<head>
  <!-- Basic SEO -->
  <title>Page Title | Site Name</title>
  <meta name="description" content="150-character summary for search results.">
  <link rel="canonical" href="https://example.com/this-page">

  <!-- Open Graph (Facebook, LinkedIn, Slack previews) -->
  <meta property="og:title" content="Page Title">
  <meta property="og:description" content="Summary">
  <meta property="og:image" content="https://example.com/preview.jpg">
  <meta property="og:url" content="https://example.com/this-page">
  <meta property="og:type" content="website">

  <!-- Twitter Card -->
  <meta name="twitter:card" content="summary_large_image">
  <meta name="twitter:title" content="Page Title">
  <meta name="twitter:image" content="https://example.com/preview.jpg">
</head>
```

## CSS Architecture

### BEM (Block Element Modifier)

BEM is a naming convention that makes CSS predictable and avoids specificity conflicts.

```css
/* Block — independent component */
.card { }

/* Element — part of a block (double underscore) */
.card__title { }
.card__body { }
.card__footer { }

/* Modifier — variant or state (double hyphen) */
.card--featured { }
.card__title--large { }
.card--is-loading { }
```

```html
<article class="card card--featured">
  <h2 class="card__title card__title--large">Title</h2>
  <div class="card__body">Content</div>
</article>
```

### CSS Layers (`@layer`)

`@layer` lets you control specificity at a structural level — later layers win regardless of specificity within them.

```css
@layer base, components, utilities;

@layer base {
  * { box-sizing: border-box; }
  body { font-family: sans-serif; }
}

@layer components {
  .button { padding: 8px 16px; }
}

@layer utilities {
  .mt-4 { margin-top: 1rem; }
}
/* utilities always beat components, components always beat base */
```

## Advanced Selectors

```css
/* :is() — matches any of the listed selectors */
:is(h1, h2, h3) { color: navy; }
section :is(h1, h2, h3) { margin-top: 0; }

/* :where() — like :is() but zero specificity */
:where(ul, ol) { padding-left: 1.5rem; }

/* :has() — parent selector (if .card contains an img) */
.card:has(img) { padding: 0; }
a:has(> img) { display: block; }
li:has(+ li) { margin-bottom: 8px; }

/* :not() */
input:not([type="submit"]):not([type="reset"]) { border: 1px solid #ccc; }

/* Attribute selectors */
[href^="https"]  { } /* starts with */
[href$=".pdf"]   { } /* ends with */
[href*="docs"]   { } /* contains */
[lang|="en"]     { } /* en or en-US etc */
```

## CSS Container Queries

Media queries respond to the **viewport** size. Container queries respond to the **parent element's** size — critical for truly reusable components.

```css
/* Declare a containment context */
.card-wrapper {
  container-type: inline-size;
  container-name: card;
}

/* Style based on the container's width, not the viewport */
@container card (min-width: 400px) {
  .card {
    display: grid;
    grid-template-columns: 200px 1fr;
  }
}

@container card (min-width: 600px) {
  .card__title { font-size: 2rem; }
}
```

The same component adapts based on where it's placed — a narrow sidebar column vs a wide main column — without any media query changes.

## CSS Grid (Advanced)

### Subgrid

A child element can participate in its parent's grid tracks instead of establishing its own:

```css
.grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 24px;
}

.card {
  display: grid;
  grid-row: span 3;
  grid-template-rows: subgrid; /* aligns header, body, footer across all cards */
}
```

### Auto-fill vs auto-fit

```css
/* auto-fill: creates as many columns as fit; leaves empty tracks */
grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));

/* auto-fit: creates columns to fit; collapses empty tracks so items stretch */
grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));
```

## Custom Properties (Advanced)

### Computed custom properties

```css
:root {
  --hue: 220;
  --saturation: 90%;
  --lightness: 50%;
  --color-primary: hsl(var(--hue) var(--saturation) var(--lightness));
}

/* Dark mode: just shift lightness */
@media (prefers-color-scheme: dark) {
  :root { --lightness: 70%; }
}
```

### Custom properties as state

```css
.button {
  --is-pressed: 0;
  transform: scale(calc(1 - var(--is-pressed) * 0.05));
  filter: brightness(calc(1 - var(--is-pressed) * 0.1));
  transition: transform 0.1s, filter 0.1s;
}

.button:active { --is-pressed: 1; }
```

### CSS `@property` (Houdini)

Register a custom property with a type, which enables animation of values that normally can't be animated:

```css
@property --gradient-angle {
  syntax: '<angle>';
  initial-value: 0deg;
  inherits: false;
}

.animated-gradient {
  background: conic-gradient(from var(--gradient-angle), #3b82f6, #8b5cf6, #3b82f6);
  animation: spin 4s linear infinite;
}

@keyframes spin {
  to { --gradient-angle: 360deg; }
}
```

## Performance Optimisation

### Critical rendering path

The browser must fetch, parse, and apply CSS before it can paint. Minimise render-blocking:

```html
<!-- Load non-critical CSS asynchronously -->
<link rel="preload" href="non-critical.css" as="style" onload="this.onload=null;this.rel='stylesheet'">

<!-- Preconnect to external domains -->
<link rel="preconnect" href="https://fonts.googleapis.com">
```

### CSS containment

```css
/* Tell the browser this element is independent — enables rendering optimisations */
.widget {
  contain: layout paint; /* changes inside don't affect layout outside */
}

.card {
  content-visibility: auto; /* skip rendering off-screen cards */
  contain-intrinsic-size: 300px; /* estimated size for layout while skipped */
}
```

### Efficient selectors

- Avoid deep descendant selectors: `.nav .list .item a span` — too broad
- Prefer direct class selectors: `.nav-link-text`
- Avoid universal selectors inside scopes: `.card *`

### `will-change`

```css
.drawer {
  will-change: transform; /* promote to GPU layer before animation starts */
}

/* Remove after animation to free resources */
.drawer.is-open {
  will-change: auto;
}
```

## Advanced Animations

### View Transitions API

Animate between page states with near-zero JavaScript:

```css
::view-transition-old(root) {
  animation: slide-out 0.3s ease forwards;
}

::view-transition-new(root) {
  animation: slide-in 0.3s ease forwards;
}

@keyframes slide-out {
  to { transform: translateX(-100%); }
}

@keyframes slide-in {
  from { transform: translateX(100%); }
}
```

```js
document.startViewTransition(() => {
  updatePage(); // DOM changes happen here, with animated transition
});
```

### Scroll-driven animations

Animate elements based on scroll position — no JavaScript required:

```css
@keyframes reveal {
  from { opacity: 0; transform: translateY(40px); }
  to   { opacity: 1; transform: translateY(0); }
}

.section {
  animation: reveal linear both;
  animation-timeline: view();
  animation-range: entry 0% entry 40%;
}
```

## Print Stylesheets

```css
@media print {
  /* Remove navigation, ads, interactive elements */
  nav, .sidebar, .ads, button { display: none; }

  /* Expand links to show URLs */
  a[href]::after { content: ' (' attr(href) ')'; }

  /* Prevent page breaks inside cards */
  .card { break-inside: avoid; }

  /* Force page break before major sections */
  h1 { break-before: page; }

  /* Black and white, high contrast */
  body { color: black; background: white; }
}
```
