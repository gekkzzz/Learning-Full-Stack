## The DOM

The **Document Object Model (DOM)** is the browser's live, programmable representation of your HTML. When the browser loads a page, it reads the HTML and builds a tree of objects — one for every element. JavaScript can read and modify this tree, which is how pages become interactive.

```js
// Selecting elements
const title = document.querySelector('h1');          // first match
const items = document.querySelectorAll('.item');    // NodeList of all matches
const btn   = document.getElementById('submit-btn');

// Reading and writing content
title.textContent = 'Hello, World!';    // safe: treats value as plain text
title.innerHTML   = '<em>Hello</em>';  // careful: parses HTML (XSS risk if user input)

// Changing styles and classes
title.style.color = 'red';
title.classList.add('active');
title.classList.remove('hidden');
title.classList.toggle('open');
title.classList.contains('active'); // true

// Reading attributes
const href = link.getAttribute('href');
link.setAttribute('href', '/about');
link.removeAttribute('disabled');
```

## Events

Events are things that happen in the browser — a click, a keypress, the page finishing loading. You respond to them by attaching **event listeners**.

```js
const button = document.querySelector('#btn');

button.addEventListener('click', function(event) {
  console.log('clicked!', event);
});

// Arrow function form (more common):
button.addEventListener('click', (event) => {
  event.preventDefault();    // stop default behaviour (e.g. form submitting)
  event.stopPropagation();   // stop the event bubbling up to parent elements
  console.log(event.target); // the element that was actually clicked
});
```

### Common event types

| Event | Fires when… |
| ----- | ----------- |
| `click` | Element is clicked |
| `dblclick` | Element is double-clicked |
| `keydown` | A key is pressed |
| `keyup` | A key is released |
| `input` | Input value changes |
| `change` | Input loses focus after changing |
| `submit` | A form is submitted |
| `mouseover` / `mouseout` | Mouse enters / leaves element |
| `focus` / `blur` | Input gains / loses focus |
| `DOMContentLoaded` | HTML is parsed (before images load) |
| `load` | Everything including images has loaded |

### Event delegation

Instead of attaching listeners to every child element, attach one listener to the parent and check `event.target`. This is more efficient and works for dynamically added elements.

```js
document.querySelector('#list').addEventListener('click', (e) => {
  if (e.target.matches('li')) {
    console.log('Clicked item:', e.target.textContent);
  }
});
```

## Functions In Depth

### Arrow functions vs regular functions

```js
// Regular function — has its own `this`
function greet(name) {
  return `Hello, ${name}!`;
}

// Arrow function — inherits `this` from surrounding scope
const greet = (name) => `Hello, ${name}!`;

// Single parameter: parentheses optional
const double = n => n * 2;

// No parameters: parentheses required
const sayHi = () => 'Hi!';
```

### Default parameters

```js
function createUser(name, role = 'viewer', active = true) {
  return { name, role, active };
}
```

### Rest and spread

```js
// Rest: collect remaining arguments into an array
function sum(...numbers) {
  return numbers.reduce((total, n) => total + n, 0);
}

// Spread: expand an array or object
const nums = [1, 2, 3];
console.log(Math.max(...nums)); // 3

const user = { name: 'Alice', role: 'admin' };
const updated = { ...user, role: 'viewer' }; // { name: 'Alice', role: 'viewer' }
```

### Closures

A closure is a function that remembers the variables from the scope where it was created, even after that outer scope has finished executing.

```js
function makeCounter() {
  let count = 0;           // lives inside makeCounter's scope
  return function() {
    count++;               // the inner function "closes over" count
    return count;
  };
}

const counter = makeCounter();
counter(); // 1
counter(); // 2
counter(); // 3 — count is private and persists
```

## Arrays In Depth

The most important array methods use callbacks — functions you pass in that run for each element.

```js
const users = [
  { name: 'Alice', age: 28, active: true },
  { name: 'Bob',   age: 34, active: false },
  { name: 'Carol', age: 22, active: true },
];

// map — transform each element, return new array (same length)
const names = users.map(user => user.name); // ['Alice', 'Bob', 'Carol']

// filter — keep elements that pass a test, return new array
const active = users.filter(user => user.active); // Alice, Carol

// find — first element that passes a test (or undefined)
const bob = users.find(user => user.name === 'Bob');

// some — true if at least one element passes
users.some(user => user.age > 30); // true

// every — true if all elements pass
users.every(user => user.active); // false

// reduce — collapse array to a single value
const totalAge = users.reduce((sum, user) => sum + user.age, 0); // 84

// sort — sorts IN PLACE (mutates the original array)
const sorted = [...users].sort((a, b) => a.age - b.age); // youngest first

// forEach — like map but returns nothing; use for side effects only
users.forEach(user => console.log(user.name));
```

## Objects In Depth

### Destructuring

```js
const { name, age, role = 'viewer' } = user; // role defaults to 'viewer' if missing

// Rename while destructuring
const { name: userName } = user;

// Nested
const { address: { city } } = user;

// Array destructuring
const [first, second, ...rest] = [1, 2, 3, 4, 5];

// In function parameters
function render({ title, body, author = 'Anonymous' }) {
  return `${title} by ${author}: ${body}`;
}
```

### Optional chaining and nullish coalescing

```js
// Optional chaining (?.) — safely access nested properties
const city = user?.address?.city; // undefined instead of throwing
const first = arr?.[0];           // safe array access
const result = fn?.();            // call only if fn is a function

// Nullish coalescing (??) — fallback only for null/undefined (not 0 or '')
const name = user.name ?? 'Anonymous';
```

## Asynchronous JavaScript

JavaScript is single-threaded — it can only do one thing at a time. Asynchronous code lets you start an operation (like a network request) and continue running other code while you wait for it to finish.

### Callbacks (old way)

```js
setTimeout(() => {
  console.log('Done after 1 second');
}, 1000);
```

Nesting multiple callbacks leads to "callback hell" — hard-to-read deeply nested code.

### Promises

A Promise represents a future value. It can be **pending**, **fulfilled** (resolved), or **rejected**.

```js
fetch('https://api.example.com/users')
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error('Failed:', error))
  .finally(() => console.log('Request finished'));
```

### Async/await

`async/await` is syntactic sugar over Promises. It makes async code read like synchronous code.

```js
async function getUsers() {
  try {
    const response = await fetch('https://api.example.com/users');
    const data = await response.json();
    console.log(data);
    return data;
  } catch (error) {
    console.error('Failed:', error);
  }
}
```

An `async` function always returns a Promise. `await` pauses execution *inside* the function until the Promise resolves.

### Running tasks in parallel

```js
// Sequential: takes 3 seconds if each takes 1 second
const a = await fetchA();
const b = await fetchB();
const c = await fetchC();

// Parallel: takes 1 second — all start at the same time
const [a, b, c] = await Promise.all([fetchA(), fetchB(), fetchC()]);
```

## Error Handling

```js
try {
  const data = JSON.parse(badJson);
} catch (error) {
  console.error(error.message); // access the error details
} finally {
  cleanup();                    // runs whether or not an error occurred
}

// Throwing your own errors
function divide(a, b) {
  if (b === 0) throw new Error('Cannot divide by zero');
  return a / b;
}
```

## Modules (ES Modules)

Modules let you split code across multiple files and explicitly control what is shared.

```js
// math.js — named exports
export function add(a, b) { return a + b; }
export function multiply(a, b) { return a * b; }
export const PI = 3.14159;

// main.js — named imports
import { add, multiply } from './math.js';
import { add as sum } from './math.js'; // rename on import
import * as Math from './math.js';      // import everything as a namespace
```

```js
// utils.js — default export (one per file)
export default function formatDate(date) { ... }

// main.js — import the default (name it whatever you want)
import formatDate from './utils.js';
```

## localStorage

`localStorage` lets you persist data in the browser between page refreshes. Data survives until explicitly cleared.

```js
// Save (values must be strings)
localStorage.setItem('theme', 'dark');
localStorage.setItem('user', JSON.stringify({ name: 'Alice' }));

// Read
const theme = localStorage.getItem('theme');
const user = JSON.parse(localStorage.getItem('user'));

// Delete
localStorage.removeItem('theme');
localStorage.clear(); // remove everything
```

## Working with APIs (Fetch)

Most modern web apps communicate with servers using JSON over HTTP.

```js
// GET
async function getPost(id) {
  const res = await fetch(`/api/posts/${id}`);
  if (!res.ok) throw new Error(`HTTP error: ${res.status}`);
  return res.json();
}

// POST
async function createPost(data) {
  const res = await fetch('/api/posts', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(data),
  });
  return res.json();
}

// PUT / DELETE follow the same pattern — just change `method`
```

HTTP status codes to know:

| Code | Meaning |
| ---- | ------- |
| 200 | OK |
| 201 | Created |
| 400 | Bad Request (your fault) |
| 401 | Unauthorised (not logged in) |
| 403 | Forbidden (logged in but not allowed) |
| 404 | Not Found |
| 500 | Internal Server Error (server's fault) |
