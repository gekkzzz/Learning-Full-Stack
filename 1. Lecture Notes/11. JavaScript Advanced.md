## Prototypes and the Prototype Chain

Every JavaScript object has a hidden internal link to another object called its **prototype**. When you access a property on an object, JavaScript first looks on the object itself. If it doesn't find it, it looks at the prototype, then the prototype's prototype, and so on. This chain ends at `Object.prototype`.

```js
const animal = {
  breathe() { return 'breathing'; }
};

const dog = Object.create(animal); // dog's prototype is animal
dog.bark = function() { return 'woof'; };

dog.bark();    // found on dog directly
dog.breathe(); // not on dog — found on animal via prototype chain
```

`class` syntax in JavaScript is **syntactic sugar** over prototypes — it does not introduce a new object model.

```js
class Animal {
  constructor(name) { this.name = name; }
  speak() { return `${this.name} makes a noise.`; }
}

class Dog extends Animal {
  speak() { return `${this.name} barks.`; }
}

// Under the hood, Dog.prototype.__proto__ === Animal.prototype
```

## `this` Keyword In Depth

`this` refers to the object the function was called *on*, not where the function was defined. Its value depends entirely on the **call site**.

| How called | `this` value |
| ---------- | ------------ |
| Regular function call | `undefined` (strict mode) or global object |
| Method call `obj.fn()` | `obj` |
| `new Fn()` | Newly created instance |
| Arrow function | Lexical — inherits `this` from enclosing scope |
| `fn.call(ctx)` / `fn.apply(ctx)` | Explicitly `ctx` |
| `fn.bind(ctx)` | Returns new function with `this` locked to `ctx` |

```js
class Timer {
  constructor() {
    this.seconds = 0;
  }

  start() {
    // Arrow function — `this` is the Timer instance, not the callback's context
    setInterval(() => {
      this.seconds++;
    }, 1000);
  }
}
```

## Generators and Iterators

A **generator** is a function that can pause and resume, yielding values one at a time.

```js
function* range(start, end) {
  for (let i = start; i <= end; i++) {
    yield i; // pauses here and returns i
  }
}

const nums = range(1, 5);
nums.next(); // { value: 1, done: false }
nums.next(); // { value: 2, done: false }

for (const n of range(1, 5)) {
  console.log(n); // 1, 2, 3, 4, 5
}
```

Generators are useful for lazy sequences, infinite data streams, and implementing custom iterables.

## Symbols and Well-Known Symbols

`Symbol` creates a unique, non-string value — useful for object keys that won't accidentally collide with other properties.

```js
const id = Symbol('id');
const user = { [id]: 42, name: 'Alice' };
user[id]; // 42 — Symbol key is not enumerable in for...in loops
```

Well-known symbols let you hook into core language behaviour:

```js
class InfiniteCounter {
  [Symbol.iterator]() {
    let n = 0;
    return { next: () => ({ value: n++, done: false }) };
  }
}

for (const n of new InfiniteCounter()) {
  if (n > 3) break;
  console.log(n); // 0, 1, 2, 3
}
```

## WeakMap and WeakSet

`WeakMap` and `WeakSet` hold *weak references* — they don't prevent garbage collection of their keys/values.

```js
const cache = new WeakMap();

function process(obj) {
  if (cache.has(obj)) return cache.get(obj);
  const result = heavyComputation(obj);
  cache.set(obj, result);
  return result;
}
// When `obj` is garbage collected, its cache entry disappears automatically
```

Use WeakMap when you need to associate private data with an object without preventing its garbage collection.

## Proxy and Reflect

`Proxy` wraps an object and intercepts fundamental operations — property access, assignment, function calls, etc.

```js
const handler = {
  get(target, key) {
    console.log(`Reading ${key}`);
    return Reflect.get(target, key);
  },
  set(target, key, value) {
    if (typeof value !== 'number') throw new TypeError('Must be a number');
    return Reflect.set(target, key, value);
  }
};

const scores = new Proxy({}, handler);
scores.alice = 95;   // sets fine
scores.bob = 'high'; // throws TypeError
```

Proxies are used by Vue 3's reactivity system and many validation/observability libraries.

## Advanced Async Patterns

### Async generators

```js
async function* streamPages(url) {
  let page = 1;
  while (true) {
    const res = await fetch(`${url}?page=${page}`);
    const data = await res.json();
    if (!data.length) return;
    yield data;
    page++;
  }
}

for await (const page of streamPages('/api/posts')) {
  renderPage(page);
}
```

### AbortController

Cancel in-flight fetch requests:

```js
const controller = new AbortController();

const timeout = setTimeout(() => controller.abort(), 5000);

try {
  const res = await fetch('/api/data', { signal: controller.signal });
  const data = await res.json();
  clearTimeout(timeout);
} catch (e) {
  if (e.name === 'AbortError') console.log('Request timed out');
}
```

### Promise combinators

```js
// Reject as soon as any promise rejects
await Promise.all([p1, p2, p3]);

// Wait for all to settle regardless of success/failure
await Promise.allSettled([p1, p2, p3]);
// Returns: [{ status: 'fulfilled', value }, { status: 'rejected', reason }]

// Resolve/reject with the first to settle
await Promise.race([p1, p2, p3]);

// Resolve with the first to fulfill (ignores rejections)
await Promise.any([p1, p2, p3]);
```

## Memory Management and Performance

### Avoiding memory leaks

```js
// Leak: event listener keeps reference to element even after it's removed from DOM
element.addEventListener('click', handler);
// Fix: remove listener when done
element.removeEventListener('click', handler);

// Leak: closures holding large data
function leaky() {
  const bigArray = new Array(1_000_000).fill(0);
  return function() { return bigArray[0]; }; // holds entire array alive
}
```

### Debounce and throttle

```js
// Debounce: fire only after N ms of silence
function debounce(fn, delay) {
  let timer;
  return (...args) => {
    clearTimeout(timer);
    timer = setTimeout(() => fn(...args), delay);
  };
}

// Throttle: fire at most once every N ms
function throttle(fn, interval) {
  let last = 0;
  return (...args) => {
    const now = Date.now();
    if (now - last >= interval) {
      last = now;
      fn(...args);
    }
  };
}

const onInput = debounce(search, 300);   // search runs 300ms after user stops typing
const onScroll = throttle(update, 100);  // update runs at most 10 times per second
```

## Design Patterns

### Module pattern (encapsulation)

```js
const counter = (() => {
  let count = 0; // private
  return {
    increment() { count++; },
    decrement() { count--; },
    value() { return count; },
  };
})();
```

### Observer pattern (pub/sub)

```js
class EventEmitter {
  #listeners = new Map();

  on(event, listener) {
    if (!this.#listeners.has(event)) this.#listeners.set(event, []);
    this.#listeners.get(event).push(listener);
  }

  emit(event, ...args) {
    this.#listeners.get(event)?.forEach(fn => fn(...args));
  }

  off(event, listener) {
    const list = this.#listeners.get(event);
    if (list) this.#listeners.set(event, list.filter(fn => fn !== listener));
  }
}
```

### Factory pattern

```js
function createUser(name, role) {
  const permissions = role === 'admin' ? ['read', 'write', 'delete'] : ['read'];
  return Object.freeze({ name, role, permissions }); // immutable
}
```

## Private Class Fields

The `#` prefix marks a field as truly private — inaccessible outside the class, not even through `obj.field`.

```js
class BankAccount {
  #balance = 0;

  deposit(amount) {
    if (amount <= 0) throw new Error('Amount must be positive');
    this.#balance += amount;
  }

  get balance() { return this.#balance; }
}

const account = new BankAccount();
account.deposit(100);
account.balance;     // 100
account.#balance;    // SyntaxError — cannot access private field
```

## Functional Programming Concepts

### Pure functions

A pure function always returns the same output for the same input and has no side effects.

```js
// Pure — no mutation, no side effects
const add = (a, b) => a + b;
const double = arr => arr.map(x => x * 2); // returns new array

// Impure — mutates external state
let total = 0;
function addToTotal(n) { total += n; } // side effect
```

### Immutability

Never mutate data directly — create new versions instead.

```js
// Mutating (avoid)
state.user.name = 'Bob';
state.items.push(newItem);

// Immutable updates
const newState = {
  ...state,
  user: { ...state.user, name: 'Bob' },
  items: [...state.items, newItem],
};
```

### Function composition

```js
const compose = (...fns) => x => fns.reduceRight((acc, fn) => fn(acc), x);
const pipe    = (...fns) => x => fns.reduce((acc, fn) => fn(acc), x);

const transform = pipe(
  trim,
  toLowerCase,
  removeSpecialChars,
);

transform('  Hello, World!  '); // 'hello world'
```
