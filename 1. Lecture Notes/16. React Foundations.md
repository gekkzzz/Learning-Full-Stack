# React Foundations

---

## What Is React?

React is a JavaScript library for building user interfaces. It was created by Facebook and is now maintained by Meta and a large open-source community. React's core idea is simple: describe **what** the UI should look like for a given state, and React handles updating the DOM to match.

Traditional DOM manipulation is imperative — you write step-by-step instructions: "find this element, change its text, add this class." React is **declarative** — you describe the end result, and the library figures out what needs to change.

### Why React became dominant

- **Components** — reusable, composable building blocks instead of monolithic templates
- **Virtual DOM** — React diffs a lightweight copy of the DOM and only updates what changed
- **Unidirectional data flow** — data flows downward, making apps predictable
- **Huge ecosystem** — libraries for routing, state, forms, animation, and everything else

## JSX

JSX is a syntax extension that lets you write HTML-like markup inside JavaScript. Babel or a bundler compiles it to regular JavaScript calls.

```jsx
// JSX
const element = <h1 className="title">Hello, {name}!</h1>;

// Compiled to (you never write this directly)
const element = React.createElement('h1', { className: 'title' }, `Hello, ${name}!`);
```

### JSX rules

```jsx
// 1. Must have a single root element — wrap siblings in a fragment
return (
  <>
    <h1>Title</h1>
    <p>Body</p>
  </>
);

// 2. All tags must be closed — even self-closing ones
<img src="photo.jpg" alt="A photo" />
<input type="text" />
<br />

// 3. Use className instead of class
<div className="container">

// 4. Use camelCase for attributes
<button onClick={handleClick} tabIndex={0}>

// 5. JavaScript expressions go inside curly braces
<p>{user.name}</p>
<p>{2 + 2}</p>
<p>{isLoggedIn ? 'Welcome!' : 'Please log in'}</p>
```

## Components

A component is a JavaScript function that returns JSX. Components are the building blocks of every React app.

```jsx
// Function component — the modern way
function Greeting({ name, role = 'visitor' }) {
  return (
    <div className="greeting">
      <h1>Hello, {name}!</h1>
      <p>You are logged in as: {role}</p>
    </div>
  );
}

// Using the component
<Greeting name="Alice" role="admin" />
```

### Component rules

- Component names must start with a capital letter (`Greeting`, not `greeting`)
- Must return JSX or `null`
- Never mutate props — they are read-only

## Props

Props (properties) are how parent components pass data to child components. They are passed like HTML attributes and received as a single object parameter.

```jsx
function UserCard({ name, email, avatar, onFollow }) {
  return (
    <div className="card">
      <img src={avatar} alt={name} />
      <h2>{name}</h2>
      <p>{email}</p>
      <button onClick={onFollow}>Follow</button>
    </div>
  );
}

// Passing props
<UserCard
  name="Alice"
  email="alice@example.com"
  avatar="/alice.jpg"
  onFollow={() => console.log('following')}
/>
```

### `children` prop

Any JSX placed between a component's opening and closing tags becomes `props.children`:

```jsx
function Card({ title, children }) {
  return (
    <div className="card">
      <h2>{title}</h2>
      <div className="card-body">{children}</div>
    </div>
  );
}

<Card title="Welcome">
  <p>This is the card content.</p>
  <button>Click me</button>
</Card>
```

## State with `useState`

State is data that belongs to a component and can change over time. When state changes, React re-renders the component with the new value.

```jsx
import { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0); // [current value, setter function]

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>+</button>
      <button onClick={() => setCount(count - 1)}>-</button>
      <button onClick={() => setCount(0)}>Reset</button>
    </div>
  );
}
```

### State is a snapshot

When React re-renders, each render sees its own snapshot of state. Don't rely on reading state immediately after setting it.

```jsx
// Wrong — count is stale inside the handler
const handleTripleIncrement = () => {
  setCount(count + 1);
  setCount(count + 1); // still uses old count
  setCount(count + 1); // still uses old count — net result: +1
};

// Correct — use the updater function form
const handleTripleIncrement = () => {
  setCount(c => c + 1);
  setCount(c => c + 1); // each gets the latest value
  setCount(c => c + 1); // net result: +3
};
```

### State with objects

```jsx
const [user, setUser] = useState({ name: 'Alice', age: 28 });

// Always create a new object — never mutate state directly
setUser({ ...user, age: 29 }); // spread existing, override changed field

// Never do this:
user.age = 29;         // mutating state — React won't detect the change
setUser(user);         // same object reference — no re-render
```

## Rendering Lists

```jsx
function UserList({ users }) {
  return (
    <ul>
      {users.map(user => (
        <li key={user.id}>         {/* key is required — must be unique and stable */}
          {user.name}
        </li>
      ))}
    </ul>
  );
}
```

The `key` prop helps React identify which items changed, were added, or removed. Use a stable, unique ID — never use the array index as a key for lists that can reorder or change.

## Conditional Rendering

```jsx
function Alert({ message, type }) {
  // Short-circuit: render nothing if no message
  if (!message) return null;

  return (
    <div className={`alert alert-${type}`}>
      {/* Ternary for two outcomes */}
      {type === 'error' ? <ErrorIcon /> : <InfoIcon />}

      {/* Short-circuit for optional content */}
      {type === 'error' && <strong>Error: </strong>}

      {message}
    </div>
  );
}
```

## Event Handling

```jsx
function Form() {
  function handleSubmit(event) {
    event.preventDefault(); // stop the form from reloading the page
    console.log('submitted');
  }

  function handleChange(event) {
    console.log(event.target.value);
  }

  return (
    <form onSubmit={handleSubmit}>
      <input onChange={handleChange} />
      <button type="submit">Submit</button>
    </form>
  );
}
```

React events are **SyntheticEvents** — they wrap native browser events for cross-browser consistency. They behave identically to native events.

## `useEffect`

`useEffect` lets you run code after a render — for fetching data, subscribing to events, or updating the DOM.

```jsx
import { useEffect, useState } from 'react';

function UserProfile({ userId }) {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    setLoading(true);

    fetch(`/api/users/${userId}`)
      .then(res => res.json())
      .then(data => {
        setUser(data);
        setLoading(false);
      });
  }, [userId]); // dependency array — re-run whenever userId changes

  if (loading) return <p>Loading…</p>;
  return <h1>{user.name}</h1>;
}
```

### Dependency array rules

| Dependencies | When the effect runs |
| ------------ | -------------------- |
| `[a, b]` | After every render where `a` or `b` changed |
| `[]` (empty) | Once, after the first render only |
| Omitted | After every render |

### Cleanup

```jsx
useEffect(() => {
  const timer = setInterval(() => setTick(t => t + 1), 1000);

  // Return a cleanup function — runs before the next effect and on unmount
  return () => clearInterval(timer);
}, []);
```

## Lifting State Up

When two sibling components need to share state, move the state up to their closest common parent and pass it down via props.

```jsx
function App() {
  const [query, setQuery] = useState('');

  return (
    <>
      <SearchInput value={query} onChange={setQuery} />
      <SearchResults query={query} />
    </>
  );
}

function SearchInput({ value, onChange }) {
  return <input value={value} onChange={e => onChange(e.target.value)} />;
}

function SearchResults({ query }) {
  // uses query to filter results
}
```

This is the fundamental pattern behind React's one-way data flow.
