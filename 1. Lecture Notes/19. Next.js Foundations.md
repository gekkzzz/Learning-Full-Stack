# Next.js Foundations

---

## What Is Next.js?

Next.js is a **framework built on top of React**. Plain React is a library for building UI — it gives you components and state, but leaves everything else (routing, data fetching, deployment) up to you. Next.js makes all those decisions for you, adding:

- **File-based routing** — your folder structure IS your URL structure
- **Server-side rendering (SSR)** and **static generation (SSG)**
- **API routes** — build a backend inside the same project
- **Image optimisation, fonts, and performance built-in**

Think of React as the engine and Next.js as the car — same engine, but the car has seats, doors, and a dashboard already assembled.

---

## App Router vs Pages Router

Next.js has two routing systems. **App Router** (introduced in Next.js 13) is the modern approach and is what all new projects should use. The older **Pages Router** still works and is found in older codebases.

This guide covers the **App Router**.

---

## Project Structure

```
my-app/
├── app/                  ← All pages and layouts live here
│   ├── layout.tsx        ← Root layout — wraps every page
│   ├── page.tsx          ← The homepage (route: /)
│   ├── about/
│   │   └── page.tsx      ← About page (route: /about)
│   └── blog/
│       ├── page.tsx      ← Blog index (route: /blog)
│       └── [slug]/
│           └── page.tsx  ← Blog post (route: /blog/any-slug)
├── components/           ← Shared React components
├── public/               ← Static files (images, fonts) served at /
├── next.config.ts        ← Next.js configuration
└── package.json
```

## Pages and Routing

Every `page.tsx` file in the `app` folder becomes a URL route. The file path = the URL.

```tsx
// app/page.tsx → accessible at /
export default function HomePage() {
  return <h1>Welcome to my site!</h1>;
}

// app/about/page.tsx → accessible at /about
export default function AboutPage() {
  return <h1>About Us</h1>;
}
```

### Dynamic routes

Square brackets create a dynamic segment that can match anything:

```tsx
// app/products/[id]/page.tsx → matches /products/1, /products/abc, etc.

// The `params` prop contains the matched segments
export default async function ProductPage({ params }: { params: { id: string } }) {
  const product = await getProduct(params.id);
  return <h1>{product.name}</h1>;
}
```

### Catch-all routes

```tsx
// app/docs/[...slug]/page.tsx → matches /docs/getting-started/installation
// params.slug = ['getting-started', 'installation']
```

## Layouts

A `layout.tsx` file wraps all pages inside its folder. The root layout at `app/layout.tsx` is required and wraps every page in your app.

```tsx
// app/layout.tsx
export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en">
      <body>
        <Header />
        <main>{children}</main>  {/* page content renders here */}
        <Footer />
      </body>
    </html>
  );
}

// app/dashboard/layout.tsx — wraps all pages under /dashboard
export default function DashboardLayout({ children }: { children: React.ReactNode }) {
  return (
    <div className="dashboard">
      <Sidebar />
      <div className="content">{children}</div>
    </div>
  );
}
```

Layouts are powerful because they **persist across navigation** — the sidebar doesn't re-mount when you navigate between dashboard pages.

## Server Components (Default in App Router)

Every component in the App Router is a **Server Component** by default. Server Components:

- Run on the server — they never become part of your JavaScript bundle
- Can directly query databases, read files, call APIs (no need for `useEffect` + `fetch`)
- Cannot use state (`useState`), effects (`useEffect`), or browser APIs

```tsx
// app/users/page.tsx — Server Component
// This runs on the server. The database call never reaches the browser.
import { db } from '@/lib/database';

export default async function UsersPage() {
  const users = await db.query('SELECT * FROM users');

  return (
    <ul>
      {users.map(user => (
        <li key={user.id}>{user.name}</li>
      ))}
    </ul>
  );
}
```

For interactivity (click handlers, state), add the `'use client'` directive at the top of the file:

```tsx
// components/LikeButton.tsx
'use client';

import { useState } from 'react';

export default function LikeButton({ initialCount }: { initialCount: number }) {
  const [count, setCount] = useState(initialCount);
  return <button onClick={() => setCount(c => c + 1)}>❤️ {count}</button>;
}
```

The pattern: fetch data in Server Components, handle interaction in Client Components.

## Metadata and SEO

Next.js has a built-in `metadata` API for setting `<title>`, `<meta>` tags, and Open Graph:

```tsx
// app/page.tsx
import type { Metadata } from 'next';

export const metadata: Metadata = {
  title: 'My App',
  description: 'The best app on the internet.',
  openGraph: {
    title: 'My App',
    description: 'The best app on the internet.',
    images: [{ url: '/og-image.jpg' }],
  },
};

// Dynamic metadata (based on route params)
export async function generateMetadata({ params }: { params: { id: string } }): Promise<Metadata> {
  const product = await getProduct(params.id);
  return {
    title: product.name,
    description: product.description,
  };
}
```

## Navigation

```tsx
import Link from 'next/link';
import { useRouter } from 'next/navigation';

// Declarative navigation — always prefer Link over <a> for internal links
<Link href="/about">About</Link>
<Link href={`/products/${id}`}>View Product</Link>

// Programmatic navigation
'use client';
function Form() {
  const router = useRouter();
  async function handleSubmit() {
    await submitForm();
    router.push('/success');   // navigate to new page
    router.refresh();          // re-fetch Server Component data on current page
    router.back();             // go back
  }
}
```

`<Link>` prefetches linked pages automatically in production, making navigation feel instant.

## API Routes

You can build a backend API inside your Next.js project using **Route Handlers**:

```tsx
// app/api/users/route.ts

import { NextResponse } from 'next/server';

// GET /api/users
export async function GET() {
  const users = await db.user.findMany();
  return NextResponse.json(users);
}

// POST /api/users
export async function POST(request: Request) {
  const body = await request.json();
  const user = await db.user.create({ data: body });
  return NextResponse.json(user, { status: 201 });
}

// app/api/users/[id]/route.ts
export async function GET(request: Request, { params }: { params: { id: string } }) {
  const user = await db.user.findUnique({ where: { id: params.id } });
  if (!user) return NextResponse.json({ error: 'Not found' }, { status: 404 });
  return NextResponse.json(user);
}
```

## Images

The built-in `<Image>` component automatically:
- Serves the right size for each device
- Converts to modern formats (WebP, AVIF)
- Prevents layout shift with reserved space
- Lazy loads images below the fold

```tsx
import Image from 'next/image';

// Local image — TypeScript knows the dimensions automatically
import profilePic from './profile.jpg';
<Image src={profilePic} alt="Profile" />

// Remote image — must specify width and height
<Image
  src="https://example.com/photo.jpg"
  alt="Photo"
  width={800}
  height={600}
  priority // load immediately (use for above-the-fold images)
/>
```

## Environment Variables

```bash
# .env.local (never commit this file — add to .gitignore)
DATABASE_URL="postgresql://..."
STRIPE_SECRET_KEY="sk_test_..."

# NEXT_PUBLIC_ prefix makes the variable available in the browser
NEXT_PUBLIC_API_URL="https://api.example.com"
```

```ts
// Server-only (secure — not sent to browser)
const db = new Database(process.env.DATABASE_URL);

// Client-accessible (visible in browser — never put secrets here)
const apiUrl = process.env.NEXT_PUBLIC_API_URL;
```

## Loading and Error States

Next.js handles loading and error states with special files:

```tsx
// app/dashboard/loading.tsx — shown while dashboard/page.tsx is loading
export default function Loading() {
  return <DashboardSkeleton />;
}

// app/dashboard/error.tsx — shown if dashboard/page.tsx throws
'use client'; // error boundaries must be client components
export default function Error({ error, reset }: { error: Error; reset: () => void }) {
  return (
    <div>
      <h2>Something went wrong!</h2>
      <p>{error.message}</p>
      <button onClick={reset}>Try again</button>
    </div>
  );
}

// app/not-found.tsx — shown for 404s
export default function NotFound() {
  return <h2>404 — Page Not Found</h2>;
}
```
