# CSS Foundations

---

## Introduction to CSS

CSS stands for **Cascading Style Sheets**. It is the language that controls how HTML looks — colours, fonts, spacing, layout, and more. Without CSS, every webpage would just be plain black text on a white background.

HTML and CSS are always used together: HTML creates the structure (the content and what things *are*), and CSS styles the presentation (what things *look like*).

```html
<!-- HTML creates the element -->
<p class="intro">Welcome to my site!</p>
```

```css
/* CSS styles it */
.intro {
  color: steelblue;
  font-size: 1.25rem;
  font-weight: bold;
}
```

---

## How to Write a CSS Rule

Every CSS rule follows the same structure:

```css
selector {
  property: value;
  property: value;
}
```

- **Selector** — identifies which HTML element(s) to style.
- **Property** — the thing you want to change (e.g., `color`, `font-size`, `margin`).
- **Value** — what you want to set it to (e.g., `red`, `16px`, `10px`).
- Each property-value pair is called a **declaration** and must end with a semicolon.

```css
/* This rule targets all <p> elements and makes their text dark grey */
p {
  color: #333333;
  font-size: 16px;
  line-height: 1.6;
}
```

---

## Three Ways to Add CSS

| Method | How | When to use |
| ------ | --- | ----------- |
| **External** | `<link rel="stylesheet" href="style.css">` in `<head>` | Always — this is the standard approach |
| **Internal** | `<style>` block inside `<head>` | Quick demos or single-page experiments |
| **Inline** | `style=""` attribute directly on an element | Avoid — hard to maintain, hard to override |

**External CSS** is the correct approach for real projects. It keeps your HTML and CSS in separate files, meaning you can change the look of the entire site by editing one file.

```html
<!-- In your HTML file -->
<head>
  <link rel="stylesheet" href="style.css">
</head>
```

```css
/* In your style.css file */
body {
  font-family: sans-serif;
  background-color: #f9f9f9;
}
```

## 1. Selectors

A selector is the part of a CSS rule that says *which elements to target*. Getting selectors right is one of the most important skills in CSS.

### Basic Selectors

| Selector | Syntax | What it targets |
| -------- | ------ | --------------- |
| **Universal** | `*` | Every element on the page |
| **Type / Element** | `p`, `h1`, `div` | All elements of that HTML tag |
| **Class** | `.classname` | All elements with that class attribute |
| **ID** | `#idname` | The one element with that exact ID |

```css
/* Universal — applies to everything. Often used to reset box-sizing. */
* {
  box-sizing: border-box;
}

/* Type — targets every single <h2> on the page */
h2 {
  color: navy;
}

/* Class — targets any element with class="card" */
.card {
  border: 1px solid #ccc;
  padding: 16px;
  border-radius: 8px;
}

/* ID — targets the one element with id="hero" */
#hero {
  background-color: steelblue;
  color: white;
}
```

**Class vs ID:**

- A **class** (`.card`) can be used on as many elements as you like on one page. It's the most common way to apply styles.
- An **ID** (`#hero`) should only exist once per page. Use IDs for unique landmark sections, not repeated styling.

---

### Grouping Selectors

Apply the same rule to multiple different selectors by separating them with commas. This avoids repeating yourself.

```css
/* Without grouping — repetitive */
h1 { font-family: sans-serif; }
h2 { font-family: sans-serif; }
h3 { font-family: sans-serif; }

/* With grouping — clean and concise */
h1,
h2,
h3 {
  font-family: sans-serif;
}
```

---

### Chaining Selectors

Combine selectors with **no space** between them to target elements that must match *all* conditions at once.

```css
/* Targets <p> elements that also have the class "highlight" */
/* An <h2 class="highlight"> would NOT be targeted */
p.highlight {
  background-color: yellow;
}

/* Targets elements that have BOTH the "btn" class AND the "primary" class */
.btn.primary {
  background-color: royalblue;
  color: white;
}
```

---

### Combinator Selectors

Combinators let you target elements based on their relationship to other elements in the HTML structure.

| Combinator | Syntax | What it targets |
| ---------- | ------ | --------------- |
| **Descendant** | `A B` | Any `B` inside `A`, at any depth |
| **Child** | `A > B` | Only `B` that is a *direct* child of `A` |
| **Adjacent sibling** | `A + B` | The `B` immediately after `A` at the same level |
| **General sibling** | `A ~ B` | All `B` elements that come after `A` at the same level |

```css
/* Descendant — any <a> anywhere inside a <nav>, no matter how deep */
nav a {
  text-decoration: none;
  color: inherit;
}

/* Child — only <li> elements directly inside a <ul>, not nested further */
ul > li {
  list-style: disc;
}

/* Adjacent sibling — the <p> immediately after an <h2> */
h2 + p {
  margin-top: 0;
}
```

## 2. The Cascade, Specificity & Inheritance

### The Cascade — How CSS Resolves Conflicts

Sometimes multiple CSS rules target the same element and the same property. For example:

```css
p { color: black; }
.intro { color: steelblue; }
```

If a `<p>` element also has `class="intro"`, which colour wins? The browser uses three rules, checked in this order:

1. **Specificity** — the more specific selector wins.
2. **Source order** — if specificity is equal, the rule written *later* in the file wins.
3. **Importance** — `!important` overrides everything (use very sparingly).

---

### Specificity

Each type of selector has a different "weight". Higher weight wins.

| Selector type | Weight |
| ------------- | ------ |
| Inline style (`style=""`) | Highest — beats everything |
| ID (`#id`) | High |
| Class (`.class`), attribute (`[type]`), pseudo-class (`:hover`) | Medium |
| Type / element (`p`, `div`, `h1`) | Low |
| Universal (`*`), combinators | No weight |

```css
p { color: black; }         /* weight: low (type selector) */
.intro { color: blue; }     /* weight: medium (class selector) — this wins */
#hero { color: red; }       /* weight: high (ID selector) — this wins over both */
```

> **Key rule:** A higher *tier* always beats a lower one. No matter how many classes you stack, they never beat a single ID.

```css
/* These 10 classes combined (0,0,10,0) still lose to one ID (0,1,0,0) */
.a.b.c.d.e.f.g.h.i.j { color: blue; }
#unique { color: red; }  /* red always wins */
```

---

### Inheritance

Some CSS properties automatically pass down from a parent element to all its children. Others do not.

```html
<div class="container">
  <p>This text inherits the container's colour and font.</p>
  <p>So does this one.</p>
</div>
```

```css
.container {
  color: #333;
  font-family: Georgia, serif;
  font-size: 18px;
}
/* The <p> elements inherit color, font-family, and font-size automatically */
```

**Properties that inherit by default:** `color`, `font-family`, `font-size`, `font-weight`, `line-height`, `text-align`, `list-style`

**Properties that do NOT inherit:** `margin`, `padding`, `border`, `background`, `width`, `height`

You can force or cancel inheritance with these keywords:

```css
.child {
  color: inherit;   /* force it to take the parent's value */
  border: initial;  /* reset to the browser's default value */
  margin: unset;    /* remove any set value and fall back to inherited or default */
}
```

## 3. Common CSS Properties

### Colour

CSS accepts colours in several formats. They all describe the same thing — a colour — just in different notations.

```css
p {
  /* Named colour — simple but limited options */
  color: tomato;

  /* Hex — the most common format. #RRGGBB, where each pair is 00–FF */
  color: #ff6347;

  /* RGB — Red, Green, Blue, each from 0 to 255 */
  color: rgb(255, 99, 71);

  /* HSL — Hue (0–360 degrees on a colour wheel), Saturation %, Lightness % */
  color: hsl(9, 100%, 64%);

  /* Background colour works the same way */
  background-color: #f0f0f0;
}
```

**Which format to use?**

- **Hex** (`#ff6347`) — the industry standard for specific colours. Copy from a design tool.
- **HSL** — great when you want to adjust colours logically (make it lighter, more saturated).
- **Named** (`red`, `steelblue`) — useful for quick prototyping.

---

### Typography

Typography properties control how text looks.

| Property | What it controls | Example |
| -------- | ---------------- | ------- |
| `font-family` | The typeface | `'Georgia', serif` |
| `font-size` | How big the text is | `16px`, `1rem`, `1.2em` |
| `font-weight` | How thick the letters are | `400` (normal), `700` (bold) |
| `font-style` | Normal or italic | `italic`, `normal` |
| `line-height` | Vertical space between lines | `1.5` (unitless — recommended) |
| `letter-spacing` | Horizontal space between characters | `0.05em` |
| `text-align` | How text lines up horizontally | `left`, `center`, `right` |
| `text-decoration` | Underlines, strikethrough | `none`, `underline` |
| `text-transform` | Force case conversion | `uppercase`, `capitalize` |

```css
body {
  font-family: 'Segoe UI', Arial, sans-serif;
  font-size: 16px;
  line-height: 1.6;   /* 1.6× the font size — comfortable reading */
  color: #222;
}

h1 {
  font-size: 2.5rem;  /* 2.5× the root font size */
  font-weight: 700;   /* bold */
  letter-spacing: 0.05em;
  text-transform: uppercase;
}

a {
  text-decoration: none;  /* remove the default underline from links */
  color: steelblue;
}
```

**`font-family` fallbacks:** Always list multiple fonts separated by commas. The browser uses the first one it has installed; if not available, it moves to the next. Always end with a generic family (`serif`, `sans-serif`, `monospace`) as a final fallback.

## 4. The Box Model

Every single HTML element — a paragraph, a button, a div — is rendered by the browser as a rectangular box. That box is made up of four layers from the inside out:

```text
┌─────────────────────────────────────┐
│              MARGIN                 │  ← Space OUTSIDE the element
│  ┌───────────────────────────────┐  │
│  │           BORDER              │  │  ← A visible line around the element
│  │  ┌─────────────────────────┐  │  │
│  │  │        PADDING          │  │  │  ← Space INSIDE the border
│  │  │  ┌───────────────────┐  │  │  │
│  │  │  │     CONTENT       │  │  │  │  ← The actual text or image
│  │  │  └───────────────────┘  │  │  │
│  │  └─────────────────────────┘  │  │
│  └───────────────────────────────┘  │
└─────────────────────────────────────┘
```

| Layer | Property | What it does |
| ----- | -------- | ------------ |
| **Content** | `width`, `height` | The area where text and child elements live |
| **Padding** | `padding` | Space *inside* the border — pushes content away from the edge |
| **Border** | `border` | A visible line that wraps around padding and content |
| **Margin** | `margin` | Space *outside* the border — pushes other elements away |

```css
.card {
  width: 300px;
  padding: 20px;           /* 20px of space between content and border */
  border: 2px solid #ccc;  /* a 2px grey border */
  margin: 16px;            /* 16px space around the outside */
}
```

**Padding vs. Margin — the key difference:**

- **Padding** is inside the element — increasing it makes the element visually bigger and pushes the content inward. The background colour fills the padding area.
- **Margin** is outside the element — it creates space between this element and its neighbours. The background colour does not fill the margin area.

---

### `box-sizing: border-box`

By default, when you set `width: 300px`, that only sizes the *content area*. Padding and border get added on top, making the final element wider than 300px. This is confusing.

Setting `box-sizing: border-box` changes the behaviour so that `width` and `height` include the padding and border. What you set is what you get.

```css
/* Apply this to everything — standard practice in modern CSS */
*,
*::before,
*::after {
  box-sizing: border-box;
}

.card {
  width: 300px;     /* now the total width is exactly 300px, including padding and border */
  padding: 20px;
  border: 2px solid #ccc;
}
```

---

### Margin & Padding Shorthand

Instead of setting each side individually, you can use shorthand. The values go clockwise: **Top → Right → Bottom → Left**.

```css
/* One value — all four sides the same */
margin: 10px;

/* Two values — top/bottom | left/right */
padding: 10px 20px;

/* Three values — top | left/right | bottom */
margin: 10px 20px 5px;

/* Four values — top | right | bottom | left (clockwise) */
padding: 10px 20px 15px 5px;

/* Individual sides */
margin-top: 10px;
padding-left: 20px;
```

## 5. Display & Layout Basics

### Normal Flow

**Normal flow** is what happens when you write HTML and apply no CSS layout at all. Elements just stack or sit next to each other based on their default display type. Every layout technique (Flexbox, Grid, positioning) is a way of changing or taking control of this default behaviour.

There are two behaviours in normal flow:

- **Block** — elements stack vertically, one below the other.
- **Inline** — elements sit side by side, flowing like words in a sentence.

---

### Block Elements

A block element:

- Always starts on a **new line**.
- Stretches to fill the **full width** of its parent container.
- Respects `width`, `height`, and `margin`/`padding` on all sides.

```html
<p>First paragraph</p>
<p>Second paragraph — starts on a new line even though they're inline in the HTML</p>
```

Common block elements: `<div>`, `<p>`, `<h1>`–`<h6>`, `<ul>`, `<ol>`, `<section>`, `<article>`, `<header>`, `<footer>`, `<form>`

---

### Inline Elements

An inline element:

- Does **not** start on a new line — it flows within surrounding text.
- Only takes up **as much width as its content**.
- **Ignores** `width` and `height` — setting them has no effect.
- Respects left/right `margin` and `padding`, but vertical margin/padding does not push other elements away.

```html
<p>
  I am learning <strong>HTML</strong>, <em>CSS</em>, and
  <a href="#">JavaScript</a> at the same time.
</p>
```

All the inline elements (`<strong>`, `<em>`, `<a>`) sit within the text flow of the `<p>`.

Common inline elements: `<span>`, `<a>`, `<strong>`, `<em>`, `<img>`, `<code>`, `<button>`

---

### `inline-block` — The Middle Ground

`inline-block` combines the best of both: it sits within the text flow like an inline element, but it also respects `width`, `height`, and vertical margin/padding like a block element.

```css
/* A badge that sits within text but has a fixed size */
.badge {
  display: inline-block;
  width: 80px;
  padding: 4px 8px;
  background-color: steelblue;
  color: white;
  border-radius: 4px;
  text-align: center;
}
```

---

### `<div>` and `<span>` — Generic Containers

`<div>` and `<span>` have no semantic meaning. Use them purely for grouping elements together so you can target them with CSS or JavaScript.

```html
<div class="card">
  <h2>Card Title</h2>
  <p>Some text with a <span class="highlight">key phrase</span> inside it.</p>
</div>
```

```css
.card {
  border: 1px solid #ddd;
  padding: 16px;
  border-radius: 8px;
  background-color: white;
}

.highlight {
  background-color: yellow;
  font-weight: bold;
}
```

| Tag | Default display | Use for |
| --- | --------------- | ------- |
| `<div>` | Block | Grouping multiple block-level elements for layout |
| `<span>` | Inline | Targeting a word or short phrase inside text |

> Use semantic elements (`<article>`, `<nav>`, `<header>`) whenever they fit. Reach for `<div>` and `<span>` only when nothing semantic applies.

---

### The `display` Property

You can override any element's default display behaviour with the `display` property.

| Value | Behaviour |
| ----- | --------- |
| `block` | Full width, starts on a new line |
| `inline` | Flows with text, no width/height |
| `inline-block` | Flows with text, but respects width/height |
| `none` | Completely removes the element from the page (not just hidden — it takes up no space) |
| `flex` | Enables Flexbox layout on the element's children |
| `grid` | Enables CSS Grid layout on the element's children |

```css
/* Make a normally block <li> sit side by side (like a nav bar) */
li {
  display: inline-block;
  margin-right: 16px;
}

/* Hide an element completely */
.hidden {
  display: none;
}
```

---

### Block vs. Inline — Quick Reference

| | Block | Inline |
| - | ----- | ------ |
| Starts on new line? | Yes | No |
| Takes full width? | Yes | No |
| Respects `width` / `height`? | Yes | No |
| Respects vertical `margin`/`padding`? | Yes | Partially |
| Can contain block elements? | Yes | No |
| Common examples | `<div>`, `<p>`, `<h1>` | `<span>`, `<a>`, `<strong>` |

## 6. CSS Units

CSS has many ways to express size. The right one to use depends on what you are sizing.

| Unit | Type | How it works |
| ---- | ---- | ------------ |
| `px` | Absolute | A fixed number of pixels. Predictable, but does not scale with user settings. |
| `%` | Relative | A percentage of the *parent* element's size. |
| `em` | Relative | Relative to the *current element's* `font-size`. If you nest elements, it compounds. |
| `rem` | Relative | Relative to the *root* (`<html>`) `font-size`. Consistent and predictable everywhere. |
| `vw` | Viewport | 1% of the browser window's *width*. |
| `vh` | Viewport | 1% of the browser window's *height*. |

```css
html {
  font-size: 16px; /* the root font size — 1rem = 16px */
}

h1 {
  font-size: 2rem;    /* 2 × 16px = 32px */
  margin-bottom: 1rem; /* 16px */
}

.sidebar {
  width: 25%;         /* 25% of the parent element's width */
}

.hero {
  height: 100vh;      /* fills the full height of the browser window */
  width: 100vw;       /* fills the full width of the browser window */
}
```

**When to use what:**

- **`rem`** — font sizes and spacing (consistent, scales with user preferences)
- **`%`** — widths inside a layout (relative to the container)
- **`vw`** / **`vh`** — full-screen sections, hero areas
- **`px`** — borders, small fixed values like `border: 1px solid`

## 7. Pseudo-classes & Pseudo-elements

### Pseudo-classes

Pseudo-classes let you style an element based on its **state** or **position**. They use a single colon (`:`).

The most common ones let you style elements when the user interacts with them:

```css
/* Change link colour when hovered */
a:hover {
  color: tomato;
  text-decoration: underline;
}

/* Highlight an input when the user clicks into it */
input:focus {
  outline: 2px solid steelblue;
  border-color: steelblue;
}

/* Style a button while it's being clicked */
button:active {
  transform: scale(0.98);
  background-color: darkblue;
}
```

Position-based pseudo-classes let you target specific items in a list or group:

```css
/* Style the first item differently */
li:first-child {
  font-weight: bold;
}

/* Alternate row colours in a table */
tr:nth-child(even) {
  background-color: #f5f5f5;
}

/* Style all items except the last one */
li:not(:last-child) {
  border-bottom: 1px solid #ddd;
}
```

| Pseudo-class | When it applies |
| ------------ | --------------- |
| `:hover` | Cursor is over the element |
| `:focus` | Element has keyboard/input focus |
| `:active` | Element is being clicked right now |
| `:visited` | An `<a>` link that the user has visited before |
| `:first-child` | Element is the first child of its parent |
| `:last-child` | Element is the last child of its parent |
| `:nth-child(n)` | Element is the nth child (can use `odd`, `even`, or a formula) |
| `:not(selector)` | Element does not match the given selector |

---

### Pseudo-elements

Pseudo-elements let you style a **specific part** of an element, or insert content before/after it. They use double colons (`::`).

```css
/* Insert a decorative character before every list item */
li::before {
  content: "→ ";
  color: steelblue;
}

/* Insert content after — useful for tooltips, badges, etc. */
.required::after {
  content: " *";
  color: red;
}

/* Style just the first line of a paragraph */
p::first-line {
  font-weight: bold;
  font-size: 1.1em;
}

/* Style the placeholder text in an input */
input::placeholder {
  color: #aaa;
  font-style: italic;
}
```

| Pseudo-element | What it targets |
| -------------- | --------------- |
| `::before` | Inserts generated content *before* the element's content |
| `::after` | Inserts generated content *after* the element's content |
| `::first-line` | The first rendered line of a block of text |
| `::first-letter` | The very first letter of a block of text |
| `::placeholder` | The placeholder text inside an `<input>` or `<textarea>` |

> `::before` and `::after` require a `content` property to work, even if it's just `content: ""`.

## 8. Flexbox

Flexbox is a layout system that makes it easy to arrange elements in a row or column, control their spacing, and align them. Before Flexbox, these tasks required hacky workarounds. Now, they take a few lines of CSS.

You enable Flexbox on a **container** element by setting `display: flex`. The container's direct children automatically become **flex items** and are laid out by Flexbox.

```html
<div class="container">
  <div class="item">One</div>
  <div class="item">Two</div>
  <div class="item">Three</div>
</div>
```

```css
.container {
  display: flex;
  /* The three divs now sit side by side in a row */
}
```

---

### Growing and Shrinking

One of Flexbox's most powerful features is that items can automatically grow to fill available space or shrink to avoid overflow. Three properties control this behaviour on **flex items** (the children):

| Property | Default | What it does |
| -------- | ------- | ------------ |
| `flex-grow` | `0` | How much the item grows to fill leftover space. `0` = don't grow. `1` = take a proportional share. |
| `flex-shrink` | `1` | How much the item shrinks when there isn't enough space. `0` = never shrink. |
| `flex-basis` | `auto` | The item's ideal starting size before growing/shrinking. Can be a length like `200px` or `auto`. |

These three properties are usually written together using the `flex` shorthand:

```css
/* flex: grow  shrink  basis */

.item {
  flex: 0 1 auto;   /* default — doesn't grow, can shrink, starts at auto size */
}

.item-grow {
  flex: 1;          /* grows to fill all available space */
}

.item-fixed {
  flex: 0 0 200px;  /* always exactly 200px — never grows, never shrinks */
}
```

**`flex: 1` explained step by step:**

`flex: 1` is shorthand for `flex: 1 1 0`. This means:

- `flex-grow: 1` — the item will grow if there's space.
- `flex-shrink: 1` — the item will shrink if there isn't enough space.
- `flex-basis: 0` — the item starts from zero width, so all available space is distributed equally.

If you have three items all set to `flex: 1`, they each get exactly one third of the container. If one is set to `flex: 2`, it gets twice the space of the others.

```css
.sidebar { flex: 1; }   /* gets 1 share */
.main    { flex: 3; }   /* gets 3 shares — 3× wider than sidebar */
```

---

### Axes

Flexbox is **one-dimensional** — it lays items out along one axis at a time. There are always two axes:

- **Main axis** — the direction items are placed. Set by `flex-direction`.
- **Cross axis** — the axis perpendicular (at 90°) to the main axis.

```text
flex-direction: row (default)

Main axis → (left to right)
┌──────┬──────┬──────┐
│  1   │  2   │  3   │  ↕ Cross axis (top to bottom)
└──────┴──────┴──────┘

flex-direction: column

Cross axis → (left to right)
┌──────┐
│  1   │
├──────┤  ↕ Main axis (top to bottom)
│  2   │
├──────┤
│  3   │
└──────┘
```

| Value | Items are placed… |
| ----- | ----------------- |
| `row` (default) | Left to right |
| `row-reverse` | Right to left |
| `column` | Top to bottom |
| `column-reverse` | Bottom to top |

```css
/* Stack items vertically instead of horizontally */
.container {
  display: flex;
  flex-direction: column;
}
```

**`flex-wrap`** controls what happens when items run out of space:

```css
.container {
  flex-wrap: nowrap;  /* default — items shrink to fit, may overflow */
  flex-wrap: wrap;    /* items wrap onto the next line when they run out of space */
}
```

---

### Alignment

This is where Flexbox really shines. You can control exactly how items are positioned and spaced along both axes.

#### `justify-content` — align along the main axis

```css
.container {
  display: flex;
  justify-content: flex-start;    /* default — packed to the start */
  justify-content: flex-end;      /* packed to the end */
  justify-content: center;        /* centred */
  justify-content: space-between; /* equal gaps between items, no gap at edges */
  justify-content: space-around;  /* equal gaps around each item */
  justify-content: space-evenly;  /* equal gaps between items AND edges */
}
```

#### `align-items` — align along the cross axis

```css
.container {
  display: flex;
  align-items: stretch;     /* default — items stretch to fill the cross axis height */
  align-items: flex-start;  /* items align to the top (for row direction) */
  align-items: flex-end;    /* items align to the bottom */
  align-items: center;      /* items are vertically centred */
  align-items: baseline;    /* items align by their text baseline */
}
```

#### `align-self` — override alignment for one item

Any flex item can override the container's `align-items` value for just itself:

```css
.special {
  align-self: flex-end; /* this item goes to the bottom, others follow align-items */
}
```

#### Perfectly centring an element

This is one of the most common patterns in web design, and Flexbox makes it trivial:

```css
.container {
  display: flex;
  justify-content: center; /* centre horizontally (main axis) */
  align-items: center;     /* centre vertically (cross axis) */
  height: 100vh;           /* needs some height for vertical centering to work */
}
```

#### `gap` — space between items

```css
.container {
  display: flex;
  gap: 16px;           /* equal gap between all items */
  gap: 16px 8px;       /* row-gap | column-gap */
}
```

#### Summary

| Property | Set on | Controls |
| -------- | ------ | -------- |
| `flex-direction` | Container | Which direction is the main axis |
| `flex-wrap` | Container | Whether items wrap to a new line |
| `justify-content` | Container | Spacing/position along the **main axis** |
| `align-items` | Container | Alignment along the **cross axis** |
| `align-self` | Individual item | Overrides `align-items` for one item |
| `gap` | Container | Space between items |
| `flex` | Individual item | How the item grows, shrinks, and what size it starts at |

## Quick Concept Reminder: The Cascade in One Sentence

The browser collects **all** CSS rules that match an element, then applies the one with the highest **specificity** — and if two rules have the same specificity, the one written **last** in the file wins.
