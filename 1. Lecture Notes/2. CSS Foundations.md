# CSS Foundations

---

## Introduction to CSS

**What is CSS and what is it used for?**
CSS (Cascading Style Sheets) is the language used to style the visual presentation of HTML documents — controlling colours, fonts, spacing, layout, and more.

---

**What is the syntax of a CSS rule?**
A CSS rule has two parts: a **selector** that targets the HTML element(s), and a **declaration block** containing one or more property-value pairs.

```css
selector {
  property: value;
  property: value;
}
```

---

**What are the three ways to add CSS to an HTML document?**

| Method | How | When to use |
| ------ | --- | ----------- |
| **External** | `<link rel="stylesheet" href="style.css">` in `<head>` | Always — keeps HTML and CSS separate |
| **Internal** | `<style>` block inside `<head>` | Quick prototypes or single-page demos |
| **Inline** | `style=""` attribute on an element | Avoid — hard to maintain and overrides everything |

## 1. Selectors

Selectors tell the browser which HTML elements a CSS rule applies to.

### Basic Selectors

| Selector | Syntax | Targets |
| -------- | ------ | ------- |
| **Universal** | `*` | Every element on the page |
| **Type / Element** | `div`, `p`, `h1` | All elements of that HTML tag |
| **Class** | `.classname` | All elements with that `class` attribute |
| **ID** | `#idname` | The single element with that `id` attribute |

```css
/* Universal */
* {
  box-sizing: border-box;
}

/* Type */
p {
  color: #333;
}

/* Class */
.card {
  border: 1px solid #ccc;
}

/* ID */
#hero {
  background-color: navy;
}
```

### Grouping & Chaining

**Grouping** — apply the same rule to multiple selectors at once by separating them with commas.

```css
h1,
h2,
h3 {
  font-family: sans-serif;
}
```

**Chaining** — combine selectors with no space between them to target elements that match *all* conditions simultaneously.

```css
/* Targets <p> elements that also have class "highlight" */
p.highlight {
  background-color: yellow;
}

/* Targets elements with BOTH classes "btn" and "primary" */
.btn.primary {
  background-color: blue;
}
```

### Combinator Selectors

| Combinator | Syntax | Targets |
| ---------- | ------ | ------- |
| **Descendant** | `A B` | Any `B` that is a descendant of `A` (any depth) |
| **Child** | `A > B` | Any `B` that is a *direct* child of `A` only |
| **Adjacent sibling** | `A + B` | The `B` immediately after `A` at the same level |
| **General sibling** | `A ~ B` | All `B` siblings that follow `A` at the same level |

```css
/* Any <a> anywhere inside a <nav> */
nav a {
  text-decoration: none;
}

/* Only <li> elements that are direct children of <ul> */
ul > li {
  list-style: disc;
}
```

## 2. The Cascade, Specificity & Inheritance

### The Cascade

When multiple CSS rules target the same element and property, the browser resolves the conflict using three factors in order:

1. **Specificity** — which selector is more specific wins.
2. **Source order** — if specificity is equal, the rule written *later* in the file wins.
3. **Importance** — `!important` overrides all of the above (use sparingly).

### Specificity

Each selector type is assigned a weight. Higher weight wins.

| Selector type | Specificity value |
| ------------- | ----------------- |
| Inline styles (`style=""`) | Highest (1,0,0,0) |
| ID (`#id`) | (0,1,0,0) |
| Class, attribute, pseudo-class (`.class`, `[attr]`, `:hover`) | (0,0,1,0) |
| Type / element (`div`, `p`) | (0,0,0,1) |
| Universal (`*`), combinators | (0,0,0,0) |

> A higher column always beats a lower one. Ten classes (0,0,10,0) do **not** outweigh a single ID (0,1,0,0).

### Inheritance

Some CSS properties are automatically passed down from a parent element to its children. Others are not.

- **Inherited by default:** `color`, `font-family`, `font-size`, `line-height`, `text-align`, `list-style`
- **Not inherited by default:** `margin`, `padding`, `border`, `background`, `width`, `height`

You can explicitly force or block inheritance with the keywords `inherit`, `initial`, and `unset`.

```css
/* Force a child to use the parent's border value */
.child {
  border: inherit;
}
```

## 3. Common CSS Properties

### Colour

```css
p {
  color: #1a1a1a;              /* Hex */
  color: rgb(26, 26, 26);      /* RGB */
  color: hsl(0, 0%, 10%);      /* HSL */
  background-color: #f5f5f5;
}
```

### Typography

| Property | What it controls |
| -------- | ---------------- |
| `font-family` | Typeface (e.g., `Arial, sans-serif`) |
| `font-size` | Size of the text (`px`, `rem`, `em`, `%`) |
| `font-weight` | Thickness of the text (`normal`, `bold`, `100`–`900`) |
| `font-style` | Italic or normal (`italic`, `normal`) |
| `line-height` | Vertical space between lines (unitless value recommended, e.g., `1.5`) |
| `letter-spacing` | Space between characters |
| `text-align` | Horizontal alignment (`left`, `center`, `right`, `justify`) |
| `text-decoration` | Underline, strikethrough, none, etc. |
| `text-transform` | Case conversion (`uppercase`, `lowercase`, `capitalize`) |

```css
body {
  font-family: 'Segoe UI', Arial, sans-serif;
  font-size: 16px;
  line-height: 1.6;
  color: #222;
}

h1 {
  font-size: 2rem;
  font-weight: 700;
  text-transform: uppercase;
  letter-spacing: 0.05em;
}
```

## 4. The Box Model

Every HTML element is rendered as a rectangular box made up of four layers from inside out:

```text
┌─────────────────────────────────────┐
│              MARGIN                 │
│  ┌───────────────────────────────┐  │
│  │           BORDER              │  │
│  │  ┌─────────────────────────┐  │  │
│  │  │        PADDING          │  │  │
│  │  │  ┌───────────────────┐  │  │  │
│  │  │  │     CONTENT       │  │  │  │
│  │  │  └───────────────────┘  │  │  │
│  │  └─────────────────────────┘  │  │
│  └───────────────────────────────┘  │
└─────────────────────────────────────┘
```

| Layer | Property | What it does |
| ----- | -------- | ------------ |
| **Content** | `width`, `height` | The area where text and child elements live |
| **Padding** | `padding` | Space *inside* the border, between content and border |
| **Border** | `border` | A line drawn around the padding and content |
| **Margin** | `margin` | Space *outside* the border, pushing other elements away |

### `box-sizing`

By default (`content-box`), `width` and `height` only size the content area — padding and border are added on top, making the element larger than you specified.

Setting `box-sizing: border-box` makes `width` and `height` include padding and border, so what you set is what you get.

```css
/* Best practice — apply to everything */
*,
*::before,
*::after {
  box-sizing: border-box;
}
```

### Shorthand

```css
/* All four sides */
margin: 10px;

/* Top/Bottom | Left/Right */
padding: 10px 20px;

/* Top | Left/Right | Bottom */
margin: 10px 20px 5px;

/* Top | Right | Bottom | Left (clockwise) */
padding: 10px 20px 10px 20px;

/* Individual sides */
margin-top: 10px;
padding-left: 20px;
```

## 5. Display & Layout Basics

### The `display` Property

`display` controls how an element participates in the layout flow.

| Value | Behaviour |
| ----- | --------- |
| `block` | Takes up the full width available; starts on a new line. |
| `inline` | Only takes up as much width as its content; no top/bottom margin. |
| `inline-block` | Sits inline but accepts `width`, `height`, and vertical `margin`/`padding`. |
| `none` | Removes the element entirely from the layout (not just hidden). |
| `flex` | Enables Flexbox layout on the element's children. |
| `grid` | Enables CSS Grid layout on the element's children. |

### Block vs. Inline — Quick Reference

| | Block | Inline |
| - | ----- | ------ |
| Starts on new line? | Yes | No |
| Takes full width? | Yes | No |
| Respects `width`/`height`? | Yes | No |
| Respects vertical `margin`/`padding`? | Yes | Partially |
| Examples | `<div>`, `<p>`, `<h1>`, `<ul>` | `<span>`, `<a>`, `<strong>`, `<img>` |

## 6. CSS Units

| Unit | Type | Description |
| ---- | ---- | ----------- |
| `px` | Absolute | Fixed pixel value. Predictable, but not responsive. |
| `%` | Relative | Relative to the *parent* element's size. |
| `em` | Relative | Relative to the *current element's* `font-size`. Compounds when nested. |
| `rem` | Relative | Relative to the *root element's* (`<html>`) `font-size`. Consistent and predictable. |
| `vw` | Viewport | 1% of the viewport *width*. |
| `vh` | Viewport | 1% of the viewport *height*. |

> **Rule of thumb:** Use `rem` for font sizes and spacing, `%` or `vw`/`vh` for layout widths and heights.

## 7. Pseudo-classes & Pseudo-elements

### Pseudo-classes

Select elements based on their **state** or **position** in the DOM. They use a single colon (`:`).

| Pseudo-class | When it applies |
| ------------ | --------------- |
| `:hover` | User's cursor is over the element |
| `:focus` | Element has keyboard/input focus |
| `:active` | Element is being clicked |
| `:visited` | `<a>` link has been visited before |
| `:first-child` | Element is the first child of its parent |
| `:last-child` | Element is the last child of its parent |
| `:nth-child(n)` | Element is the *n*th child of its parent |
| `:not(selector)` | Element does **not** match the given selector |

```css
a:hover {
  color: tomato;
  text-decoration: underline;
}

li:nth-child(odd) {
  background-color: #f0f0f0;
}
```

### Pseudo-elements

Target a **specific part** of an element. They use a double colon (`::`).

| Pseudo-element | What it targets |
| -------------- | --------------- |
| `::before` | Inserts generated content before the element's content |
| `::after` | Inserts generated content after the element's content |
| `::first-line` | The first rendered line of a block element |
| `::first-letter` | The first letter of a block element |
| `::placeholder` | The placeholder text inside an `<input>` or `<textarea>` |

```css
/* Adds a decorative bullet before each list item */
li::before {
  content: "→ ";
  color: steelblue;
}

/* Styles the placeholder text */
input::placeholder {
  color: #aaa;
  font-style: italic;
}
```

---

## Quick Concept Reminder: The Cascade in One Sentence

The browser reads **all** CSS rules that match an element, then applies the one with the highest **specificity** — and if specificity ties, the one written **last** wins.
