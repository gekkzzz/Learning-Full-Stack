# Node.js Foundations

---

## What Is Node.js?

Node.js is a **JavaScript runtime** — it lets you run JavaScript on a server, outside of a browser. Before Node.js, JavaScript could only run in browsers. Now the same language you write for web UIs can also power your servers, CLI tools, build scripts, and APIs.

Node.js is built on Chrome's V8 JavaScript engine (the same engine that runs JavaScript in Google Chrome) plus a set of APIs for things the browser doesn't have: reading files, creating HTTP servers, accessing databases, and working with the operating system.

### Why Node.js became popular for servers

- **Same language everywhere** — one language for both frontend and backend
- **Non-blocking I/O** — Node handles many simultaneous requests efficiently without creating a thread for each one
- **npm ecosystem** — the largest package registry in the world
- **Fast for I/O-heavy work** — API servers, real-time apps, file processing

Node.js is not ideal for CPU-intensive work like video encoding or machine learning — for that, Python or a compiled language is better.

## The Event Loop

This is the most important concept to understand about Node.js. JavaScript is **single-threaded** — it can only do one thing at a time. Yet a Node.js server can handle thousands of simultaneous requests. How?

When Node.js starts an operation that takes time (reading a file, making a database query, calling an API), it doesn't wait. It starts the operation and moves on to the next piece of work. When the operation finishes, Node is notified via a **callback** and runs the follow-up code.

This is called **non-blocking, event-driven I/O**.

```js
// Blocking (how most languages work)
const data = readFileSync('large-file.txt'); // stops here until file is read
console.log('File read'); // runs after

// Non-blocking (Node.js way)
readFile('large-file.txt', (error, data) => {
  console.log('File read'); // runs when ready, later
});
console.log('This runs immediately, before the file is read');
```

Think of it like a restaurant: a waiter doesn't stand at your table watching your food cook. They take your order, submit it to the kitchen, then go serve other tables. When your food is ready, they're notified and bring it out.

## Built-in Modules

Node.js has a standard library of built-in modules. You import them with `require()` (CommonJS) or `import` (ES Modules).

### `fs` — File System

```js
import { readFile, writeFile, readdir, mkdir } from 'node:fs/promises';

// Read a file (async version — always prefer this)
const content = await readFile('data.txt', 'utf-8');
console.log(content);

// Write a file
await writeFile('output.txt', 'Hello, World!', 'utf-8');

// List files in a directory
const files = await readdir('./src');
console.log(files); // ['index.js', 'utils.js', ...]

// Create a directory
await mkdir('./logs', { recursive: true });
```

### `path` — Path Utilities

```js
import path from 'node:path';

path.join('/home', 'user', 'file.txt');     // '/home/user/file.txt'
path.resolve('src', 'index.js');             // absolute path from cwd
path.basename('/home/user/file.txt');        // 'file.txt'
path.dirname('/home/user/file.txt');         // '/home/user'
path.extname('photo.jpg');                   // '.jpg'
path.parse('/home/user/file.txt');
// { root: '/', dir: '/home/user', base: 'file.txt', ext: '.txt', name: 'file' }
```

Always use `path.join()` instead of manually concatenating paths with `/` — it handles OS differences (Windows uses `\`).

### `http` — Creating Servers

```js
import http from 'node:http';

const server = http.createServer((request, response) => {
  console.log(`${request.method} ${request.url}`);

  if (request.url === '/') {
    response.writeHead(200, { 'Content-Type': 'text/html' });
    response.end('<h1>Hello, World!</h1>');
  } else {
    response.writeHead(404);
    response.end('Not found');
  }
});

server.listen(3000, () => {
  console.log('Server running at http://localhost:3000');
});
```

In practice, you won't use the raw `http` module — frameworks like Express abstract over it.

### `os` — Operating System Info

```js
import os from 'node:os';

os.platform();    // 'linux', 'darwin', 'win32'
os.arch();        // 'x64', 'arm64'
os.homedir();     // '/home/username'
os.tmpdir();      // '/tmp'
os.cpus().length; // number of CPU cores
os.freemem();     // available memory in bytes
```

## npm and Package Management

npm (Node Package Manager) is included with Node.js. It installs packages from the npm registry and manages project dependencies.

```bash
# Initialise a new project (creates package.json)
npm init -y

# Install a package and save it as a dependency
npm install express

# Install a development-only package (not needed in production)
npm install -D typescript nodemon

# Install all dependencies listed in package.json
npm install

# Remove a package
npm uninstall express

# Run a script defined in package.json
npm run dev
npm run build
npm test
```

### `package.json`

```json
{
  "name": "my-app",
  "version": "1.0.0",
  "type": "module",
  "scripts": {
    "start": "node dist/index.js",
    "dev": "nodemon src/index.ts",
    "build": "tsc"
  },
  "dependencies": {
    "express": "^4.18.2"
  },
  "devDependencies": {
    "typescript": "^5.0.0",
    "nodemon": "^3.0.0"
  }
}
```

`"type": "module"` makes Node.js treat `.js` files as ES Modules (using `import`/`export`) instead of CommonJS (`require`).

### `node_modules` and `.gitignore`

The `node_modules` folder contains all installed packages. It can be hundreds of megabytes. **Never commit it to git** — add it to `.gitignore`. Anyone can recreate it by running `npm install`.

```
# .gitignore
node_modules/
dist/
.env
```

## Express — Building APIs

Express is a minimal web framework for Node.js. It gives you routing, middleware, and request/response helpers.

```js
import express from 'express';

const app = express();
const PORT = process.env.PORT ?? 3000;

// Middleware: parse JSON request bodies
// Without this, req.body would be undefined for POST requests
app.use(express.json());

// Route handlers
app.get('/', (req, res) => {
  res.json({ message: 'Hello, World!' });
});

// Route with URL parameter
app.get('/users/:id', async (req, res) => {
  const user = await db.findUser(req.params.id);
  if (!user) return res.status(404).json({ error: 'User not found' });
  res.json(user);
});

// POST route — receives body data
app.post('/users', async (req, res) => {
  const { name, email } = req.body;
  const user = await db.createUser({ name, email });
  res.status(201).json(user);
});

app.listen(PORT, () => console.log(`Server at http://localhost:${PORT}`));
```

### Route parameters and query strings

```js
// URL: /products/42
req.params.id // '42'

// URL: /products?category=electronics&page=2
req.query.category // 'electronics'
req.query.page     // '2'

// Request body (POST/PUT)
req.body.name  // from JSON or form data
```

## Middleware

Middleware functions run between the request arriving and the response being sent. They can modify the request and response, end the request, or call the next middleware in the chain.

```js
// Logger middleware — runs for every request
app.use((req, res, next) => {
  console.log(`${new Date().toISOString()} ${req.method} ${req.url}`);
  next(); // call next() to pass control to the next middleware
});

// Authentication middleware
function requireAuth(req, res, next) {
  const token = req.headers.authorization?.split(' ')[1];
  if (!token) return res.status(401).json({ error: 'No token provided' });

  try {
    req.user = verifyToken(token); // attach user to request
    next();
  } catch {
    res.status(401).json({ error: 'Invalid token' });
  }
}

// Apply to specific routes
app.get('/protected', requireAuth, (req, res) => {
  res.json({ message: `Hello, ${req.user.name}` });
});
```

### Commonly used third-party middleware

```js
import cors from 'cors';           // allow cross-origin requests
import helmet from 'helmet';       // security headers
import rateLimit from 'express-rate-limit';

app.use(cors({ origin: 'https://myapp.com' }));
app.use(helmet());
app.use(rateLimit({ windowMs: 15 * 60 * 1000, max: 100 })); // 100 req per 15 min
```

## Environment Variables

Never hardcode secrets (API keys, database passwords) in your code. Use environment variables instead.

```bash
# .env file (never commit this)
PORT=3000
DATABASE_URL=postgresql://user:pass@localhost/mydb
JWT_SECRET=super-secret-key
```

```js
import 'dotenv/config'; // loads .env file into process.env

const port = process.env.PORT ?? 3000;
const dbUrl = process.env.DATABASE_URL;
```

Install dotenv: `npm install dotenv`

## Error Handling

```js
// Async route handlers need try/catch, or use a wrapper
app.get('/users/:id', async (req, res, next) => {
  try {
    const user = await getUser(req.params.id);
    res.json(user);
  } catch (error) {
    next(error); // pass to Express error handler
  }
});

// A wrapper to avoid try/catch on every route
const asyncHandler = fn => (req, res, next) => Promise.resolve(fn(req, res, next)).catch(next);

app.get('/users/:id', asyncHandler(async (req, res) => {
  const user = await getUser(req.params.id);
  res.json(user);
}));

// Global error handler — must have 4 parameters
app.use((error, req, res, next) => {
  console.error(error.stack);
  res.status(error.status ?? 500).json({
    error: error.message ?? 'Internal Server Error',
  });
});
```
