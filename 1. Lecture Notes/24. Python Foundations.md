## What Is Python?

Python is a general-purpose programming language famous for being readable and beginner-friendly. Python code reads almost like plain English — there are no semicolons, curly braces are replaced by indentation, and the syntax is deliberately minimal.

Python is used everywhere:
- **Web development** — backends with Django and FastAPI
- **Data science and machine learning** — NumPy, pandas, TensorFlow, PyTorch
- **Scripting and automation** — replacing shell scripts for complex tasks
- **DevOps** — Ansible, cloud SDK tools
- **APIs and microservices**

For full-stack development, Python is most commonly the backend language powering an API that a React or Next.js frontend consumes.

## Variables and Types

Unlike JavaScript or TypeScript, Python variables have no declaration keyword. You just assign.

```python
name = "Alice"         # str (string)
age = 28               # int
height = 1.75          # float
is_active = True       # bool (note: capital T/F)
nothing = None         # equivalent to null/nil/undefined

# Python infers types automatically — but they are still real types
type(name)    # <class 'str'>
type(age)     # <class 'int'>
type(nothing) # <class 'NoneType'>

# Type hints (optional — like TypeScript but not enforced at runtime)
name: str = "Alice"
age: int = 28
```

## Lists

A list is Python's equivalent of a JavaScript array — an ordered, mutable collection.

```python
fruits = ["apple", "banana", "cherry"]

# Access by index (0-based, negative indexes count from end)
fruits[0]    # 'apple'
fruits[-1]   # 'cherry'
fruits[1:3]  # ['banana', 'cherry']  (slicing: from index 1 up to but not including 3)

# Modify
fruits.append("date")         # add to end
fruits.insert(1, "avocado")   # insert at index
fruits.remove("banana")       # remove by value
fruits.pop()                  # remove and return last item
fruits.pop(0)                 # remove and return item at index

# Other operations
len(fruits)              # number of items
"apple" in fruits        # True
fruits.sort()            # sort in place (alphabetically for strings)
fruits.reverse()         # reverse in place
sorted(fruits)           # return new sorted list without modifying original
list(reversed(fruits))   # return new reversed list

# Comprehensions — create a new list based on another
numbers = [1, 2, 3, 4, 5]
squares = [n ** 2 for n in numbers]         # [1, 4, 9, 16, 25]
evens   = [n for n in numbers if n % 2 == 0] # [2, 4]
```

## Dictionaries

A dictionary (dict) stores key-value pairs — Python's equivalent of a JavaScript object.

```python
user = {
    "name": "Alice",
    "age": 28,
    "active": True,
}

# Access
user["name"]         # 'Alice'
user.get("email")    # None (no error if key missing)
user.get("email", "default@example.com")  # returns default if missing

# Modify
user["email"] = "alice@example.com"  # add or update
del user["active"]                   # delete a key

# Checks
"name" in user       # True
"email" in user      # False (we deleted it... wait we added it above)

# Iteration
for key in user:
    print(key, user[key])

for key, value in user.items():
    print(f"{key}: {value}")

user.keys()    # dict_keys(['name', 'age', 'email'])
user.values()  # dict_values(['Alice', 28, 'alice@example.com'])

# Dict comprehension
squared = {n: n**2 for n in range(1, 6)}  # {1:1, 2:4, 3:9, 4:16, 5:25}
```

## Tuples and Sets

```python
# Tuple — like a list but IMMUTABLE (cannot be changed after creation)
point = (10, 20)
x, y = point          # unpacking
rgb = (255, 128, 0)
r, g, b = rgb

# Sets — unordered collection of UNIQUE values
tags = {"python", "backend", "api", "python"}  # duplicates removed
tags.add("database")
tags.remove("backend")
"python" in tags      # True
{1, 2, 3} | {3, 4, 5} # union: {1, 2, 3, 4, 5}
{1, 2, 3} & {3, 4, 5} # intersection: {3}
{1, 2, 3} - {3, 4, 5} # difference: {1, 2}
```

## Control Flow

```python
# if / elif / else
age = 18

if age >= 65:
    print("Senior")
elif age >= 18:
    print("Adult")
elif age >= 13:
    print("Teen")
else:
    print("Child")

# Ternary (one-liner if/else)
label = "Adult" if age >= 18 else "Minor"

# Falsy values: None, 0, "", [], {}, set()
if user:        # True if user is not None/empty
    print(user)

# for loop
for fruit in ["apple", "banana", "cherry"]:
    print(fruit)

# range
for i in range(5):        # 0, 1, 2, 3, 4
    print(i)

for i in range(2, 10, 2): # 2, 4, 6, 8
    print(i)

# enumerate — get index and value
for i, fruit in enumerate(["apple", "banana", "cherry"]):
    print(f"{i}: {fruit}")

# while loop
count = 0
while count < 5:
    print(count)
    count += 1
```

## Functions

```python
def greet(name, greeting="Hello"):
    """
    Greets a person. The docstring (triple quotes) explains what the function does.
    """
    return f"{greeting}, {name}!"

greet("Alice")           # 'Hello, Alice!'
greet("Alice", "Hi")     # 'Hi, Alice!'
greet(greeting="Hey", name="Bob")  # keyword arguments — order doesn't matter

# *args — collect extra positional arguments as a tuple
def add(*numbers):
    return sum(numbers)

add(1, 2, 3, 4)  # 10

# **kwargs — collect extra keyword arguments as a dict
def create_user(name, **details):
    return {"name": name, **details}

create_user("Alice", age=28, role="admin")
# {'name': 'Alice', 'age': 28, 'role': 'admin'}

# Lambda — anonymous function (for simple one-liners)
square = lambda x: x ** 2
sorted_users = sorted(users, key=lambda u: u["age"])
```

## Classes

```python
class User:
    # Class variable — shared across all instances
    count = 0

    def __init__(self, name, email):
        # Instance variables — unique to each instance
        self.name = name
        self.email = email
        self.active = True
        User.count += 1

    def greet(self):
        return f"Hi, I'm {self.name}"

    def deactivate(self):
        self.active = False

    # String representation — what print(user) shows
    def __repr__(self):
        return f"User(name={self.name!r}, email={self.email!r})"


# Create instances
alice = User("Alice", "alice@example.com")
bob   = User("Bob", "bob@example.com")

alice.greet()   # 'Hi, I'm Alice'
User.count      # 2

# Inheritance
class Admin(User):
    def __init__(self, name, email, permissions):
        super().__init__(name, email)   # call parent __init__
        self.permissions = permissions

    def greet(self):
        return f"{super().greet()} — I'm an admin"
```

## Error Handling

```python
try:
    result = 10 / 0
except ZeroDivisionError:
    print("Cannot divide by zero")

try:
    data = int("not a number")
except ValueError as e:
    print(f"Value error: {e}")
except (TypeError, AttributeError) as e:
    print(f"Type or attribute error: {e}")
else:
    print("No error occurred")  # runs only if no exception
finally:
    print("Always runs")         # runs regardless

# Raise your own exceptions
def get_user(id):
    if id <= 0:
        raise ValueError(f"ID must be positive, got {id}")
    return db.find(id)
```

## Virtual Environments

A virtual environment is an isolated Python installation for a project. Without it, all your Python projects share the same packages — and different projects often need different versions.

```bash
# Create a virtual environment
python -m venv venv

# Activate it
source venv/bin/activate       # Mac/Linux
venv\Scripts\activate          # Windows

# Now install packages — they go into venv/, not system Python
pip install fastapi uvicorn

# Save dependencies to a file
pip freeze > requirements.txt

# Install from requirements.txt (e.g. on a new machine)
pip install -r requirements.txt

# Deactivate
deactivate
```

Always activate your virtual environment before working on a project. Most IDEs (VS Code, PyCharm) do this automatically if configured correctly.
