# Python Full Stack Development

---

## Python as a Backend Language

In full-stack development, Python typically runs on the **server side** — handling API requests, business logic, authentication, and database access. A JavaScript/TypeScript frontend (React, Next.js, or plain HTML) communicates with the Python backend over HTTP, exchanging JSON data.

The flow looks like this:

```
Browser (React)  →  HTTP request  →  Python API  →  Database
Browser (React)  ←  JSON response ←  Python API  ←  Database
```

The two dominant Python web frameworks for modern APIs are **FastAPI** and **Django**.

| | FastAPI | Django |
| - | ------- | ------ |
| Philosophy | Minimal, modern, fast | Batteries-included |
| API style | REST or GraphQL | REST (with DRF) or GraphQL |
| Performance | Very fast (async-first) | Fast (supports async) |
| ORM included | No (use SQLAlchemy or Tortoise) | Yes (Django ORM) |
| Admin panel | No | Yes (built-in) |
| Best for | APIs, microservices | Full apps with admin |

## FastAPI — Building Modern APIs

FastAPI uses Python type hints to automatically validate requests and generate API documentation. It's designed for speed and developer experience.

```bash
pip install fastapi uvicorn[standard]
uvicorn main:app --reload  # run with auto-reload
```

```python
# main.py
from fastapi import FastAPI, HTTPException, Depends
from pydantic import BaseModel, EmailStr
from typing import Optional

app = FastAPI(title="My API", version="1.0.0")

# Pydantic models: define the shape of request/response data
# FastAPI validates incoming data against these models automatically
class UserCreate(BaseModel):
    name: str
    email: EmailStr
    age: Optional[int] = None  # Optional field

class UserResponse(BaseModel):
    id: int
    name: str
    email: str

    class Config:
        from_attributes = True  # allow creating from ORM objects


# In-memory "database" for the example
users_db: list[dict] = []
next_id = 1

# Routes
@app.get("/")
def root():
    return {"message": "API is running"}

@app.get("/users", response_model=list[UserResponse])
def list_users():
    return users_db

@app.get("/users/{user_id}", response_model=UserResponse)
def get_user(user_id: int):
    user = next((u for u in users_db if u["id"] == user_id), None)
    if not user:
        raise HTTPException(status_code=404, detail="User not found")
    return user

@app.post("/users", response_model=UserResponse, status_code=201)
def create_user(user: UserCreate):
    global next_id
    new_user = {"id": next_id, **user.model_dump()}
    users_db.append(new_user)
    next_id += 1
    return new_user

@app.delete("/users/{user_id}", status_code=204)
def delete_user(user_id: int):
    global users_db
    original_len = len(users_db)
    users_db = [u for u in users_db if u["id"] != user_id]
    if len(users_db) == original_len:
        raise HTTPException(status_code=404, detail="User not found")
```

FastAPI auto-generates interactive documentation at `/docs` (Swagger UI) and `/redoc`.

## Pydantic — Data Validation

Pydantic is the validation library FastAPI is built on. It validates data using Python type hints and raises clear errors if the data doesn't match.

```python
from pydantic import BaseModel, EmailStr, Field, validator
from datetime import datetime
from typing import Optional

class Product(BaseModel):
    name: str = Field(..., min_length=1, max_length=100)
    price: float = Field(..., gt=0, description="Must be greater than 0")
    description: Optional[str] = None
    tags: list[str] = []

    # Custom validator
    @validator("name")
    def name_must_not_be_empty(cls, v):
        if v.strip() == "":
            raise ValueError("Name cannot be blank")
        return v.strip()

# Validation happens automatically when you create an instance
product = Product(name="Laptop", price=999.99, tags=["electronics"])
product.model_dump()  # convert to dict
product.model_dump_json()  # convert to JSON string

# Invalid data raises ValidationError with detailed messages
try:
    bad = Product(name="", price=-50)
except Exception as e:
    print(e)  # clear error: price must be > 0, name cannot be blank
```

## SQLAlchemy — Database ORM

SQLAlchemy is the most powerful Python ORM. It lets you work with databases using Python classes instead of raw SQL.

```bash
pip install sqlalchemy psycopg2-binary  # SQLAlchemy + PostgreSQL driver
```

```python
from sqlalchemy import create_engine, Column, Integer, String, Boolean, DateTime, ForeignKey
from sqlalchemy.ext.declarative import declarative_base
from sqlalchemy.orm import sessionmaker, relationship
from datetime import datetime

DATABASE_URL = "postgresql://user:password@localhost/dbname"

engine = create_engine(DATABASE_URL)
SessionLocal = sessionmaker(autocommit=False, autoflush=False, bind=engine)
Base = declarative_base()

# Models — each class maps to a table
class User(Base):
    __tablename__ = "users"

    id         = Column(Integer, primary_key=True, index=True)
    name       = Column(String(100), nullable=False)
    email      = Column(String(255), unique=True, nullable=False, index=True)
    active     = Column(Boolean, default=True)
    created_at = Column(DateTime, default=datetime.utcnow)
    posts      = relationship("Post", back_populates="author")

class Post(Base):
    __tablename__ = "posts"

    id         = Column(Integer, primary_key=True)
    title      = Column(String(255), nullable=False)
    content    = Column(String, nullable=False)
    user_id    = Column(Integer, ForeignKey("users.id"), nullable=False)
    author     = relationship("User", back_populates="posts")

# Create all tables
Base.metadata.create_all(bind=engine)
```

```python
# Querying the database
def get_db():
    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()

# In FastAPI with dependency injection
from fastapi import Depends
from sqlalchemy.orm import Session

@app.get("/users/{user_id}")
def get_user(user_id: int, db: Session = Depends(get_db)):
    user = db.query(User).filter(User.id == user_id).first()
    if not user:
        raise HTTPException(404, "Not found")
    return user

@app.post("/users")
def create_user(data: UserCreate, db: Session = Depends(get_db)):
    user = User(name=data.name, email=data.email)
    db.add(user)
    db.commit()
    db.refresh(user)
    return user

@app.get("/users")
def list_users(skip: int = 0, limit: int = 20, db: Session = Depends(get_db)):
    return db.query(User).filter(User.active == True).offset(skip).limit(limit).all()
```

## Alembic — Database Migrations

Alembic tracks schema changes in version-controlled migration files, similar to what Prisma does for Node.js.

```bash
pip install alembic
alembic init alembic          # create alembic/ directory
alembic revision --autogenerate -m "create users table"  # generate migration
alembic upgrade head          # apply all migrations
alembic downgrade -1          # undo last migration
```

## Authentication with JWT

```python
from fastapi import Depends, HTTPException, status
from fastapi.security import OAuth2PasswordBearer, OAuth2PasswordRequestForm
import jwt
from passlib.context import CryptContext
from datetime import datetime, timedelta

SECRET_KEY = "your-secret-key"
ALGORITHM = "HS256"

pwd_context = CryptContext(schemes=["bcrypt"])
oauth2_scheme = OAuth2PasswordBearer(tokenUrl="/auth/token")

def hash_password(password: str) -> str:
    return pwd_context.hash(password)

def verify_password(plain: str, hashed: str) -> bool:
    return pwd_context.verify(plain, hashed)

def create_token(user_id: int) -> str:
    payload = {
        "sub": str(user_id),
        "exp": datetime.utcnow() + timedelta(days=7),
    }
    return jwt.encode(payload, SECRET_KEY, algorithm=ALGORITHM)

def get_current_user(token: str = Depends(oauth2_scheme), db: Session = Depends(get_db)):
    try:
        payload = jwt.decode(token, SECRET_KEY, algorithms=[ALGORITHM])
        user_id = int(payload["sub"])
    except jwt.ExpiredSignatureError:
        raise HTTPException(status_code=401, detail="Token expired")
    except jwt.InvalidTokenError:
        raise HTTPException(status_code=401, detail="Invalid token")

    user = db.query(User).get(user_id)
    if not user:
        raise HTTPException(status_code=401, detail="User not found")
    return user

# Protected route
@app.get("/me")
def get_me(current_user: User = Depends(get_current_user)):
    return current_user

# Login endpoint
@app.post("/auth/token")
def login(form: OAuth2PasswordRequestForm = Depends(), db: Session = Depends(get_db)):
    user = db.query(User).filter(User.email == form.username).first()
    if not user or not verify_password(form.password, user.hashed_password):
        raise HTTPException(status_code=401, detail="Invalid credentials")
    return {"access_token": create_token(user.id), "token_type": "bearer"}
```

## CORS — Connecting to Your Frontend

When your React app on `localhost:3000` calls your Python API on `localhost:8000`, the browser blocks it unless the server explicitly allows it (Cross-Origin Resource Sharing).

```python
from fastapi.middleware.cors import CORSMiddleware

app.add_middleware(
    CORSMiddleware,
    allow_origins=["http://localhost:3000", "https://myapp.com"],
    allow_credentials=True,
    allow_methods=["*"],     # GET, POST, PUT, DELETE, etc.
    allow_headers=["*"],     # Authorization, Content-Type, etc.
)
```

## Background Tasks and Queues

For long-running tasks (sending emails, processing uploads, generating reports), don't block the API response — run them in the background.

```python
from fastapi import BackgroundTasks

def send_welcome_email(email: str, name: str):
    # This runs after the response is sent
    email_client.send(to=email, subject=f"Welcome, {name}!")

@app.post("/users")
def create_user(data: UserCreate, background: BackgroundTasks, db: Session = Depends(get_db)):
    user = User(**data.model_dump())
    db.add(user)
    db.commit()
    background.add_task(send_welcome_email, user.email, user.name)
    return user  # response sent immediately; email sent in background
```

For heavier workloads, use **Celery** with Redis or RabbitMQ as the message broker.

## Testing with pytest

```python
# test_users.py
import pytest
from fastapi.testclient import TestClient
from main import app
from database import Base, engine

@pytest.fixture(autouse=True)
def reset_db():
    Base.metadata.create_all(engine)
    yield
    Base.metadata.drop_all(engine)

client = TestClient(app)

def test_create_user():
    response = client.post("/users", json={"name": "Alice", "email": "alice@test.com"})
    assert response.status_code == 201
    assert response.json()["name"] == "Alice"

def test_get_user_not_found():
    response = client.get("/users/999")
    assert response.status_code == 404

def test_list_users():
    client.post("/users", json={"name": "Alice", "email": "a@test.com"})
    client.post("/users", json={"name": "Bob", "email": "b@test.com"})
    response = client.get("/users")
    assert len(response.json()) == 2
```

```bash
pytest                   # run all tests
pytest -v                # verbose output
pytest --cov=. --cov-report=html  # coverage report
```

## Deployment

### Docker

```dockerfile
FROM python:3.12-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

EXPOSE 8000

CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
```

### docker-compose for local development

```yaml
# docker-compose.yml
services:
  api:
    build: .
    ports:
      - "8000:8000"
    environment:
      DATABASE_URL: postgresql://postgres:password@db/mydb
    depends_on:
      - db
    volumes:
      - .:/app  # mount source for hot reload

  db:
    image: postgres:16
    environment:
      POSTGRES_PASSWORD: password
      POSTGRES_DB: mydb
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data

volumes:
  postgres_data:
```

```bash
docker-compose up --build     # start everything
docker-compose exec api bash  # shell inside the container
```
