## Why Django Instead of FastAPI?

Both Django and FastAPI are Python web frameworks — so which do you pick?

| | FastAPI | Django |
| - | ------- | ------ |
| Philosophy | Minimal, you assemble the pieces | Batteries-included — everything built in |
| Learning curve | Lower — smaller surface area | Higher — more concepts upfront |
| Admin panel | None (build your own) | Built-in, auto-generated from your models |
| ORM | Bring your own (SQLAlchemy) | Built-in Django ORM |
| Authentication | Bring your own | Built-in (users, sessions, permissions) |
| Best for | Pure APIs, microservices, data pipelines | Full applications with admin, auth, CMS features |

**Choose FastAPI** when you're building a JSON API consumed by a separate frontend (React, mobile app) and you want to keep things lean.

**Choose Django** when you need a full application with an admin interface, built-in user management, and want the "just works" experience. Django's admin panel alone saves days of work for content-heavy projects.

Many teams use both: Django manages the admin/backoffice side, FastAPI handles high-performance public APIs.

---

## What Is Django?

Django is a high-level Python web framework built on the **Model-View-Template (MVT)** pattern. It was created in 2005 to power news websites and has since become one of the most popular web frameworks in the world. Instagram, Pinterest, Disqus, and Mozilla all use or have used Django.

Django's philosophy: **"Don't repeat yourself"** and **"Batteries included."** You get an ORM, admin panel, authentication, form handling, URL routing, templating, migrations, and security features out of the box.

```bash
# Create a virtual environment
python -m venv venv
```

## Models and the Django ORM

A **model** is a Python class that maps directly to a database table. Django generates SQL migrations from your model definitions.

```python
# blog/models.py
from django.db import models
from django.contrib.auth.models import User

class Category(models.Model):
    name = models.CharField(max_length=100)
    slug = models.SlugField(unique=True)  # URL-friendly version: "web-development"

    class Meta:
        verbose_name_plural = "categories"
        ordering = ['name']

    def __str__(self):
        return self.name


class Post(models.Model):
    STATUS_DRAFT = 'draft'
    STATUS_PUBLISHED = 'published'
    STATUS_CHOICES = [
        (STATUS_DRAFT, 'Draft'),
        (STATUS_PUBLISHED, 'Published'),
    ]

    title      = models.CharField(max_length=255)
    slug       = models.SlugField(unique=True)
    content    = models.TextField()
    excerpt    = models.TextField(blank=True)
    author     = models.ForeignKey(User, on_delete=models.CASCADE, related_name='posts')
    category   = models.ForeignKey(Category, on_delete=models.SET_NULL, null=True, blank=True)
    status     = models.CharField(max_length=10, choices=STATUS_CHOICES, default=STATUS_DRAFT)
    created_at = models.DateTimeField(auto_now_add=True)  # set once on creation
    updated_at = models.DateTimeField(auto_now=True)      # updated every save

    class Meta:
        ordering = ['-created_at']  # newest first

    def __str__(self):
        return self.title
```

### Common field types

| Field | Use case |
| ----- | -------- |
| `CharField(max_length=n)` | Short text |
| `TextField()` | Long text |
| `IntegerField()` | Integer |
| `FloatField()` | Decimal number |
| `BooleanField()` | True/False |
| `DateField()` | Date only |
| `DateTimeField()` | Date and time |
| `EmailField()` | Email (validates format) |
| `URLField()` | URL |
| `ImageField()` | Image upload |
| `ForeignKey()` | Many-to-one relationship |
| `ManyToManyField()` | Many-to-many relationship |
| `OneToOneField()` | One-to-one relationship |

### Migrations

```bash

# Apply migrations to the database
python manage.py migrate

# See migration status
python manage.py showmigrations
```

## Querying the Database

Django's ORM uses **QuerySets** — lazy, chainable queries that translate to SQL.

```python
from blog.models import Post, Category

# Get all posts
posts = Post.objects.all()

# Filter
published = Post.objects.filter(status='published')
drafts_by_alice = Post.objects.filter(status='draft', author__username='alice')

# Lookups (double underscore = field traversal or lookup type)
Post.objects.filter(title__icontains='django')       # case-insensitive contains
Post.objects.filter(created_at__year=2025)           # year part of date
Post.objects.filter(author__email='alice@example.com')  # follow FK relationship

# Exclude
Post.objects.exclude(status='draft')

# Get a single object (raises exception if not found or multiple)
post = Post.objects.get(slug='my-first-post')

# get_or_404 in views (raises 404 response instead of exception)
from django.shortcuts import get_object_or_404
post = get_object_or_404(Post, slug=slug, status='published')

# Ordering
Post.objects.order_by('-created_at')   # newest first
Post.objects.order_by('title')         # alphabetical

# Limit
Post.objects.all()[:10]       # first 10
Post.objects.all()[10:20]     # rows 11-20

# Count
Post.objects.filter(status='published').count()

# Exists
Post.objects.filter(status='draft').exists()  # True or False

# Create
post = Post.objects.create(title='Hello', slug='hello', content='...', author=user)

# Update
Post.objects.filter(status='draft').update(status='published')

# Delete
Post.objects.filter(status='draft').delete()

# Related objects (following FK backwards)
user.posts.all()              # all posts by this user (via related_name='posts')
user.posts.filter(status='published')
```

## Views

A **view** is a Python function or class that receives an HTTP request and returns an HTTP response.

### Function-based views (FBVs)

```python
# blog/views.py
from django.shortcuts import render, get_object_or_404, redirect
from django.contrib.auth.decorators import login_required
from .models import Post
from .forms import PostForm

def post_list(request):
    posts = Post.objects.filter(status='published').select_related('author', 'category')
    return render(request, 'blog/post_list.html', {'posts': posts})

def post_detail(request, slug):
    post = get_object_or_404(Post, slug=slug, status='published')
    return render(request, 'blog/post_detail.html', {'post': post})

@login_required
def post_create(request):
    if request.method == 'POST':
        form = PostForm(request.POST)
        if form.is_valid():
            post = form.save(commit=False)
            post.author = request.user
            post.save()
            return redirect('post_detail', slug=post.slug)
    else:
        form = PostForm()
    return render(request, 'blog/post_form.html', {'form': form})
```

### Class-based views (CBVs)

CBVs provide common patterns (list, detail, create, update, delete) with less boilerplate:

```python
from django.views.generic import ListView, DetailView, CreateView, UpdateView, DeleteView
from django.contrib.auth.mixins import LoginRequiredMixin
from django.urls import reverse_lazy

class PostListView(ListView):
    model = Post
    template_name = 'blog/post_list.html'
    context_object_name = 'posts'
    paginate_by = 10

    def get_queryset(self):
        return Post.objects.filter(status='published').select_related('author')

class PostDetailView(DetailView):
    model = Post
    template_name = 'blog/post_detail.html'

class PostCreateView(LoginRequiredMixin, CreateView):
    model = Post
    fields = ['title', 'slug', 'content', 'category', 'status']
    template_name = 'blog/post_form.html'

    def form_valid(self, form):
        form.instance.author = self.request.user
        return super().form_valid(form)

class PostDeleteView(LoginRequiredMixin, DeleteView):
    model = Post
    success_url = reverse_lazy('post_list')
```

## URL Routing

```python
# blog/urls.py
from django.urls import path
from . import views

urlpatterns = [
    path('', views.PostListView.as_view(), name='post_list'),
    path('<slug:slug>/', views.PostDetailView.as_view(), name='post_detail'),
    path('new/', views.PostCreateView.as_view(), name='post_create'),
    path('<slug:slug>/delete/', views.PostDeleteView.as_view(), name='post_delete'),
]
```

```python
# mysite/urls.py — include the app's URLs
from django.contrib import admin
from django.urls import path, include

urlpatterns = [
    path('admin/', admin.site.urls),
    path('blog/', include('blog.urls')),
]
```

URL patterns use **path converters** to capture typed values:

| Converter | Matches | Example |
| --------- | ------- | ------- |
| `<int:id>` | Integer | `42` |
| `<slug:slug>` | Slug (letters, numbers, hyphens) | `my-post` |
| `<str:name>` | Any string without `/` | `alice` |
| `<uuid:pk>` | UUID | `abc123-...` |

## Templates

Django templates use a simple tag language to render HTML with dynamic data.

```html
<!-- blog/templates/blog/post_list.html -->
{% extends "base.html" %}

{% block title %}Blog Posts{% endblock %}

{% block content %}
  <h1>Blog Posts</h1>

  {% for post in posts %}
    <article>
      <h2><a href="{% url 'post_detail' slug=post.slug %}">{{ post.title }}</a></h2>
      <p>By {{ post.author.username }} — {{ post.created_at|date:"M d, Y" }}</p>
      <p>{{ post.excerpt }}</p>
    </article>
  {% empty %}
    <p>No posts yet.</p>
  {% endfor %}

  <!-- Pagination -->
  {% if is_paginated %}
    {% if page_obj.has_previous %}
      <a href="?page={{ page_obj.previous_page_number }}">Previous</a>
    {% endif %}
    Page {{ page_obj.number }} of {{ page_obj.num_pages }}
    {% if page_obj.has_next %}
      <a href="?page={{ page_obj.next_page_number }}">Next</a>
    {% endif %}
  {% endif %}
{% endblock %}
```

```html
<!-- templates/base.html -->
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>{% block title %}My Site{% endblock %}</title>
</head>
<body>
  <nav>
    {% if user.is_authenticated %}
      Hello, {{ user.username }}! <a href="{% url 'logout' %}">Log out</a>
    {% else %}
      <a href="{% url 'login' %}">Log in</a>
    {% endif %}
  </nav>

  {% block content %}{% endblock %}
</body>
</html>
```

### Common template tags and filters

```django
{{ variable }}                    {# output a variable #}
{{ post.title|upper }}            {# filter: convert to uppercase #}
{{ post.created_at|date:"Y-m-d" }}
{{ post.content|truncatewords:50 }}
{{ post.content|safe }}           {# render HTML, not escaped — only for trusted content #}

{% if condition %}...{% elif %}...{% else %}...{% endif %}
{% for item in list %}...{% empty %}...{% endfor %}
{% url 'view_name' arg1 arg2 %}   {# reverse URL lookup #}
{% include "partials/card.html" %}
{% csrf_token %}                  {# required in every POST form #}
{% static 'css/style.css' %}      {# reference a static file #}
```

## The Admin Panel

Django's admin is one of its killer features — a fully functional CRUD interface auto-generated from your models, free of charge.

```python
# blog/admin.py
from django.contrib import admin
from .models import Post, Category

@admin.register(Category)
class CategoryAdmin(admin.ModelAdmin):
    list_display = ['name', 'slug']
    prepopulated_fields = {'slug': ('name',)}  # auto-fill slug from name

@admin.register(Post)
class PostAdmin(admin.ModelAdmin):
    list_display  = ['title', 'author', 'status', 'created_at']
    list_filter   = ['status', 'category', 'created_at']
    search_fields = ['title', 'content']
    prepopulated_fields = {'slug': ('title',)}
    raw_id_fields = ['author']  # use a search popup instead of a dropdown
    date_hierarchy = 'created_at'
    ordering = ['-created_at']
```

## Forms

Django forms handle validation, rendering, and saving data.

```python
# blog/forms.py
from django import forms
from .models import Post

class PostForm(forms.ModelForm):
    class Meta:
        model = Post
        fields = ['title', 'slug', 'content', 'excerpt', 'category', 'status']
        widgets = {
            'content': forms.Textarea(attrs={'rows': 15}),
        }

    def clean_slug(self):
        slug = self.cleaned_data['slug']
        if Post.objects.filter(slug=slug).exclude(pk=self.instance.pk).exists():
            raise forms.ValidationError("A post with this slug already exists.")
        return slug
```

```python
# Check permissions
@login_required
def delete_post(request, pk):
    post = get_object_or_404(Post, pk=pk)
```

## Django REST Framework (DRF) — Building APIs

When using Django as an API backend (instead of for server-rendered templates), **Django REST Framework** is the standard add-on.

```bash
pip install djangorestframework
```

```python
# settings.py
INSTALLED_APPS = [
    ...
    'rest_framework',
]
```

```python
# blog/serializers.py
from rest_framework import serializers
from .models import Post

class PostSerializer(serializers.ModelSerializer):
    author_name = serializers.CharField(source='author.username', read_only=True)

    class Meta:
        model = Post
        fields = ['id', 'title', 'slug', 'content', 'author_name', 'status', 'created_at']
        read_only_fields = ['id', 'created_at']
```

```python
# blog/views.py — API views
from rest_framework import generics, permissions
from .models import Post
from .serializers import PostSerializer

class PostListCreateView(generics.ListCreateAPIView):
    serializer_class = PostSerializer
    permission_classes = [permissions.IsAuthenticatedOrReadOnly]

    def get_queryset(self):
        return Post.objects.filter(status='published').select_related('author')

    def perform_create(self, serializer):
        serializer.save(author=self.request.user)

class PostDetailView(generics.RetrieveUpdateDestroyAPIView):
    serializer_class = PostSerializer
    lookup_field = 'slug'

    def get_queryset(self):
        return Post.objects.filter(status='published')
```

DRF auto-generates a browsable API at your endpoints — visit them in a browser to explore your API interactively.
