# MongoDB

---

## Why MongoDB Instead of SQL?

SQL databases like PostgreSQL store data in rigid tables with a fixed schema — every row must have the same columns. MongoDB stores data as flexible **documents** (JSON-like objects), where each document can have its own structure.

Neither is universally better. The right choice depends on your data.

| | PostgreSQL (SQL) | MongoDB |
| - | ---------------- | ------- |
| Data model | Tables with rows and columns | Collections of JSON documents |
| Schema | Enforced — all rows follow the same structure | Flexible — each document can differ |
| Relationships | Joins between tables | Embed data or reference by ID |
| Transactions | Full ACID transactions | Multi-document transactions (since v4) |
| Query language | SQL | MongoDB Query Language (MQL) |
| Best for | Structured, relational data (orders, users, finance) | Variable-structure data, rapid iteration, nested objects |

**Choose MongoDB when:**
- Your data is naturally document-shaped (a product with varying attributes, a user profile with optional nested fields)
- You're building fast-moving projects where the schema changes often
- You need to store data that doesn't fit neatly into tables (content management, catalogs, event logs)
- You want to scale horizontally across many servers (MongoDB shards easily)

**Choose PostgreSQL when:**
- Your data is relational — you need many joins
- Data integrity and ACID transactions are critical (financial data, inventory)
- You need complex aggregations and reporting

Many real-world applications use **both**: PostgreSQL for core business data (orders, payments, users) and MongoDB for less structured data (activity logs, product metadata, CMS content).

---

## What Is MongoDB?

MongoDB is a **document-oriented NoSQL database**. Instead of rows in a table, you store **documents** — BSON (Binary JSON) objects that look like JavaScript objects. Documents are grouped into **collections** (analogous to tables).

```
SQL:     database  → table      → row
MongoDB: database  → collection → document
```

A document can contain nested objects and arrays — you can store a blog post and all its comments in a single document rather than joining two tables.

## Defining Models with Mongoose

Mongoose adds a schema layer on top of MongoDB's flexible documents. This gives you validation and structure while keeping MongoDB's flexibility.

```javascript
import mongoose from 'mongoose';

const { Schema, model } = mongoose;

// A schema defines the shape and validation of documents
const userSchema = new Schema({
  name:      { type: String, required: true, trim: true },
  email:     { type: String, required: true, unique: true, lowercase: true },
  age:       { type: Number, min: 0 },
  role:      { type: String, enum: ['user', 'admin'], default: 'user' },
  active:    { type: Boolean, default: true },
  createdAt: { type: Date, default: Date.now },

  // Nested object — no separate table needed
  address: {
    street: String,
    city:   String,
    country: { type: String, default: 'UK' },
  },

  // Array of strings
  tags: [String],
});

// A model compiles the schema into a collection
const User = model('User', userSchema);
export default User;
```

```javascript
const postSchema = new Schema({
  title:     { type: String, required: true },
  content:   { type: String, required: true },
  slug:      { type: String, required: true, unique: true },
  status:    { type: String, enum: ['draft', 'published'], default: 'draft' },

  // Reference to another document (like a foreign key)
  author:    { type: Schema.Types.ObjectId, ref: 'User', required: true },

  // Embedded array of comment objects — stored inside the post document
  comments: [{
    user:      { type: Schema.Types.ObjectId, ref: 'User' },
    text:      String,
    createdAt: { type: Date, default: Date.now },
  }],

  createdAt: { type: Date, default: Date.now },
  updatedAt: { type: Date, default: Date.now },
}, {
  timestamps: true,  // automatically manages createdAt and updatedAt
});

// Index for faster slug lookups
postSchema.index({ slug: 1 });
postSchema.index({ status: 1, createdAt: -1 });  // compound index

const Post = model('Post', postSchema);
```

## CRUD Operations

### Create

```javascript
// Create one document
const user = await User.create({
  name: 'Alice',
  email: 'alice@example.com',
  address: { city: 'London', country: 'UK' },
  tags: ['developer', 'typescript'],
});

console.log(user._id);  // MongoDB auto-generated ObjectId

// Or: new + save
const user = new User({ name: 'Bob', email: 'bob@example.com' });
await user.save();
```

### Read

```javascript
// Find all documents
const users = await User.find();

// Find with filter
const activeUsers = await User.find({ active: true });

// Find with multiple conditions
const admins = await User.find({ active: true, role: 'admin' });

// Find one
const user = await User.findOne({ email: 'alice@example.com' });

// Find by ID
const user = await User.findById('64abc123...');

// Projection — only return certain fields (1 = include, 0 = exclude)
const users = await User.find({}, { name: 1, email: 1, _id: 0 });

// Sort, skip, limit (for pagination)
const page = 2;
const limit = 10;
const posts = await Post.find({ status: 'published' })
  .sort({ createdAt: -1 })   // newest first
  .skip((page - 1) * limit)
  .limit(limit);

// Populate referenced documents (like a JOIN)
const posts = await Post.find({ status: 'published' })
  .populate('author', 'name email')  // only fetch name and email from User
  .sort({ createdAt: -1 });
```

### Query operators

```javascript
// Comparison
User.find({ age: { $gt: 18 } });           // greater than
User.find({ age: { $gte: 18, $lte: 65 } }); // between
User.find({ role: { $in: ['admin', 'moderator'] } });
User.find({ role: { $ne: 'banned' } });    // not equal

// Text search (requires text index)
Post.find({ $text: { $search: 'mongodb tutorial' } });

// Regex
User.find({ email: { $regex: /@gmail\.com$/ } });

// Array operators
User.find({ tags: 'developer' });              // array contains this value
User.find({ tags: { $all: ['ts', 'react'] } }); // array contains all of these
User.find({ tags: { $size: 3 } });             // array has exactly 3 items

// Logical
User.find({ $or: [{ role: 'admin' }, { age: { $gt: 60 } }] });
User.find({ $and: [{ active: true }, { age: { $gte: 18 } }] });
```

### Update

```javascript
// Update one document
await User.updateOne(
  { email: 'alice@example.com' },    // filter
  { $set: { name: 'Alice Smith' } }  // update
);

// Update many
await User.updateMany({ active: false }, { $set: { role: 'user' } });

// Find and update — returns the document
const updated = await User.findByIdAndUpdate(
  userId,
  { $set: { name: 'Alice Smith' } },
  { new: true }  // return updated document, not the original
);

// Update operators
await User.updateOne({ _id: id }, {
  $set:   { name: 'Alice' },           // set field value
  $unset: { age: '' },                 // remove field
  $inc:   { loginCount: 1 },           // increment number
  $push:  { tags: 'nodejs' },          // add to array
  $pull:  { tags: 'old-tag' },         // remove from array
  $addToSet: { tags: 'typescript' },   // add to array only if not present
});
```

### Delete

```javascript
await User.deleteOne({ email: 'alice@example.com' });
await User.deleteMany({ active: false });
const deleted = await User.findByIdAndDelete(userId);
```

## Aggregation Pipeline

The aggregation pipeline processes documents through a series of stages — each stage transforms the data and passes it to the next. Equivalent to SQL's `GROUP BY`, `JOIN`, `WHERE`, and window functions combined.

```javascript
// Total revenue per user for the last 30 days
const results = await Order.aggregate([
  // Stage 1: filter
  {
    $match: {
      createdAt: { $gte: new Date(Date.now() - 30 * 24 * 60 * 60 * 1000) },
      status: 'completed',
    },
  },
  // Stage 2: group by user and sum amounts
  {
    $group: {
      _id: '$userId',
      totalRevenue: { $sum: '$amount' },
      orderCount:   { $sum: 1 },
      avgOrder:     { $avg: '$amount' },
    },
  },
  // Stage 3: join with users collection
  {
    $lookup: {
      from:         'users',
      localField:   '_id',
      foreignField: '_id',
      as:           'user',
    },
  },
  // Stage 4: flatten the joined array
  { $unwind: '$user' },
  // Stage 5: shape the output
  {
    $project: {
      userName:     '$user.name',
      totalRevenue: 1,
      orderCount:   1,
      avgOrder:     { $round: ['$avgOrder', 2] },
    },
  },
  // Stage 6: sort by revenue
  { $sort: { totalRevenue: -1 } },
  // Stage 7: top 10
  { $limit: 10 },
]);
```

### Common pipeline stages

| Stage | Purpose |
| ----- | ------- |
| `$match` | Filter documents (like `WHERE`) |
| `$group` | Group and aggregate (like `GROUP BY`) |
| `$project` | Shape output fields (like `SELECT`) |
| `$sort` | Sort results |
| `$limit` / `$skip` | Pagination |
| `$lookup` | Join another collection |
| `$unwind` | Flatten an array field into separate documents |
| `$addFields` | Add computed fields |
| `$count` | Count matching documents |

## Indexes

Without indexes, MongoDB scans every document in a collection (called a **collection scan**). Indexes make queries fast at the cost of slightly slower writes.

```javascript
// In Mongoose schema definition
postSchema.index({ slug: 1 });             // single field
postSchema.index({ status: 1, createdAt: -1 }); // compound
postSchema.index({ title: 'text', content: 'text' }); // text search index

// Create programmatically
await Post.createIndexes();
```

Always add indexes to fields used in:

- `.find({ field: value })` — filter fields
- `.sort({ field: 1 })` — sort fields
- `.populate()` source fields

## Using MongoDB with Next.js / Node.js

A typical Next.js + MongoDB setup:

```javascript
// lib/db.js — connection singleton
import mongoose from 'mongoose';

let isConnected = false;

export async function connectDB() {
  if (isConnected) return;

  await mongoose.connect(process.env.MONGODB_URI, {
    dbName: 'myapp',
  });
  isConnected = true;
}
```

```javascript
// app/api/posts/route.js (Next.js App Router)
import { connectDB } from '@/lib/db';
import Post from '@/models/Post';

export async function GET() {
  await connectDB();
  const posts = await Post.find({ status: 'published' })
    .populate('author', 'name')
    .sort({ createdAt: -1 })
    .limit(20);
  return Response.json(posts);
}

export async function POST(request) {
  await connectDB();
  const data = await request.json();
  const post = await Post.create(data);
  return Response.json(post, { status: 201 });
}
```
