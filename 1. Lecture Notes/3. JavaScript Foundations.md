## Introduction to JavaScript

JavaScript is the programming language of the web. While HTML defines what is on a page and CSS controls how it looks, JavaScript controls what the page *does*. It lets you respond to user actions, update content without reloading, validate forms, fetch data from servers, build games, and much more.

JavaScript runs directly in the browser — no installation needed to use it on a webpage. It can also run on a server using Node.js, which is covered below.

### The three languages of the web

| Language | What it does | Example |
| -------- | ------------ | ------- |
| **HTML** | Structure and content | A button element on the page |
| **CSS** | Appearance and layout | The button is blue with rounded corners |
| **JavaScript** | Behaviour and interactivity | Clicking the button shows a popup |

## 1. Installing Node.js

Node.js is a JavaScript runtime — a program that lets you run JavaScript directly on your computer, outside of a browser. It also comes bundled with **npm** (Node Package Manager), which is used to install libraries and tools built by other developers.

You do not need Node.js to write JavaScript for a browser, but you will need it for most modern web development workflows.

### How to install

1. Go to [nodejs.org](https://nodejs.org) and download the **LTS** (Long-Term Support) version. LTS versions are the most stable and are recommended for most developers.
2. Run the installer and follow the prompts.
3. Open your terminal and verify it installed correctly:

```bash
node --version   # prints something like: v20.11.0
npm --version    # prints something like: 10.2.4
```

### Running JavaScript with Node

Once installed, you can run any JavaScript file from the terminal:

```bash
node myFile.js
```

You can also open an **interactive REPL** (Read-Eval-Print Loop) by just typing `node` with no arguments. This is a live JavaScript prompt where you can type and run code immediately — great for quickly testing a snippet.

```bash
node
> 2 + 2
4
> "hello".toUpperCase()
'HELLO'
```

Press `Ctrl + C` twice to exit the REPL.

## 2. Variables and Operators

### What is a variable?

A variable is a named container that stores a value. You give it a name, assign a value to it, and then use that name to refer to the value later in your code.

```js
let age = 25;
console.log(age); // 25

age = 26;         // the value can be updated
console.log(age); // 26
```

### Declaring Variables

JavaScript has three keywords for creating variables:

| Keyword | Scope | Can be reassigned? | Use today? |
| ------- | ----- | ------------------ | ---------- |
| `const` | Block | No | Yes — your default choice |
| `let` | Block | Yes | Yes — when the value will change |
| `var` | Function | Yes | No — old, avoid it |

```js
// const — use this by default
const name = "Alice";
const pi = 3.14159;
// name = "Bob"; // Error! You cannot reassign a const

// let — use when you know the value will be reassigned
let score = 0;
score = 10;   // this is fine
score += 5;   // also fine — score is now 15

// var — old behaviour, causes confusing bugs, never use it
var oldWay = true; // avoid
```

**Why default to `const`?** It makes your code easier to reason about. When you see `const`, you know the value never changes. Only switch to `let` when you genuinely need to reassign.

### Naming Variables

Good names make code readable. Follow these conventions:

- Use **camelCase**: `userName`, `totalScore`, `isLoggedIn`
- Names are **case-sensitive**: `score` and `Score` are two different variables
- Cannot start with a number: `1score` is invalid
- Use descriptive names: `userAge` is better than `a`, `ua`, or `x`

```js
// Bad names
let x = 30;
let fn = "Alice";
let d = new Date();

// Good names
let userAge = 30;
let firstName = "Alice";
let currentDate = new Date();
```

### Operators

Operators let you perform calculations, compare values, and combine conditions.

#### Arithmetic operators

```js
10 + 3   // 13  — addition
10 - 3   // 7   — subtraction
10 * 3   // 30  — multiplication
10 / 4   // 2.5 — division
10 % 3   // 1   — modulo (remainder after division — useful for even/odd checks)
2 ** 3   // 8   — exponentiation (2 to the power of 3)
```

The **modulo** operator (`%`) is especially useful. `number % 2 === 0` tells you if a number is even.

#### Assignment operators

These are shortcuts that update a variable in place:

```js
let x = 10;

x += 5;   // same as: x = x + 5  → x is now 15
x -= 3;   // same as: x = x - 3  → x is now 12
x *= 2;   // same as: x = x * 2  → x is now 24
x /= 4;   // same as: x = x / 4  → x is now 6
x++;      // same as: x = x + 1  → x is now 7 (increment by 1)
x--;      // same as: x = x - 1  → x is now 6 (decrement by 1)
```

#### Comparison operators

Comparison operators always return `true` or `false`.

```js
5 === 5      // true  — strict equality: same value AND same type
5 !== 3      // true  — strict inequality
"5" === 5    // false — string "5" is not the same type as number 5
5 == "5"     // true  — loose equality: ignores type (avoid this)

5 > 3        // true  — greater than
5 < 3        // false — less than
5 >= 5       // true  — greater than or equal to
5 <= 4       // false — less than or equal to
```

> **Always use `===` and `!==`** instead of `==` and `!=`. Loose equality (`==`) does automatic type conversion which leads to confusing results like `0 == false` being `true`.

#### Logical operators

Used to combine multiple conditions:

```js
// AND (&&) — true only if BOTH sides are true
true && true    // true
true && false   // false

// OR (||) — true if AT LEAST ONE side is true
true || false   // true
false || false  // false

// NOT (!) — flips a boolean
!true           // false
!false          // true

// Real example:
const age = 20;
const hasTicket = true;

if (age >= 18 && hasTicket) {
  console.log("You may enter.");
}
```

## 3. Data Types and Conditionals

### What is a data type?

Every value in JavaScript has a type. The type determines what kind of value it is and what you can do with it.

### Primitive Data Types

These are the basic building blocks of data in JavaScript:

| Type | Example values | Notes |
| ---- | -------------- | ----- |
| `String` | `"hello"`, `'world'`, `` `name` `` | Text. Can use single quotes, double quotes, or backticks. |
| `Number` | `42`, `3.14`, `-7`, `0` | All numbers — integers and decimals — use one type. |
| `Boolean` | `true`, `false` | Used for logic and conditions. Only two possible values. |
| `Undefined` | `undefined` | A variable that has been declared but not yet given a value. |
| `Null` | `null` | An intentional "no value". Explicitly set by the developer. |
| `BigInt` | `9007199254740991n` | Integers too large for `Number` to handle precisely. |
| `Symbol` | `Symbol("id")` | Unique identifiers, mostly used in advanced patterns. |

```js
// You can check the type of any value using typeof
typeof "hello"      // "string"
typeof 42           // "number"
typeof true         // "boolean"
typeof undefined    // "undefined"
typeof null         // "object"  ← this is a known bug in JavaScript; null is not an object
typeof [1, 2, 3]    // "object"
typeof function(){} // "function"
```

**`undefined` vs `null`:**

- `undefined` means a variable exists but has never been assigned a value — JavaScript sets this automatically.
- `null` means "no value" and is something you set on purpose. It signals intentional emptiness.

```js
let city;            // declared but never assigned — JavaScript gives it undefined
console.log(city);   // undefined

let nickname = null; // intentionally set to null — "this person has no nickname"
```

### Strings in Detail

Strings are text values. You can create them with single quotes, double quotes, or backticks. Backtick strings (called **template literals**) are the most powerful because they let you embed JavaScript expressions directly inside the string.

```js
// All three are valid strings
const a = 'Hello';
const b = "Hello";
const c = `Hello`;

// Template literals — embed values with ${}
const name = "Alice";
const age = 30;
console.log(`My name is ${name} and I am ${age} years old.`);
// → My name is Alice and I am 30 years old.

// You can put any JavaScript expression inside ${}
console.log(`2 + 2 = ${2 + 2}`);
// → 2 + 2 = 4

// Template literals can also span multiple lines naturally
const message = `Dear Alice,

Welcome to the site!

Regards,
The Team`;
```

**Useful string methods:**

```js
const greeting = "Hello, World!";

greeting.length           // 13 — number of characters
greeting.toUpperCase()    // "HELLO, WORLD!"
greeting.toLowerCase()    // "hello, world!"
greeting.includes("World") // true — does it contain this text?
greeting.startsWith("Hello") // true
greeting.endsWith("!")       // true
greeting.indexOf("World")    // 7 — position of the first character of "World"
greeting.slice(7, 12)        // "World" — extract characters from index 7 up to 12
greeting.replace("World", "JavaScript") // "Hello, JavaScript!"
greeting.trim()              // removes whitespace from both ends
"  spaces  ".trim()          // "spaces"
"a,b,c".split(",")           // ["a", "b", "c"] — splits into an array
```

### Truthy and Falsy

In JavaScript, every value has an inherent "truthiness". When you use a value in a condition (like an `if` statement), JavaScript automatically converts it to `true` or `false` if it's not already a boolean.

**Falsy values** — these all evaluate to `false` in a condition:

- `false`
- `0`
- `""` (empty string)
- `null`
- `undefined`
- `NaN` (Not a Number)

**Everything else is truthy**, including:

- Any non-zero number (`1`, `-1`, `0.5`)
- Any non-empty string (`"hello"`, `"0"`, `"false"`)
- Arrays and objects (even empty ones: `[]`, `{}`)

```js
if ("hello") {
  console.log("This runs — non-empty string is truthy");
}

if (0) {
  console.log("This never runs — 0 is falsy");
}

// Practical example: check if a username was provided
const username = "";
if (!username) {
  console.log("Please enter a username");
}
```

### Conditionals

Conditionals let your code make decisions — running different code depending on whether something is true or false.

#### `if / else if / else`

The most fundamental way to branch your logic:

```js
const score = 75;

if (score >= 90) {
  console.log("Grade: A");
} else if (score >= 70) {
  console.log("Grade: B");  // this runs because 75 >= 70
} else if (score >= 50) {
  console.log("Grade: C");
} else {
  console.log("Grade: Fail");
}
```

The `else if` and `else` blocks are optional. Once a condition is met, the rest of the chain is skipped.

#### Ternary Operator

A compact shorthand for a simple `if/else`. Best used when you're choosing between two values, not when you're running complex blocks of code.

```js
// Syntax: condition ? valueIfTrue : valueIfFalse
const age = 20;
const access = age >= 18 ? "Allowed" : "Denied";
console.log(access); // "Allowed"

// Equivalent if/else:
let access;
if (age >= 18) {
  access = "Allowed";
} else {
  access = "Denied";
}
```

#### `switch`

Useful when you want to compare one variable against many specific values. Cleaner than a long chain of `else if`:

```js
const day = "Monday";

switch (day) {
  case "Saturday":
  case "Sunday":
    console.log("Weekend — rest!");
    break;  // break stops execution from "falling through" to the next case
  case "Monday":
    console.log("Start of the work week");
    break;
  case "Friday":
    console.log("Almost the weekend!");
    break;
  default:
    // default runs if no case matched
    console.log("Regular weekday");
}
```

> Always include `break` at the end of each `case` unless you intentionally want multiple cases to share the same block (like `Saturday` and `Sunday` above).

Your browser has powerful built-in tools for writing and debugging JavaScript. Learning to use them is just as important as learning the language itself.

### Opening DevTools

| Browser | Shortcut |
| ------- | -------- |
| Chrome / Edge | `F12` or `Ctrl + Shift + I` |
| Firefox | `F12` or `Ctrl + Shift + I` |
| Mac (any browser) | `Cmd + Option + I` |

### The Console Tab

The Console is where JavaScript output appears. You can also type and run JavaScript directly in it. This makes it a great scratchpad for experimenting.

```js
// These are the most useful console methods
```

## 5. Function Basics

A function is a named, reusable block of code. Instead of writing the same logic in multiple places, you write it once in a function and call it whenever you need it.

### Why use functions?

```js
// Without a function — repeated code
console.log("Hello, Alice! Welcome.");
console.log("Hello, Bob! Welcome.");
console.log("Hello, Charlie! Welcome.");

// With a function — write once, use anywhere
function greet(name) {
  console.log(`Hello, ${name}! Welcome.`);
}

greet("Alice");    // Hello, Alice! Welcome.
greet("Bob");      // Hello, Bob! Welcome.
greet("Charlie");  // Hello, Charlie! Welcome.
```

### Function Declaration

The most common way to define a function. Function declarations are **hoisted** — the function is available throughout the entire file even if you call it before you define it.

```js
function add(a, b) {
  return a + b;
}

const result = add(3, 4);
console.log(result); // 7
```

### Function Expression

A function stored in a variable. Not hoisted — you must define it before using it.

```js
const multiply = function(a, b) {
  return a * b;
};

console.log(multiply(4, 5)); // 20
```

### Arrow Functions

A shorter, more modern syntax introduced in ES6. Most commonly used for short, simple functions.

```js
// Full arrow function syntax
const greet = (name) => {
  return `Hello, ${name}!`;
};

// Concise form — when the body is a single expression, omit the braces and 'return'
const double = (n) => n * 2;

// Single parameter — parentheses are optional
const square = n => n * n;

// No parameters — empty parentheses are required
const sayHello = () => "Hello!";

console.log(double(5));    // 10
console.log(square(4));    // 16
console.log(sayHello());   // Hello!
```

### Parameters vs. Arguments

These two words describe the same thing from different perspectives:

- **Parameters** are the placeholder names listed in the function definition.
- **Arguments** are the actual values passed in when you call the function.

```js
//              a and b are PARAMETERS
function add(a, b) {
  return a + b;
}

//       3 and 4 are ARGUMENTS
add(3, 4); // → 7
```

### Default Parameter Values

If a caller doesn't provide an argument, a default value is used instead:

```js
function greet(name = "stranger") {
  return `Hello, ${name}!`;
}

greet("Alice");   // "Hello, Alice!"
greet();          // "Hello, stranger!"  ← uses the default
```

### Return Values

A function sends a value back to the caller using `return`. If there is no `return` statement, the function returns `undefined` automatically.

```js
function calculateArea(width, height) {
  return width * height;
  // anything after return is never reached
}

const area = calculateArea(5, 10);
console.log(area); // 50

// A function without return
function logMessage(msg) {
  console.log(msg);
  // implicitly returns undefined
}

const result = logMessage("Hi");
console.log(result); // undefined
```

### Scope

**Scope** determines where in your code a variable can be accessed. Variables declared with `let` and `const` are **block scoped** — they only exist within the `{}` block they were declared in.

```js
function calculateTotal() {
  const tax = 0.2;    // tax only exists inside this function
  const price = 100;
  return price + price * tax;
}

console.log(tax); // ReferenceError — tax doesn't exist out here

// Block scope inside if/for/etc.
if (true) {
  const message = "Inside the block";
  console.log(message); // works fine
}
console.log(message); // ReferenceError — message is out of scope
```

## 6. Problem Solving

The hardest part of programming is not learning the syntax — it's figuring out *how to approach a problem*. Having a structured process is more valuable than memorising any language feature.

### A Repeatable Process

**1. Understand the problem**
Before writing a single line of code, make sure you fully understand what is being asked.

- What are the inputs? What are the expected outputs?
- Try to restate the problem in your own words.
- If you can't explain it simply, you don't understand it fully yet.

**2. Plan before you code**
Write out your approach in plain English (called **pseudo-code**) before writing actual code. This prevents you from going in circles.

**3. Divide the problem**
Break the big problem into smaller sub-problems. Solve each one independently. A complex problem is just a collection of simple problems.

**4. Implement**
Now write the actual code, one small step at a time. Run it frequently to catch errors early.

**5. Test thoroughly**
Does it work for normal inputs? What about edge cases — empty inputs, negative numbers, very large values?

**6. Refactor**
Once it works, review your code. Can you make it clearer, shorter, or more efficient?

---

### Pseudo-code Example

Problem: *Write a function that takes a sentence and returns the number of words in it.*

Start by thinking through the steps in plain English:

```text
function countWords(sentence):
  if sentence is empty:
    return 0
  split the sentence into a list of words by splitting on spaces
  return the length of that list
```

Then translate to JavaScript:

```js
function countWords(sentence) {
  if (sentence.trim() === "") {
    return 0;
  }
  const words = sentence.trim().split(" ");
  return words.length;
}

console.log(countWords("Hello world"));      // 2
console.log(countWords("I love JavaScript")); // 3
console.log(countWords(""));                 // 0
```

## 7. Understanding Errors

Errors are a normal and expected part of programming. Every developer, no matter how experienced, encounters them daily. The skill is learning to read and fix them quickly.

### Common Error Types

| Error | What it means | Common cause |
| ----- | ------------- | ------------ |
| `SyntaxError` | Your code is not valid JavaScript | Missing bracket, typo in a keyword, unclosed string |
| `ReferenceError` | You used a variable or function that doesn't exist | Typo in a name, used before declaring, out of scope |
| `TypeError` | You tried to do something with the wrong type of value | Calling a method that doesn't exist (e.g., `.toUpperCase()` on a number) |
| `RangeError` | A value is outside the allowed range | Infinite recursion, invalid array length |

### How to Read an Error Message

Error messages look scary but they contain everything you need:

```text
TypeError: Cannot read properties of undefined (reading 'name')
    at getUser (app.js:12:18)
    at main (app.js:25:3)
```

Read it like this:

1. **`TypeError`** — the type of error. A `TypeError` means you used the wrong type of value.
2. **`Cannot read properties of undefined (reading 'name')`** — the actual problem: something that should be an object is `undefined`, so you can't access `.name` on it.
3. **`at getUser (app.js:12:18)`** — go to line 12, column 18 of `app.js`. That's exactly where it broke.
4. **`at main (app.js:25:3)`** — `getUser` was called from line 25. The stack trace shows you the call chain.

Always start reading the stack trace from the **top**, which is where the error actually occurred.

---

### Debugging Strategies

**1. `console.log()` everything**
When something isn't working, print the values your code is using at each step. You will almost always find a value that isn't what you expected.

```js
function getDiscount(price, code) {
  console.log("price:", price);     // is it the right value?
  console.log("code:", code);       // is the code coming in correctly?
  const discount = code === "SAVE10" ? 0.1 : 0;
  console.log("discount:", discount);
  return price - price * discount;
}
```

**2. Set a breakpoint in DevTools**
Open the Sources tab, find your JS file, and click a line number to set a breakpoint. When the code reaches that line, it pauses. You can then inspect every variable's value and step through line by line.

**3. Rubber duck debugging**
Explain your code out loud — to a rubber duck, a colleague, or just yourself. Describe exactly what each line does. You will very often spot the bug while explaining it. This sounds silly but genuinely works.

**4. Read the error message carefully**
The error message tells you exactly what went wrong and where. Resist the urge to randomly change things. Read the message, find the line, understand what it's saying.

**5. Search the exact error message**
Copy the error message (without the filename/line number) and search for it. Somebody else has had this problem.

## 8. Clean Code

Clean code is code that is easy to read, understand, and change — not just for others, but for yourself when you return to it six months later.

### Naming

Names are the most important part of clean code. A good name tells you exactly what something is or does without needing a comment.

```js
// Bad — what does this do?
function proc(d, r) {
  return d * r * 0.01;
}

// Good — immediately clear
function calculateDiscountAmount(price, discountPercent) {
  return price * discountPercent * 0.01;
}
```

**Conventions to follow:**

- Variables and functions: `camelCase` — `userName`, `totalPrice`, `getUserData`
- Booleans should read like a yes/no question: `isLoggedIn`, `hasPermission`, `isEmpty`, `canEdit`
- Functions should be named after what they *do*, using a verb: `fetchUser`, `calculateTotal`, `renderCard`, `validateEmail`
- Constants that never change and are meaningful: some teams use `UPPER_SNAKE_CASE` for these — `MAX_RETRIES`, `API_URL`

### Functions Should Do One Thing

A function that does one thing is easy to name, test, and reuse. A function that does many things is hard to understand and hard to debug.

```js
// Bad — this function does too many things
function processUser(userData) {
  // validates the data
  if (!userData.email.includes("@")) throw new Error("Bad email");
  // saves to database
  database.save(userData);
  // sends a welcome email
  emailService.send(userData.email, "Welcome!");
  // logs to analytics
  analytics.track("user_registered");
}

// Good — each function does one thing
function validateUser(userData) {
  if (!userData.email.includes("@")) throw new Error("Bad email");
}

function registerUser(userData) {
  validateUser(userData);
  database.save(userData);
  emailService.sendWelcome(userData.email);
  analytics.track("user_registered");
}
```

### When to Write Comments

Good code mostly explains itself through clear naming. Write a comment only when the **why** is not obvious — a business rule, a workaround, a non-obvious constraint.

```js
// Don't comment obvious code:
let total = 0; // set total to zero  ← not helpful

// Do comment non-obvious decisions:
// Retry up to 3 times because the payment provider occasionally
// drops the first request under high load.
for (let attempt = 0; attempt < 3; attempt++) {
  const result = await chargeCard(amount);
  if (result.success) break;
}
```

### Formatting

Consistent formatting makes code easier to scan. Use a tool like **Prettier** to automate this — there's no reason to format code by hand.

- Use **2 spaces** for indentation (the standard in JavaScript).
- Keep lines under ~100 characters.
- Put spaces around operators: `x = a + b`, not `x=a+b`.
- Put a space before `{` in blocks: `if (x) {`, not `if (x){`.

## 9. Loops and Arrays

### Arrays

An array is an ordered list of values stored in a single variable. You access values by their **index** (position), which starts at `0`.

```js
const fruits = ["apple", "banana", "cherry", "date"];

fruits[0]   // "apple"   — first item (index 0)
fruits[2]   // "cherry"  — third item (index 2)
fruits[fruits.length - 1]  // "date" — last item

fruits.length  // 4 — total number of items
```

Think of an array like a numbered list. Position 0 is the first slot, position 1 is the second, and so on.

---

#### Modifying Arrays

```js
const items = ["a", "b", "c"];

// Adding items
items.push("d");      // adds to the END → ["a", "b", "c", "d"]
items.unshift("z");   // adds to the START → ["z", "a", "b", "c", "d"]

// Removing items
items.pop();          // removes and returns the LAST item → ["z", "a", "b", "c"]
items.shift();        // removes and returns the FIRST item → ["a", "b", "c"]

// Finding items
items.includes("b");  // true — does the array contain "b"?
items.indexOf("b");   // 1 — what index is "b" at? (-1 if not found)

// Creating new arrays from existing ones
items.slice(0, 2);    // ["a", "b"] — copies items from index 0 up to (not including) 2
["a", "b"].concat(["c", "d"]);  // ["a", "b", "c", "d"] — joins two arrays

// Transforming
items.reverse();      // reverses the array in place
[3, 1, 2].sort((a, b) => a - b); // [1, 2, 3] — sort numerically ascending
items.join(", ");     // "a, b, c" — combines all items into a string
```

---

#### Higher-Order Array Methods

These methods take a **callback function** as an argument and apply it to each item in the array. They are the standard, modern way to work with arrays.

```js
const numbers = [1, 2, 3, 4, 5];

// forEach — loop through every item. Returns nothing.
numbers.forEach((num) => {
  console.log(num); // prints 1, 2, 3, 4, 5
});

// map — transform every item. Returns a NEW array of the same length.
const doubled = numbers.map((num) => num * 2);
// [2, 4, 6, 8, 10]

// filter — keep only items that pass a test. Returns a NEW array (shorter or same length).
const evens = numbers.filter((num) => num % 2 === 0);
// [2, 4]

// find — returns the FIRST item that passes a test, or undefined if none do.
const firstOver3 = numbers.find((num) => num > 3);
// 4

// reduce — accumulate all items into a single value.
const total = numbers.reduce((accumulator, num) => accumulator + num, 0);
// 0 + 1 + 2 + 3 + 4 + 5 = 15

// some — returns true if AT LEAST ONE item passes the test.
numbers.some((num) => num > 4);   // true

// every — returns true only if ALL items pass the test.
numbers.every((num) => num > 0);  // true
numbers.every((num) => num > 3);  // false
```

**`map`, `filter`, and `reduce`** are the three most important. They replace most `for` loops in modern JavaScript and make the code's intent much clearer.

### Loops

#### `for` loop

The classic loop — use when you need to repeat something a known number of times or need access to the index.

```js
for (let i = 0; i < 5; i++) {
  console.log(i); // 0, 1, 2, 3, 4
}

// Using index to access array items
const colors = ["red", "green", "blue"];
for (let i = 0; i < colors.length; i++) {
  console.log(`Color ${i + 1}: ${colors[i]}`);
}
// Color 1: red
// Color 2: green
// Color 3: blue
```

#### `for...of` loop

The cleanest way to loop over an array when you don't need the index:

```js
const fruits = ["apple", "banana", "cherry"];

for (const fruit of fruits) {
  console.log(fruit);
}
// apple
// banana
// cherry
```

#### `while` loop

Repeats as long as a condition is true. Use when you don't know in advance how many iterations you need.

```js
let attempts = 0;

while (attempts < 3) {
  console.log(`Attempt ${attempts + 1}`);
  attempts++;
}
// Attempt 1
// Attempt 2
// Attempt 3
```

> Be careful with `while` loops — if the condition never becomes false, you get an **infinite loop** that freezes the browser. Always make sure something inside the loop will eventually make the condition false.

#### `for...in` loop

Iterates over the **keys** (property names) of an object:

```js
const person = { name: "Alice", age: 30, city: "London" };

for (const key in person) {
  console.log(`${key}: ${person[key]}`);
}
// name: Alice
// age: 30
// city: London
```

## 10. Object Basics

An object is a collection of **key-value pairs**. It is the primary way to group related data together in JavaScript. Think of it as a record: a `user` object groups together a user's name, age, email, and any other related information.

```js
const user = {
  name: "Alice",
  age: 30,
  email: "alice@example.com",
  isAdmin: false,
};
```

Each key-value pair is called a **property**. The key is the name, the value is the data.

### Accessing Properties

```js
// Dot notation — the preferred and most readable way
user.name    // "Alice"
user.age     // 30

// Bracket notation — needed when the key is a variable or contains special characters
user["email"]  // "alice@example.com"

const field = "age";
user[field]    // 30 — looks up the value at the key stored in 'field'
```

### Adding, Updating, and Deleting

Objects are mutable — you can change them after they're created, even if declared with `const`. (`const` prevents you from reassigning the variable itself, not from changing the object's contents.)

```js
const user = { name: "Alice", age: 30 };

user.age = 31;                  // update an existing property
user.city = "London";           // add a new property
delete user.age;                // remove a property

console.log(user); // { name: "Alice", city: "London" }
```

### Methods — Functions as Properties

When a function is stored as a property on an object, it is called a **method**. Methods describe what an object can *do*.

```js
const user = {
  name: "Alice",
  age: 30,
  greet() {
    return `Hi, I'm ${this.name} and I'm ${this.age} years old.`;
  },
  birthday() {
    this.age++;
  }
};

user.greet();    // "Hi, I'm Alice and I'm 30 years old."
user.birthday();
console.log(user.age); // 31
```

**`this`** inside a method refers to the object the method belongs to. It lets a method access other properties on the same object.

### Destructuring

Destructuring lets you extract specific properties from an object into individual variables in one line:

```js
const user = { name: "Alice", age: 30, city: "London" };

// Without destructuring
const name = user.name;
const age = user.age;

// With destructuring — much more concise
const { name, age } = user;
console.log(name); // "Alice"
console.log(age);  // 30

// Rename while destructuring
const { name: userName, age: userAge } = user;
console.log(userName); // "Alice"

// Set a default value if the property doesn't exist
const { country = "Unknown" } = user;
console.log(country); // "Unknown"
```

Destructuring is especially useful in function parameters:

```js
// Without destructuring
function displayUser(user) {
  console.log(`${user.name}, ${user.age}`);
}

// With destructuring in the parameter
function displayUser({ name, age }) {
  console.log(`${name}, ${age}`);
}
```

### Spread Operator

The `...` spread operator lets you copy or merge objects:

```js
const defaults = { theme: "light", language: "en", notifications: true };
const userPrefs = { theme: "dark", language: "fr" };

// Merge — later values overwrite earlier ones
const settings = { ...defaults, ...userPrefs };
// { theme: "dark", language: "fr", notifications: true }

// Copy an object (shallow copy)
const copy = { ...user };
```

### `Object` Utility Methods

```js
const person = { name: "Alice", age: 30, city: "London" };

Object.keys(person);     // ["name", "age", "city"] — array of property names
Object.values(person);   // ["Alice", 30, "London"] — array of values
Object.entries(person);  // [["name","Alice"], ["age",30], ["city","London"]]

// entries() is great for looping over an object
for (const [key, value] of Object.entries(person)) {
  console.log(`${key}: ${value}`);
}
```

## 11. DOM Manipulation and Events

The **DOM** (Document Object Model) is the browser's internal representation of the HTML page as a tree of objects. JavaScript can read and change this tree to update what the user sees — adding elements, changing text, responding to clicks, and much more — all without reloading the page.

Think of the DOM as a live version of your HTML that you can reach into and modify with JavaScript.

### Selecting Elements

Before you can change something on the page, you need to grab a reference to it:

```js
// Get one element by its ID — returns null if not found
const hero = document.getElementById("hero");

// Get the FIRST element matching a CSS selector
const card = document.querySelector(".card");
const title = document.querySelector("h1");
const firstItem = document.querySelector("ul > li:first-child");

// Get ALL elements matching a selector — returns a NodeList (like an array)
const allCards = document.querySelectorAll(".card");
const allLinks = document.querySelectorAll("a");

// Loop over a NodeList with forEach
allLinks.forEach((link) => {
  console.log(link.href);
});
```

### Reading and Changing Content

```js
const heading = document.querySelector("h1");

// Read the text content
console.log(heading.textContent); // "Welcome"

// Set the text content (safe — treats everything as plain text, never runs HTML)
heading.textContent = "Hello World";

// Set HTML content (more powerful but be careful — never insert untrusted user input
// this way, as it can execute malicious scripts)
heading.innerHTML = "Hello <em>World</em>";
```

### Changing Styles and Classes

Manipulating CSS classes is the preferred approach — it keeps your styles in the CSS file where they belong, and lets you change the appearance of an element with a single class toggle.

```js
const box = document.querySelector(".box");

// Add, remove, and toggle CSS classes
box.classList.add("active");           // adds the class
box.classList.remove("hidden");        // removes the class
box.classList.toggle("open");          // adds if absent, removes if present
box.classList.contains("active");      // true or false — check if a class exists

// Inline styles (use sparingly — prefer classes)
box.style.backgroundColor = "tomato";
box.style.width = "200px";
box.style.display = "none";
```

### Creating and Inserting Elements

```js
// Create a new element
const newItem = document.createElement("li");
newItem.textContent = "A new list item";
newItem.classList.add("item");

// Insert it into the page
const list = document.querySelector("ul");
list.appendChild(newItem);                   // add to the END of the list
list.prepend(newItem);                       // add to the START of the list
list.insertBefore(newItem, list.children[1]); // insert before a specific child

// Remove an element from the page
newItem.remove();
```

### Reading and Setting Attributes

```js
const link = document.querySelector("a");

link.getAttribute("href");              // get the value of an attribute
link.setAttribute("href", "/about");   // set an attribute
link.removeAttribute("disabled");       // remove an attribute
link.hasAttribute("class");             // true or false
```

### Events

An event is anything that happens in the browser — a click, a keypress, the page loading, a form being submitted. JavaScript listens for these events and runs a function (called a **handler** or **listener**) in response.

#### Adding an Event Listener

```js
const button = document.querySelector("button");

// addEventListener(eventType, handlerFunction)
button.addEventListener("click", () => {
  console.log("The button was clicked!");
});

// You can also store the handler in a named function
function handleClick() {
  console.log("Clicked!");
}
button.addEventListener("click", handleClick);

// And remove it later
button.removeEventListener("click", handleClick);
```

#### The Event Object

When an event fires, JavaScript automatically passes an **event object** into your handler. It contains information about what happened.

```js
button.addEventListener("click", (event) => {
  console.log(event.target);       // the element that was actually clicked
  console.log(event.type);         // "click"
  console.log(event.clientX);      // x position of the mouse at click time

  event.preventDefault();          // stop the browser's default action
                                    // (e.g., stop a link from navigating, or stop a form from submitting)

  event.stopPropagation();          // stop the event from bubbling up to parent elements
});
```

#### Common Event Types

| Event | When it fires |
| ----- | ------------- |
| `click` | The element is clicked |
| `dblclick` | The element is double-clicked |
| `mouseover` | The cursor moves over the element |
| `mouseout` | The cursor leaves the element |
| `keydown` | A keyboard key is pressed |
| `keyup` | A keyboard key is released |
| `submit` | A form is submitted |
| `input` | The value of an input field changes with each keystroke |
| `change` | An input value is committed (e.g., leaving a text field after typing) |
| `focus` | An element gains keyboard focus |
| `blur` | An element loses focus |
| `DOMContentLoaded` | The HTML has fully loaded and the DOM is ready to use |
| `load` | The page and all its resources (images, etc.) have fully loaded |

```js
// Wait for the DOM to be ready before running your code
document.addEventListener("DOMContentLoaded", () => {
  // safe to access and manipulate the DOM here
  const button = document.querySelector("button");
  button.addEventListener("click", () => console.log("Clicked!"));
});

// Read what the user is typing in real time
const input = document.querySelector("input");
input.addEventListener("input", (e) => {
  console.log("Current value:", e.target.value);
});
```

#### Event Bubbling and Delegation

When an event fires on an element, it **bubbles up** — it also fires on every parent element all the way up to `document`. You can use this to your advantage.

**Event delegation** means attaching one listener to a parent element instead of many listeners to each child. This is more efficient and works even for elements added to the page after the listener was set up.

```js
// Without delegation — one listener per item (bad for dynamic lists)
document.querySelectorAll("li").forEach((li) => {
  li.addEventListener("click", (e) => console.log(e.target.textContent));
});

// With delegation — one listener on the parent (much better)
document.querySelector("ul").addEventListener("click", (e) => {
  if (e.target.tagName === "LI") {
    console.log("You clicked:", e.target.textContent);
  }
});
```

| - | ----- | ----- | ------- |

| Hoisted? | Yes (as `undefined`) | Yes (but not usable — "temporal dead zone") | Yes (but not usable) |

## Quick Reference: Choosing the Right Loop

| Loop | Best for |
| ---- | -------- |
| `array.forEach()` | Doing something with each item, no return value needed |
| `array.map()` | Transforming each item into something else |
| `array.filter()` | Keeping only items that match a condition |
| `array.reduce()` | Combining all items into one value (sum, count, etc.) |
| `for...of` | Iterating over an array when you need `break` or `continue` |
| `for` | When you need the index, or a specific number of iterations |
| `while` | When you don't know in advance how many iterations you need |
| `for...in` | Iterating over the keys of an object |
